## Problem

The docs/wiki site has its own branding and colour scheme that is not aligned with the dashboard UI. Brand identity should be consistent across both surfaces — the dashboard is the primary interface and should be the source of truth for colours, typography, and visual identity.

## Solution

1. Identify the colour palette, fonts, and key brand variables used in the dashboard (`puppeteer/dashboard/src/`)
2. Apply matching theme to the docs/wiki site (colours, fonts, logo treatment)
3. Ensure navigation, headings, and code blocks follow the same visual language as the dashboard
