## Bash-Specific Tools

- `/usr/bin/time -f "%e %U"` — wall time (%e) and user CPU (%U) in seconds (fractional)
- `date +%s%N` — nanosecond-precision timestamp (10 digits seconds + 9 digits nanoseconds)
- `dd if=/dev/zero of=/tmp/...` or `fallocate -l {SIZE} /tmp/...` — memory allocation
- `sleep 1` — standard sleep with fractional seconds support (e.g., `sleep 0.5`)
- Manual JSON construction via printf/echo (jq is optional)

## Script Output Contract

All three Bash scripts follow identical I/O to Python variants:
- Line 1: JSON object
- Line 2+: Human summary
- Exit: 0 (pass), 1 (capability missing), 2 (enforcement not detected)

Create Bash noisy-neighbour monitor:
- Shebang: `#!/bin/bash`
- Set -e; set -u

- DRIFT_THRESHOLD_S env var (default "1.1"), validate with regex `^[0-9]+\.[0-9]+$|^[0-9]+$`
- Algorithm: 60 iterations of sleep(1) with per-iteration nanosecond timestamp tracking
  ```bash
  measurements=()
  for i in {1..60}; do
    start_ns=$(date +%s%N)
    sleep 1.0
    end_ns=$(date +%s%N)
    elapsed_s=$(echo "scale=3; ($end_ns - $start_ns) / 1000000000" | bc)
    measurements+=("$elapsed_s")
  done
  max_drift=$(printf '%s\n' "${measurements[@]}" | sort -rn | head -1)
  mean_drift=$(printf '%s\n' "${measurements[@]}" | awk '{sum+=$1} END {printf "%.3f", sum/NR}')
  pass=$(echo "$max_drift < $DRIFT_THRESHOLD_S" | bc)
  ```
- Output JSON with measurements array as stringified list
- Exit: 0 (pass), 2 (fail)
