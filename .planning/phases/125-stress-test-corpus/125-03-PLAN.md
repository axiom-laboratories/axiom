## PowerShell-Specific Tools

- `System.Diagnostics.Stopwatch::StartNew()` — high-resolution performance counter (~100ns precision)
- `Start-Sleep -Seconds 1` — standard sleep; supports fractional seconds (e.g., `-Seconds 0.5`)
- `[byte[]]::new($size)` — allocate uninitialized byte array in memory
- `Get-Date -Millisecond` or `[DateTime]::UtcNow.Ticks` for fine-grained timing
- `ConvertTo-Json` — JSON serialization (built-in, no jq dependency)
- Exit via `exit 0`, `exit 1`, `exit 2`

## Script Output Contract

All three PowerShell scripts follow identical I/O to Python and Bash variants:
- Line 1: JSON object (via ConvertTo-Json or Write-Output)
- Line 2+: Human summary via Write-Host or Write-Output
- Exit: 0 (pass), 1 (capability missing), 2 (enforcement not detected)

## Note on PowerShell Execution

PowerShell Core (pwsh) is the preferred runtime on modern Linux/macOS. Scripts may also run on Windows PowerShell 5.1. Key differences:
- `[Environment]::ProcessorCount` for CPU detection
- `Get-Process` returns process objects with CPU/memory properties
- `[GC]::GetTotalMemory()` for heap introspection

  ```powershell
  $measurements = @()
  for ($i = 0; $i -lt 60; $i++) {
    $sw = [System.Diagnostics.Stopwatch]::StartNew()
    Start-Sleep -Seconds 1
    $sw.Stop()
    $elapsed_s = $sw.Elapsed.TotalSeconds
    $measurements += $elapsed_s
  }
  $max_drift = ($measurements | Measure-Object -Maximum).Maximum
  $mean_drift = ($measurements | Measure-Object -Average).Average
  $pass = $max_drift -lt $DRIFT_THRESHOLD_S
  ```
