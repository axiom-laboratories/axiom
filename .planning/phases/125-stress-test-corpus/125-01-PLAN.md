## Existing Code Patterns

From tools/example-jobs/validation/:
- `cpu-spin.py`: 5-second CPU loop, wall/CPU time ratio < 0.8 threshold
- `memory-hog.py`: 256 MB bytearray with page-touching, 30-second hold, exit code 2 on non-kill
- Both check `AXIOM_CAPABILITIES` for `resource_limits_supported`
- Exit codes: 0 = pass/exit, 1 = capability missing, 2 = sentinel (enforcement not detected)

From mop_validation/scripts/:
- `run_signed_job.py`: Pattern for Ed25519 signing + API dispatch
- `test_local_stack.py`: Pattern for login, dispatch, poll /jobs/{id}, parse output
- All use `mop_validation/secrets.env` for connection details

## Script Interface

All three scripts in this plan follow the same contract:

**Input:**
- AXIOM_CAPABILITIES env var (space/comma-separated): must contain `resource_limits_supported` for CPU/memory tests (monitor has no gating)
- DRIFT_THRESHOLD_S env var (monitor only): max sleep drift tolerance, default "1.1"
- CPU_DURATION_S env var (cpu_burn only): burn duration, default "5"
- MEMORY_SIZE_MB env var (memory_hog only): allocation size, default "256"

**Output:**
- Line 1: JSON object with type, language, measurements, pass/fail
- Line 2+: Human-readable summary + details

**Exit codes:**
- 0 = pass (enforcement detected or working as expected)
- 1 = capability missing / preflight fail
- 2 = enforcement NOT detected (sentinel for orchestrator to treat as failure)

  <name>Task 3: Create Python Noisy Monitor (noisy_monitor.py)</name>
  <files>mop_validation/scripts/stress/python/noisy_monitor.py</files>
  <action>
Create Python noisy-neighbour monitor:
- Shebang: `#!/usr/bin/env python3`
- No capability gating (monitor can run without resource_limits_supported)
- DRIFT_THRESHOLD_S env var (default "1.1"), parse as float
- Measure: 60 iterations of sleep(1) with per-iteration latency tracking
- Algorithm:
  * measurements = []
  * for i in range(60):
    - start = time.perf_counter()
    - time.sleep(1.0)
    - elapsed = time.perf_counter() - start
    - measurements.append(elapsed)
  * max_drift = max(measurements)
  * mean_drift = sum(measurements) / len(measurements)
  * pass = max_drift < threshold
- Exit code: 0 = pass, 2 = fail (drift exceeded)

**JSON output** (first line):
```json
{
  "type": "noisy_monitor",

  "mean_drift_s": 1.001,
  "threshold_s": 1.1,
  "pass": true,
  "measurements": [1.00, 1.01, 1.00, ..., 1.05]
}
```
