In mirror_service.py, locate mirror_ingredient_and_dependencies() method (lines 200-236). Currently it unconditionally calls _mirror_pypi() on line 219. Replace with ecosystem-based dispatch logic:

```python
# Before (line 219):
await MirrorService._mirror_pypi(db, ingredient)

# After:
if ingredient.ecosystem == "NPM":
    await MirrorService._mirror_npm(db, ingredient)
elif ingredient.ecosystem == "NUGET":
    await MirrorService._mirror_nuget(db, ingredient)
elif ingredient.ecosystem == "APT":
    await MirrorService._mirror_apt(db, ingredient)
elif ingredient.ecosystem == "APK":
    await MirrorService._mirror_apk(db, ingredient)
else:  # Default to PYPI for backward compatibility
    await MirrorService._mirror_pypi(db, ingredient)
```

Also fix line 232 (for transitive mirrors) with the same dispatch logic. This ensures npm ingredients call _mirror_npm(), NuGet call _mirror_nuget(), etc.

Why this fix: npm/NuGet packages don't have Python wheels, so calling _mirror_pypi() fails. The correct mirror method must be called based on ingredient.ecosystem enum value.
