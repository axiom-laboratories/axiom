<objective>
Implement NuGet mirror backend with BaGetter sidecar and OCI pull-through caching for air-gapped NuGet/OCI support.

Purpose: Operators can approve NuGet packages and have them cached locally; Foundry builds transparently use pull-through OCI caches for base images from Docker Hub and GHCR, enabling complete STRICT air-gap support for .NET and container-based workloads.

Output:
- `_mirror_nuget()` service method using throwaway mcr.microsoft.com/dotnet/sdk:latest container with `nuget install`
- `get_nuget_config_content()` config template for Foundry nuget.config injection
- `get_oci_mirror_prefix()` helper to rewrite FROM lines for Docker Hub and GHCR caching
- BaGetter service in compose.ee.yaml on port 5555 with Mirror.Enabled configuration
- Two registry:2 instances (ports 5001 and 5002) for Docker Hub and GHCR pull-through caching
- Foundry integration: ecosystem-based config injection, FROM line rewriting, MIRRORED status validation
- Background task to auto-warm OCI cache from Approved OS list
- Extended health checks to include NUGET_MIRROR_URL and OCI cache endpoints
- Unit and integration tests
</objective>

   - Determines if base_image is Docker Hub or GHCR and returns appropriate cache prefix
   - Logic:
     - If base_image starts with "ghcr.io/": return "oci-cache-ghcr:5002"
     - If base_image contains "/" (registry-qualified): assume Docker Hub if not explicit registry, return "oci-cache:5001"
     - If base_image is unqualified (e.g., "node", "python"): Docker Hub convention, return "oci-cache:5001"
   - Example rewrites:
     - "node:18" → "oci-cache:5001/library/node:18"
     - "alpine:3.20" → "oci-cache:5001/library/alpine:3.20"
     - "ghcr.io/owner/image:latest" → "oci-cache-ghcr:5002/owner/image:latest"
   - Return rewritten image reference
