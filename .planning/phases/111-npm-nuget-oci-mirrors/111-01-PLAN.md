  <action>
Add `_mirror_npm()` async method to MirrorService class following the established pattern from _mirror_pypi(), _mirror_apt(), and _mirror_apk():

1. Signature: `async def _mirror_npm(self, db, ingredient: ApprovedIngredient) -> None`
2. Version constraint parsing: Accept {name}@{version} format (e.g., "lodash@4.17.21"), validate version is semver or npm dist-tags ("latest", "next", etc.)
3. Throwaway container: Run `docker run --rm node:latest npm pack {name}@{version} -C /mirror` in temporary directory
   - Use asyncio.to_thread(subprocess.run, ..., capture_output=True, text=True, timeout=120)
   - Capture stdout to extract tarball filename
4. Storage: Save downloaded tarball to `{MIRROR_DATA_PATH}/npm/{name}/{version}.tgz`
   - Create directories as needed (os.makedirs)
   - Validate tarball was saved before updating DB status
5. Status lifecycle: Set ingredient.mirror_status sequentially
   - PENDING → MIRRORING at method start
   - MIRRORING → MIRRORED on success, log path to ingredient.mirror_path
   - MIRRORING → FAILED on exception, log error message to ingredient.mirror_log
6. Error handling: Catch subprocess.CalledProcessError, TimeoutError, FileNotFoundError
   - Log descriptive error to mirror_log (e.g., "npm pack failed: package not found")
   - Do not re-raise (status update to FAILED is sufficient for fail-fast in Foundry)
7. Database commit: Use the same async session pattern as _mirror_pypi (await db.commit() after status update)

Reference existing _mirror_apt() for subprocess pattern and _mirror_pypi() for dependency/status handling.
