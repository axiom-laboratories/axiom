**Step 1: Add validation function**

Add this function near the top of main.py (after imports, before app definition, around line 40-60):

```python
import re

_YAML_UNSAFE = re.compile(r'[\n\r"\x00-\x1f:{}\[\]#&*!|>\'%@`]')

def _validate_compose_param(name: str, value: str) -> str:
    """Reject values containing YAML structural characters or control chars.
    
    Prevents YAML injection attacks via newlines in query parameters.
    """
    if value and _YAML_UNSAFE.search(value):
        raise HTTPException(
            status_code=422,
            detail=f"Query parameter '{name}' contains characters that are not allowed"
        )
    return value
```

If `re` is already imported, skip the import line. If `HTTPException` is already imported, skip that import.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Log output | Admin password must not appear in logs (could be captured in container logs, system logs, or monitoring systems) |
| HTTP query parameters | Untrusted user input crosses here; newlines/control chars can inject YAML syntax |
| YAML f-string interpolation | Parameters interpolated directly into YAML without escaping vulnerable to injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-171-06 | Information Disclosure | Startup logging | mitigate | Admin password removed from logger.warning call (line 281); plaintext secret never exposed in logs, stderr, or stdout. User retrieves password from secrets.env file on disk (more secure than parsing logs). |
| T-171-07 | Tampering / Denial of Service | Compose file generation | mitigate | Query parameters (tags, mounts, token, execution_mode) validated before YAML f-string interpolation. Any newline, control char, or YAML structural char rejected with HTTP 422. Prevents attacker from injecting arbitrary YAML nodes like `bar: injected` via newline in tags parameter. |
| T-171-08 | Elevation of Privilege | Compose file generation | mitigate | Validation rejects `:`, `{`, `}`, `[`, `]` which are YAML structural chars used to inject object definitions. Prevents attacker from injecting environment variables, volumes, or capabilities into generated compose file. |

</threat_model>
