<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP endpoint entry | Unauthenticated requests crossed here; authenticated requests carry JWT with embedded role |
| Permission check | User role checked against required permission; admin role has implicit all-permissions |
| Database query | Role permissions read from DB table; no caching (see Plan 04) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-171-01 | Elevation of Privilege | admin_router.py endpoints | mitigate | All signature/alert/token endpoints now require explicit permission (was: just require_auth); unauthorized viewers cannot generate tokens or access sensitive operations. Per D-04, admin role automatically bypasses all checks via `if current_user.role == "admin": return` in require_permission. |
| T-171-02 | Elevation of Privilege | jobs_router.py endpoints | mitigate | All job CRUD endpoints now require jobs:read or jobs:write (was: just require_auth); viewers can only read, operators can dispatch, admins unlimited. Job dispatch (POST /jobs) critical operation now explicitly gated on jobs:write. |
| T-171-03 | Elevation of Privilege | node enrollment token generation | mitigate | POST /admin/generate-token now requires nodes:write (was: require_auth); viewers cannot create enrollment tokens. This prevents unauthorized node joining. |
| T-171-04 | Information Disclosure | system alerts and signals | mitigate | GET /api/alerts now requires system:read (was: require_auth); non-privileged users cannot enumerate system state. Viewer role has system:read, operator has system:read+system:write. |
| T-171-05 | Elevation of Privilege | permission cache removal | mitigate | Plan 04 will remove per-process in-memory cache; all permission checks now hit DB on every request. Multi-worker deployments will have correct per-request access control (current single-worker can survive cache, but this is safer architecture). |

</threat_model>
