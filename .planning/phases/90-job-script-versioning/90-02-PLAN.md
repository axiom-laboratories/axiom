# Plan 90-02: Frontend — Script Viewer, Interleaved Timeline, and Job Detail Integration

## Goal

Deliver the operator-facing versioning UI:
1. A reusable `ScriptViewerModal` component with syntax highlighting, copy button, and compare-with-previous diff.
2. An updated `DefinitionHistoryPanel` in `JobDefinitions.tsx` that shows an interleaved timeline of execution rows and definition change rows, with version badges on executions.
3. A "View script (vN)" action in the `Jobs.tsx` job detail sheet.

## must_haves

- `ScriptViewerModal` opens from both entry points (job detail sheet and history tab) and shows the exact script for the selected version
- History tab shows version badges (e.g. `v3`) on execution rows that have `definition_version_id`
- History tab includes definition change rows (version change events) interleaved with execution rows, visually distinct (lighter, different icon, non-clickable)
- Jobs.tsx job detail sheet shows "View script (vN)" when `definition_version_id` is present; falls back to "View script" (from payload) for pre-phase-90 jobs
- "Compare with previous" button in `ScriptViewerModal` works when a previous version exists

## Context

Plan 90-01 must be complete first (provides the API endpoints and `definition_version_id` on Job responses). The existing `DefinitionHistoryPanel` queries `/api/executions?scheduled_job_id=...`; this plan adds a parallel query for versions. The dark Zinc UI theme is established; new components must match it. `react-diff-viewer-continued` is the chosen diff library.

<task id="90-02-03" title="Update DefinitionHistoryPanel to interleaved timeline with version badges">

In `puppeteer/dashboard/src/views/JobDefinitions.tsx`, refactor `DefinitionHistoryPanel`:

1. **Add a parallel versions query** alongside the existing executions query:
```typescript
const { data: versions } = useQuery({
    queryKey: ['definition-versions', definitionId],
    queryFn: async () => {
        const res = await authenticatedFetch(`/api/jobs/definitions/${definitionId}/versions`);
        return res.json() as Promise<any[]>;
    },
    enabled: !!definitionId,
});
```

2. **Build interleaved timeline**: merge execution rows and version rows into a single sorted array. Version rows get a `_rowType: 'version'` marker; execution rows get `_rowType: 'execution'`. Sort by timestamp descending (use `created_at` for version rows, `started_at` for executions).

```typescript
const timeline = React.useMemo(() => {
    const execRows = (executions ?? []).map(e => ({ ...e, _rowType: 'execution' }));
    const verRows = (versions ?? []).map(v => ({
        ...v,
        _rowType: 'version',
        _sortTs: v.created_at,
    }));
    return [...execRows, ...verRows].sort((a, b) => {
        const ta = new Date(a._sortTs ?? a.started_at ?? 0).getTime();
        const tb = new Date(b._sortTs ?? b.started_at ?? 0).getTime();
        return tb - ta;
    });
}, [executions, versions]);
```

3. **Add "Version" column header** to the table: add `<th className="px-4 py-2">Version</th>` between Status and Duration.

4. **Version badge on execution rows**: in each execution row `<tr>`, add a `<td>` that shows a version badge when `row.definition_version_id` is present:
```tsx
<td className="px-4 py-2">
    {row.definition_version_id ? (
        <button
            onClick={() => setViewingVersion({ jobDefId: definitionId, versionNumber: row.definition_version_number, runtime: row.runtime })}
            className="text-[10px] font-bold px-1.5 py-0.5 rounded border bg-blue-500/10 text-blue-400 border-blue-500/20 hover:bg-blue-500/20 transition-colors"
        >
            v{row.definition_version_number}
        </button>
    ) : <span className="text-zinc-600 text-xs">—</span>}
</td>
```

   Note: `definition_version_number` is available directly on the execution row from the API response (added in plan 90-01 task 06 — JobResponse includes `definition_version_number`).

5. **Version change rows**: render a distinct row for `_rowType === 'version'` entries:

```tsx
{row._rowType === 'version' && (
    <tr key={`ver-${row.id}`} className="border-t border-zinc-900/50">

            </span>

            {row.created_by || '—'}
        </td>
        <td colSpan={2} />
    </tr>

```
