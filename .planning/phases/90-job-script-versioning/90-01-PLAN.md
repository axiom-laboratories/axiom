# Plan 90-01: DB Model, Migration, and Backend Versioning Logic

## Goal

Add the `job_definition_versions` table, link it from `Job`, write a version snapshot on every `create_job_definition` and `update_job_definition` call, stamp the version ID at dispatch time, and expose two read-only API endpoints for listing and fetching versions.

## must_haves

- Every call to `create_job_definition` creates a `JobDefinitionVersion` row with `version_number=1` and `is_signed=True`
- Every call to `update_job_definition` creates a new `JobDefinitionVersion` row with an auto-incremented `version_number`; `is_signed` reflects whether the update includes a valid signature
- `dispatch_scheduled_job` stamps the latest signed version ID onto the new `Job` row as `definition_version_id`
- `GET /jobs/definitions/{id}/versions` returns all versions for a definition ordered by `version_number DESC`
- `GET /jobs/definitions/{id}/versions/{version_num}` returns a single version including full `script_content`
- `migration_v44.sql` creates the `job_definition_versions` table and adds `definition_version_id` to `jobs` table (IF NOT EXISTS guards for Postgres compatibility)

## Context

The codebase uses SQLAlchemy async with `Base.metadata.create_all` — new tables are auto-created on fresh deployments. Existing deployments require the migration SQL. The scheduler service is the single source of truth for all job definition mutations. The `dispatch_scheduled_job` method runs inside APScheduler and already has an `AsyncSession` via `AsyncSessionLocal()`.

```sql
-- Phase 90: Job Script Versioning
-- Creates job_definition_versions table and adds definition_version_id to jobs

CREATE TABLE IF NOT EXISTS job_definition_versions (
    id VARCHAR PRIMARY KEY,
    job_def_id VARCHAR NOT NULL REFERENCES scheduled_jobs(id) ON DELETE CASCADE,
    version_number INTEGER NOT NULL,
    script_content TEXT NOT NULL,
    signature_id VARCHAR REFERENCES signatures(id) ON DELETE SET NULL,
    signature_payload TEXT,
    cron_expression VARCHAR,
    target_tags TEXT,
    target_node_id VARCHAR,
    runtime VARCHAR(32),
    max_retries INTEGER DEFAULT 0,
    backoff_multiplier FLOAT DEFAULT 2.0,
    change_summary VARCHAR,
    is_signed BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW(),
    created_by VARCHAR,
    CONSTRAINT uq_jobdef_version UNIQUE (job_def_id, version_number)
);

ALTER TABLE jobs ADD COLUMN IF NOT EXISTS definition_version_id VARCHAR REFERENCES job_definition_versions(id) ON DELETE SET NULL;

CREATE INDEX IF NOT EXISTS ix_jobdefver_job_def_id ON job_definition_versions (job_def_id);
CREATE INDEX IF NOT EXISTS ix_jobs_definition_version_id ON jobs (definition_version_id);
```

<task id="90-01-03" title="Add _create_version_snapshot helper and call it in create/update">

In `puppeteer/agent_service/services/scheduler_service.py`:

1. Add import for `JobDefinitionVersion` at the top (alongside existing DB imports).

2. Add a private helper method `_create_version_snapshot` to `SchedulerService`:

```python
async def _create_version_snapshot(
    self,
    db_session: AsyncSession,
    job: ScheduledJob,
    created_by: str,
    is_signed: bool,
    change_summary: Optional[str] = None,
) -> "JobDefinitionVersion":
    """Write an immutable version snapshot for a job definition. Auto-increments version_number."""
    from ..db import JobDefinitionVersion
    import uuid as _uuid
    # Get current max version_number for this definition
    from sqlalchemy import func as sqlfunc
    result = await db_session.execute(
        select(sqlfunc.max(JobDefinitionVersion.version_number)).where(
            JobDefinitionVersion.job_def_id == job.id
        )
    )
    max_ver = result.scalar_one_or_none() or 0
    version = JobDefinitionVersion(
        id=_uuid.uuid4().hex,
        job_def_id=job.id,
        version_number=max_ver + 1,
        script_content=job.script_content,
        signature_id=job.signature_id,
        signature_payload=job.signature_payload,
        cron_expression=job.schedule_cron,
        target_tags=job.target_tags,
        target_node_id=job.target_node_id,
        runtime=job.runtime,
        max_retries=job.max_retries,
        backoff_multiplier=job.backoff_multiplier,
        change_summary=change_summary,
        is_signed=is_signed,
        created_at=datetime.utcnow(),
        created_by=created_by,
    )
    db_session.add(version)
    return version
```

3. In `create_job_definition()`: after `await db_session.commit()` and `await db_session.refresh(new_def)`, add:
```python
await self._create_version_snapshot(db_session, new_def, current_user.username, is_signed=True, change_summary="Initial version")
await db_session.commit()
```

4. In `update_job_definition()`: determine `is_signed` and `change_summary` based on which case ran, then call `_create_version_snapshot` before `job.updated_at = datetime.utcnow()`. The snapshot should be created after all field mutations but before the final `commit()`. Build change summary by comparing fields:
   - Script changed → include "script updated"
   - Cron changed → include "schedule updated"
   - Tags changed → include "tags updated"
   - Re-sign only → "re-signed"
   - `is_signed = True` if the job's final status after update is "ACTIVE", else `False`

5. The snapshot call must happen inside the existing function before the final `await db_session.commit()` call (line 572), so it commits atomically with the job update.

```python
async def list_job_definition_versions(self, job_id: str, db_session: AsyncSession) -> list:
    """Returns all versions for a job definition, ordered newest first."""
    from fastapi import HTTPException
    from ..db import JobDefinitionVersion
    # Verify definition exists
    result = await db_session.execute(select(ScheduledJob).where(ScheduledJob.id == job_id))
    if not result.scalar_one_or_none():
        raise HTTPException(status_code=404, detail="Job definition not found")
    ver_result = await db_session.execute(
        select(JobDefinitionVersion)
        .where(JobDefinitionVersion.job_def_id == job_id)
        .order_by(JobDefinitionVersion.version_number.desc())
    )
    return ver_result.scalars().all()

async def get_job_definition_version(self, job_id: str, version_num: int, db_session: AsyncSession):
    """Returns a specific version of a job definition."""
    from fastapi import HTTPException
    from ..db import JobDefinitionVersion
    ver_result = await db_session.execute(
        select(JobDefinitionVersion)
        .where(JobDefinitionVersion.job_def_id == job_id, JobDefinitionVersion.version_number == version_num)
    )
    ver = ver_result.scalar_one_or_none()
    if not ver:
        raise HTTPException(status_code=404, detail="Version not found")
    return ver
```
