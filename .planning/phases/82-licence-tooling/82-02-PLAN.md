```toml
# Gitleaks configuration
# https://github.com/gitleaks/gitleaks

# Allowlists — values that are known non-secrets used in CI
# Uses [[allowlists]] plural array syntax (required since gitleaks v8.25.0)
[[allowlists]]
description = "Known CI dummy values — not real secrets"
regexTarget = "match"
regexes = [
  '''ci-dummy-key''',
  '''AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA=''',
]
```

Edit `.github/workflows/ci.yml` — add a `secret-scan` job at the end of the file as a new top-level peer job (not a step inside an existing job). The job must use `fetch-depth: 0` so gitleaks can scan full git history:

```yaml
  secret-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: gitleaks/gitleaks-action@v2
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

This job has no `needs:` dependency — it runs in parallel with the other CI jobs.

Verify `.gitleaks.toml` syntax by running gitleaks locally if available, or by checking the file structure manually. The `[[allowlists]]` double-bracket is mandatory — a single `[allowlist]` is silently ignored in gitleaks v8.25.0+.
