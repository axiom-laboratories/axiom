# Plan 89-01: CE Webhook Notification Backend

## Goal

Replace the CE no-op `WebhookService.dispatch_event` stub with real outbound HTTP POST delivery. Add three Admin API endpoints for alerting config management. Add a test endpoint for synthetic delivery. Gate all config endpoints on `nodes:write` permission so operators can access them without a licence.

## Context

- `webhook_service.py` is currently a 15-line no-op stub — the entire `dispatch_event` body is `pass`
- `job_service.py:1218–1225` already calls `dispatch_event` for all terminal statuses — needs filtering and payload enrichment
- `httpx` is already in `requirements.txt`
- Config table exists (key/value) — same pattern used for retention, signing key, network mounts
- Phase 87 locked the payload shape (see CONTEXT.md `code_context`)

## must_haves

- `dispatch_event` performs real HTTP POST when `alerts.webhook_enabled == 'true'` and URL is set
- Delivery fires only on FAILED and DEAD_LETTER (+ SECURITY_REJECTED when `alerts.webhook_security_rejections == 'true'`)
- Payload matches the Phase 87 locked shape: `event`, `job_guid`, `job_name`, `node_id`, `error_summary`, `failed_at`
- `alerts.last_delivery_status` Config key updated after every dispatch (real and test)
- `GET /api/admin/alerts/config` returns all four config keys
- `PATCH /api/admin/alerts/config` validates URL format (http/https) and saves
- `POST /api/admin/alerts/test` fires synthetic payload, returns `{success, status_code, body_snippet}`
- All three endpoints use `require_permission("nodes:write")` — accessible to operators
- Delivery failures are logged, job status is never affected

Replace the `pass` body in `WebhookService.dispatch_event` with:

1. Read config keys from DB: `alerts.webhook_url`, `alerts.webhook_enabled`, `alerts.webhook_security_rejections`
2. If `webhook_enabled != 'true'` or URL is empty: return immediately (no-op path)
3. Check event type filter: only deliver for `job:failed`, `job:dead_letter`; deliver for `job:security_rejected` only if `webhook_security_rejections == 'true'`
4. Build the locked payload shape:
   ```python
   outbound = {
       "event": "job.failed" if event_type != "job:security_rejected" else "job.security_rejected",
       "job_guid": payload.get("guid", ""),
       "job_name": payload.get("job_name", ""),
       "node_id": payload.get("node_id", ""),
       "error_summary": payload.get("error_summary", ""),
       "failed_at": payload.get("failed_at", datetime.utcnow().isoformat() + "Z"),
   }
   ```
5. POST to URL with 5-second timeout using `httpx.AsyncClient`:
   ```python
   async with httpx.AsyncClient(timeout=5.0) as client:
       response = await client.post(url, json=outbound)
   ```
6. After dispatch (success or failure): write `alerts.last_delivery_status` as JSON:
   ```json
   {"status_code": 200, "timestamp": "2026-03-29T22:15:00Z", "body_snippet": "...first 200 chars..."}
   ```
   On connection error: `{"status_code": null, "timestamp": "...", "body_snippet": "Connection refused"}`
7. Log delivery result at INFO level; never raise — delivery failure is silent to caller

Add required imports: `import httpx`, `import json`, `from datetime import datetime`

@app.patch("/api/admin/alerts/config", tags=["Admin"])
async def update_alerts_config(
    body: AlertsConfigUpdate,
    current_user: User = Depends(require_permission("nodes:write")),
    db: AsyncSession = Depends(get_db),
):
    """Update alerting/webhook configuration. Only provided fields are updated."""
    updates = {}
    if body.webhook_url is not None:
        updates["alerts.webhook_url"] = body.webhook_url
    if body.webhook_enabled is not None:
        updates["alerts.webhook_enabled"] = "true" if body.webhook_enabled else "false"
    if body.webhook_security_rejections is not None:
        updates["alerts.webhook_security_rejections"] = "true" if body.webhook_security_rejections else "false"

    for key, value in updates.items():
        existing = await db.execute(select(Config).where(Config.key == key))
        row = existing.scalar_one_or_none()
        if row:
            row.value = value
        else:
            db.add(Config(key=key, value=value))

    await db.commit()
    return {"updated": list(updates.keys())}
```

@app.post("/api/admin/alerts/test", tags=["Admin"])
async def test_alerts_delivery(
    current_user: User = Depends(require_permission("nodes:write")),
    db: AsyncSession = Depends(get_db),
):
    """Fire a synthetic test notification to the configured webhook URL."""
    url_row = await db.execute(select(Config).where(Config.key == "alerts.webhook_url"))
    url_val = url_row.scalar_one_or_none()
    webhook_url = url_val.value if url_val else ""

    if not webhook_url or not (webhook_url.startswith("http://") or webhook_url.startswith("https://")):
        raise HTTPException(status_code=400, detail="No valid webhook URL configured")

    synthetic_payload = {
        "event": "job.failed",
        "job_guid": "00000000-0000-0000-0000-000000000000",
        "job_name": "test-alert",
        "node_id": "system",
        "error_summary": "This is a test notification",
        "failed_at": datetime.utcnow().isoformat() + "Z",
    }

    success = False
    status_code = None
    body_snippet = None
    error_msg = None

    try:
        async with httpx.AsyncClient(timeout=5.0) as client:
            resp = await client.post(webhook_url, json=synthetic_payload)
            status_code = resp.status_code
            body_snippet = resp.text[:200]
            success = 200 <= status_code < 300
    except Exception as e:
        error_msg = str(e)[:200]

    # Update last_delivery_status
    status_json = json.dumps({
        "status_code": status_code,
        "timestamp": datetime.utcnow().isoformat() + "Z",
        "body_snippet": body_snippet or error_msg or "",
    })
    existing = await db.execute(select(Config).where(Config.key == "alerts.last_delivery_status"))
    row = existing.scalar_one_or_none()
    if row:
        row.value = status_json
    else:
        db.add(Config(key="alerts.last_delivery_status", value=status_json))
    await db.commit()

    return {"success": success, "status_code": status_code, "body_snippet": body_snippet, "error": error_msg}
```
