   - `ScheduleEntryResponse`: id (str) | type (Literal["JOB", "FLOW"]) | name (str) | next_run_time (Optional[datetime]) | last_run_status (Optional[str])
   - `ScheduleListResponse`: entries (List[ScheduleEntryResponse]) | total (int)

2. **scheduler_service.py**: Add async method `get_unified_schedule(db: AsyncSession) -> ScheduleListResponse` to SchedulerService class:
   - Query all ScheduledJob records where is_active=true AND schedule_cron IS NOT NULL
   - Query all Workflow records where is_paused=false AND schedule_cron IS NOT NULL
   - For each job/workflow:
     - Parse schedule_cron as space-separated 5-part string (minute hour day month day_of_week)
     - Create CronTrigger with explicit timezone=timezone.utc
     - Call trigger.get_next_fire_time(None, now) where now = datetime.now(timezone.utc)
     - Wrap CronTrigger construction in try/except; skip entry if invalid, log warning
     - Query last-run status: for jobs, select Job.status where scheduled_job_id=job.id order by created_at desc limit 1; for workflows, select WorkflowRun.status where workflow_id=workflow.id order by created_at desc limit 1
   - Build unified list of dicts: {id, type ("JOB" or "FLOW"), name, next_run_time, last_run_status}
   - Sort by next_run_time ascending (handle None values by treating as datetime.max)
   - Return ScheduleListResponse(entries=[ScheduleEntryResponse(**e) for e in entries], total=len(entries))

Pitfalls to avoid (from RESEARCH.md):
- Do NOT compute next-run times on frontend; they must be server-side
- Handle invalid cron gracefully — skip entry, don't crash endpoint
- Use timezone.utc explicitly when creating CronTrigger
- Use Optional[datetime] for next_run_time (may be None for edge cases)
- Check last_run_status is Optional[str]; frontend will show "Never" if None

@app.get("/api/schedule", response_model=ScheduleListResponse)
async def get_schedule(
    db: AsyncSession = Depends(get_db),
    _: User = Depends(require_permission("jobs:read"))

    Returns sorted by next_run_time ascending (soonest first).
    Only includes active items with cron schedules.

    service = SchedulerService(db)
