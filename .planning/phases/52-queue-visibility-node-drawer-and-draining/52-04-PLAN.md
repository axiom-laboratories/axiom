  <action>
Create `puppeteer/dashboard/src/views/Queue.tsx`. This is a read-only monitoring view — no dispatch form.

**Architecture: Two parallel React Query fetches**
- `activeJobs`: `GET /jobs` with no status filter but explicit status list `PENDING,ASSIGNED,RUNNING` — always shown in full
- `terminalJobs`: `GET /jobs` with `status=COMPLETED,FAILED,CANCELLED,DEAD_LETTER,SECURITY_REJECTED` AND `date_from=<ISO timestamp>` computed from the recency window dropdown (1h/6h/24h)

Use `date-fns` `subHours` for the date_from timestamp (already installed — used in Jobs.tsx).

**State:**
```typescript
const [recencyWindow, setRecencyWindow] = useState<1 | 6 | 24>(1); // hours
```

**React Query keys:** `['queue', 'active']` and `['queue', 'terminal', recencyWindow]` so invalidation targets both. Use `queryClient.invalidateQueries({ queryKey: ['queue'] })` to refresh both on WebSocket events.

**WebSocket integration:** Call `useWebSocket` hook. On `job:created`, `job:updated`, `node:updated` events, invalidate `['queue']`. Do NOT use `setInterval` or `refetchInterval` — WebSocket is the only refresh mechanism.

**Layout:**
- Page header: "Queue" title + recency dropdown (Select with 1h / 6h / 24h options)
- Two sections in the same table or clearly separated:
  - "Active" section: PENDING + ASSIGNED + RUNNING jobs (from activeJobs query)
  - "Recent" section (with recency label like "Last 1 hour"): terminal jobs from terminalJobs query
- Table columns: Status badge, Job name/GUID (truncated), Task type + runtime display_type, Node (if ASSIGNED/RUNNING), Created at (relative time)
- DRAINING badge: if a PENDING job's target_tags includes a node that is DRAINING, show an amber "Node Draining" badge — note this is a best-effort display (the diagnosis endpoint provides authoritative info)
- Empty state: "No active jobs" / "No completed jobs in this window"
- Loading skeleton while fetching

**Important UX note:** This is read-only — no action buttons (no cancel, no resubmit). Users who need to act should go to Jobs view. A subtle "Manage jobs →" link at the top-right pointing to `/jobs` is appropriate.

**Status badge colors:** Reuse the same StatusIcon/status color mapping pattern from Jobs.tsx (or recreate minimally inline). DRAINING nodes get amber color (same as DRAFT jobs in Phase 48).

**Imports needed:**
- `authenticatedFetch` from `../auth`
- `useWebSocket` from `../hooks/useWebSocket`
- `useQuery, useQueryClient` from `@tanstack/react-query`
- `subHours, formatDistanceToNow` from `date-fns`
- Lucide icons: `ListOrdered` (or similar) for the page, `Activity` for status
- shadcn/ui: `Select`, `Badge`, `Card`, `CardContent`, `CardHeader`, `CardTitle`
