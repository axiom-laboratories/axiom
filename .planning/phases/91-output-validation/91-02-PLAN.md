# Plan 91-02: Frontend — Validation Rules Form and Failure Display

## Goal

Surface validation rules in the job definition form and display validation failure reasons distinctly from runtime errors in both the execution history panel and the job detail sheet.

## must_haves

- `JobDefinitionModal` has a collapsible "Validation Rules" section with exit code, regex, and JSON path+expected inputs
- Section collapsed by default when no rules set; auto-expanded when definition has rules
- Exit code field defaults to `0`; clearing it disables exit code checking
- Validation rules are serialized into `validation_rules` dict on form submit and sent to the API
- On edit, validation rules pre-populate from the definition's `validation_rules`
- `DefinitionHistoryPanel` shows `"Validation failed: {rule}"` in orange for FAILED rows with `failure_reason`
- `Jobs.tsx` job detail sheet shows the same distinct validation failure label
- Frontend tests cover the new form section and the failure_reason display

<task id="91-02-03" title="Serialize validation_rules on form submit in JobDefinitions.tsx">
In `puppeteer/dashboard/src/views/JobDefinitions.tsx`, find the `handleSubmit` (or equivalent submit handler) that builds the request body for `POST /jobs/definitions` and `PATCH /jobs/definitions/{id}`.

Add `validation_rules` serialization before the API call:

```typescript
// Phase 91: build validation_rules from flat form fields
const buildValidationRules = (fd: typeof formData) => {
    const rules: Record<string, unknown> = {};
    if (fd.validation_exit_code !== '') {
        const n = parseInt(fd.validation_exit_code, 10);
        if (!isNaN(n)) rules.exit_code = n;
    }
    if (fd.validation_stdout_regex) rules.stdout_regex = fd.validation_stdout_regex;
    if (fd.validation_json_path && fd.validation_json_expected !== '') {
        rules.json_path = fd.validation_json_path;
        rules.json_expected = fd.validation_json_expected;
    }
    return Object.keys(rules).length > 0 ? rules : null;
};
```

Include in the request body:
```typescript
validation_rules: buildValidationRules(formData),
```

Also update the `resetForm` function to include:
```typescript
validation_exit_code: '0',
validation_stdout_regex: '',
validation_json_path: '',
validation_json_expected: '',
```
