  <name>Task 4: Add GET /api/workflows/{id}/runs endpoint for run history list</name>
  <files>puppeteer/agent_service/main.py</files>
  <action>
    Add a new FastAPI route to puppeteer/agent_service/main.py:

    **Route signature:**
    ```python
    @app.get(
        "/api/workflows/{workflow_id}/runs",
        response_model=WorkflowRunListResponse,
        dependencies=[Depends(require_permission("workflows:read"))]
    )
    async def get_workflow_runs(
        workflow_id: str,
        skip: int = Query(0, ge=0),
        limit: int = Query(100, ge=1, le=1000),
        db: AsyncSession = Depends(get_db),
    ) -> WorkflowRunListResponse:
        """Get paginated list of runs for a workflow."""
        # Query workflow to ensure it exists
        workflow = await db.execute(
            select(Workflow).where(Workflow.id == workflow_id)
        )
        workflow_obj = workflow.scalar_one_or_none()
        if not workflow_obj:
            raise HTTPException(status_code=404, detail="Workflow not found")

        # Count total runs
        count_stmt = select(func.count(WorkflowRun.id)).where(
            WorkflowRun.workflow_id == workflow_id
        )
        count_result = await db.execute(count_stmt)
        total = count_result.scalar()

        # Fetch paginated runs (order by started_at DESC, most recent first)
        stmt = (
            select(WorkflowRun)
            .where(WorkflowRun.workflow_id == workflow_id)
            .order_by(WorkflowRun.started_at.desc())
            .offset(skip)
            .limit(limit)
        )
        runs_result = await db.execute(stmt)
        runs = runs_result.scalars().all()

        # Convert to response models
        run_responses = [
            WorkflowRunResponse.model_validate(run, from_attributes=True)
            for run in runs
        ]

        return WorkflowRunListResponse(
            runs=run_responses,
            total=total,
            skip=skip,
            limit=limit,
        )
    ```

    **Placement:** Add after the existing workflow CRUD routes (likely around line 2500–2700 in main.py where workflow routes are grouped). Follow the same pattern as GET /api/jobs and GET /api/job-definitions.

    **Query optimization:**
    - Use SQLAlchemy async query with proper joins if WorkflowRun includes related data
    - Order by started_at DESC to show most recent runs first
    - Use skip/limit for pagination (standard REST pattern)
    - Include proper error handling (404 if workflow not found)
    - Require "workflows:read" permission
