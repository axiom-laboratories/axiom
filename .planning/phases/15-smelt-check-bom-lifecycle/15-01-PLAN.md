# Plan 15-01: Wave 1 — Schema & Lifecycle Foundation

**Objective**: Implement the database schema changes and model updates required for image lifecycle governance and BOM tracking.

## Context
Phase 15 adds governance states (DEPRECATED, REVOKED) to images and requires storage for Bill of Materials (BOM) data and a searchable package index.

## Tasks
1. **Create `puppeteer/migration_v30.sql`**:
    - Update `puppet_templates` table: Add `status` (String, default 'DRAFT') and `bom_captured` (Boolean, default False).
    - Create `image_boms` table (id, template_id, raw_data_json, created_at).
    - Create `package_index` table (id, template_id, name, version, type ['pip', 'apt']).
    - Add index on `package_index(name, version)`.
2. **Update `puppeteer/agent_service/db.py`**:
    - Update `PuppetTemplate` model with `status` and `bom_captured`.
    - Implement `ImageBOM` and `PackageIndex` SQLAlchemy models.
3. **Update `puppeteer/agent_service/models.py`**:
    - Add `ImageBOMResponse` and `PackageIndexResponse` Pydantic models.
    - Update `PuppetTemplateResponse` to include `status` and `bom_captured`.
4. **Verification**:
    - Run `PYTHONPATH=. .venv/bin/python -c "from puppeteer.agent_service.db import ImageBOM, PackageIndex; print('✅ Models valid')"`
    - Verify SQL migration matches the models.

## Success Criteria
- Database schema supports image lifecycle states.
- Storage for raw and indexed BOM data is established.
- SQL migration script is ready for application.
