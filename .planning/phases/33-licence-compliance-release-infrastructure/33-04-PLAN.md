  <name>Task 2: Configure PyPI pending publishers on test.pypi.org and pypi.org</name>
  <action>
A "pending publisher" allows the PyPI project to be created automatically on first publish via OIDC — no API token needed. All four fields must match release.yml exactly.

**Field values to use on BOTH pypi registries:**
- PyPI project name: `axiom-agent-sdk` (check release.yml URL — it says axiom-agent-sdk, NOT axiom-sdk)
- GitHub owner: `axiom-laboratories`
- Repository name: `axiom`
- Workflow filename: `release.yml`

**Step 1 — Configure pending publisher on test.pypi.org:**
1. Log in to test.pypi.org (create account if needed — separate from pypi.org)
2. Go to Account settings (top-right menu) > Publishing
3. Click "Add a new pending publisher"
4. Fill in the fields above, set Environment name: `testpypi`
5. Save

**Step 2 — Configure pending publisher on pypi.org (production):**
1. Log in to pypi.org
2. Go to Account settings > Publishing
3. Click "Add a new pending publisher"
4. Fill in the fields above, set Environment name: `pypi`
5. Save

**Critical:** If any field does not match exactly, the OIDC exchange fails with a 403 and a misleading error. Double-check `axiom-agent-sdk` (not `axiom-sdk`) and environment names match `testpypi` / `pypi` character-for-character.
