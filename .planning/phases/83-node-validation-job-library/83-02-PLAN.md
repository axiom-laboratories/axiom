Network filtering: Docker-native --network=none isolation only. The script validates existing isolation — no iptables manipulation.

Capability guard pattern (memory-hog.py, cpu-spin.py):
```python
import os, sys
caps_raw = os.environ.get("AXIOM_CAPABILITIES", "")
if "resource_limits_supported" not in caps_raw:
    print("FAIL: resource limits are not supported on this node "
          "(resource_limits_supported capability missing)", flush=True)
    sys.exit(1)
```

Memory page-touching (forces RSS commitment, avoids Linux overcommit):
```python
chunk = bytearray(256 * 1024 * 1024)
chunk[0::4096] = b'\x00' * (len(chunk) // 4096)
```

CPU-spin measurable throttle pattern (from RESEARCH.md):
```python
import time
DURATION = 5
start_wall = time.monotonic()
start_cpu  = time.process_time()
deadline = start_wall + DURATION
while time.monotonic() < deadline:
    _ = 2 ** 31
wall = time.monotonic() - start_wall
cpu  = time.process_time() - start_cpu
ratio = cpu / wall if wall > 0 else 0
```

manifest.yaml capability_requirements format:
```yaml
required_capabilities:
  resource_limits_supported: "1.0"   # string, not boolean
```

JobCreate.capability_requirements: Optional[Dict[str, str]] — values MUST be strings (for semver comparison in job_service.py)

    - volume-mapping.sh: exits 0 when AXIOM_VOLUME_PATH dir exists and sentinel write/read succeeds; exits 1 when dir missing; cleans up sentinel on success
    - network-filter.py: exits 0 when blocked host (AXIOM_BLOCKED_HOST) times out (isolation confirmed); exits 1 when blocked host is reachable (isolation NOT active)
    - memory-hog.py: exits 1 with capability guard message when AXIOM_CAPABILITIES does not contain "resource_limits_supported"; touches all pages of 256MB bytearray; sleeps 30s; has sys.exit(2) sentinel after sleep
    - cpu-spin.py: exits 1 with capability guard message when AXIOM_CAPABILITIES does not contain "resource_limits_supported"; spins for 5s; reports wall/cpu/ratio; prints PASS if ratio < 0.8 (throttled), INFO if not
  </behavior>
  <action>
    **validation/volume-mapping.sh:**
    Shebang `#!/usr/bin/env bash`, `set -euo pipefail`.
    `MOUNT_PATH="${AXIOM_VOLUME_PATH:-/mnt/axiom-data}"`.
    `SENTINEL="${MOUNT_PATH}/axiom-validation-$$.txt"` (PID-unique filename).
    Check `[[ -d "${MOUNT_PATH}" ]]` — if not, echo "FAIL: mount path ${MOUNT_PATH} does not exist inside container" and exit 1.
    Write `"axiom-validation-$(date -u +%Y-%m-%dT%H:%M:%SZ)"` to sentinel.
    Read back with `cat "${SENTINEL}"`.
    Remove sentinel with `rm -f "${SENTINEL}"`.
    Echo "=== PASS: volume mount is readable and writable ===".
    Make executable.

    **validation/network-filter.py:**
    Shebang `#!/usr/bin/env python3`.
    Import `urllib.request, socket, sys, os`.
    `ALLOWED = os.environ.get("AXIOM_ALLOWED_HOST", "")`.
    `BLOCKED = os.environ.get("AXIOM_BLOCKED_HOST", "8.8.8.8")`.
    `socket.setdefaulttimeout(5)`.
    Try `socket.create_connection((BLOCKED, 80), timeout=5)` — if succeeds: print "FAIL: connected to blocked host ... network isolation is NOT active" and sys.exit(1).
    Except (socket.timeout, OSError): print "PASS: blocked host {BLOCKED} is unreachable (expected)".
    If ALLOWED: try create_connection — success prints PASS, failure prints WARN.
    Final print "=== network-filter validation complete ===".
    Do NOT manipulate iptables — the script only validates existing isolation.

    **validation/memory-hog.py:**
    Shebang `#!/usr/bin/env python3`.
    Import `os, sys, time`.
    Capability guard: check `os.environ.get("AXIOM_CAPABILITIES", "")` — if "resource_limits_supported" not in value: print FAIL message and sys.exit(1).
    Print "resource_limits_supported: present — proceeding with memory allocation test".
    Print "Allocating 256 MB ...", flush=True.
    `chunk = bytearray(256 * 1024 * 1024)`.
    Touch all pages: `chunk[0::4096] = b'\x00' * (len(chunk) // 4096)`.
    Print "Allocation complete — container should be OOM-killed or exceed memory limit".
    `time.sleep(30)`.
    Print "ERROR: should have been killed before reaching this line" and sys.exit(2).
    Note: sys.exit(2) after the sleep is an error sentinel — the job should be OOM-killed before reaching it.

    **validation/cpu-spin.py:**
    Shebang `#!/usr/bin/env python3`.
    Import `os, sys, time`.
    Capability guard: same pattern as memory-hog.py — exit 1 with FAIL message if capability missing.
    Print "resource_limits_supported: present — proceeding with CPU spin test".
    `DURATION = 5`.
    Record `start_wall = time.monotonic()`, `start_cpu = time.process_time()`.
    Spin loop for DURATION seconds: `while time.monotonic() < start_wall + DURATION: _ = 2 ** 31`.
    Calculate wall, cpu, ratio.
    Print `f"Wall time: {wall:.2f}s  CPU time: {cpu:.2f}s  Ratio: {ratio:.2f}"`.
    If ratio < 0.8: print "PASS: CPU throttling confirmed ..." else print "INFO: No throttling detected ...".
    Print "=== cpu-spin validation complete ===".
    Exit 0 (this job completes successfully and reports measurements; it is NOT killed).
