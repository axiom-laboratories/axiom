<task type="auto">
  <name>Task 1: Update Containerfile.server with appuser, chown, and USER directive</name>
  <files>puppeteer/Containerfile.server</files>
  <action>
Read the current Containerfile.server. At the end of the file (after all RUN package install commands but before or replacing any existing USER/CMD directives), add the following three lines in order:

```dockerfile
RUN adduser appuser
RUN chown -R appuser:appuser /app
USER appuser
```

Rationale for placement:
- adduser must run as root (before USER switch) — place at end after all package installs
- chown must run as root (before USER switch) — place immediately after adduser
- USER directive switches execution context for any subsequent RUN/CMD — place last, so all file operations are complete

Verify the Containerfile syntax by checking:
- "adduser appuser" exists
- "chown -R appuser:appuser /app" exists immediately after
- "USER appuser" exists at the end
- No other USER directives exist earlier in the file

If the Containerfile already has a USER directive, replace it. If there are multiple RUN commands after chown, those must execute as root (unlikely given current structure).

Current file context: Alpine-based python:3.12-alpine, installs packages, copies agent_service and model_service code, runs Python app.


Rationale for placement:
- useradd must run as root (before USER switch) — place at end after all package installs
- The `-m` flag creates a home directory (/home/appuser), which is harmless though unused in containers
- chown must run as root (before USER switch) — place immediately after useradd
- USER directive switches execution context for any subsequent RUN/CMD — place last, so all file operations are complete

Verify the Containerfile syntax by checking:

- "USER appuser" exists at the end

If the Containerfile already has a USER directive, replace it.
