- Import pytest, subprocess, json
- Fixture: get container IDs from `docker ps --filter label=... -q` or by service name (puppeteer-agent-1, puppeteer-model-1, node)
- Each test calls docker exec with appropriate command, parses output, asserts result
- Use subprocess.run(..., capture_output=True, text=True, check=True) for reliability

Example test structure:
```python
def test_agent_process_uid(agent_container_id):
    result = subprocess.run(
        ['docker', 'exec', agent_container_id, 'ps', '-o', 'uid=,comm='],
        capture_output=True, text=True, check=True
    )
    # Find the python process uid
    for line in result.stdout.split('\n'):
        if 'python' in line:
            uid = line.split()[0]
            assert uid == '1000', f"Expected uid 1000, got {uid}"
            break
```

Reference VALIDATION.md for test mapping and expected assertions.

  <action>
Create a new shell script `mop_validation/scripts/verify_nonroot.sh` with the following checks:

1. Check agent container process UID: `docker exec puppeteer-agent-1 ps -o uid=,comm= | grep python | awk '{print $1}'` should equal "1000"
2. Check model container process UID: `docker exec puppeteer-model-1 ps -o uid=,comm= | awk '{print $1}'` should equal "1000"
3. Check node container process UID: `docker exec node ps -o uid=,comm= | grep python | awk '{print $1}'` should equal "1000"
4. Check agent /app ownership: `docker exec puppeteer-agent-1 stat -c %U:%G /app` should equal "appuser:appuser"
5. Check model /app ownership: `docker exec puppeteer-model-1 stat -c %U:%G /app` should equal "appuser:appuser"
6. Check node /app ownership: `docker exec node stat -c %U:%G /app` should equal "appuser:appuser"
7. Check secrets volume ownership: `docker exec puppeteer-agent-1 stat -c %U:%G /app/secrets` should equal "appuser:appuser"
8. Check volume write access: `docker exec puppeteer-agent-1 touch /app/secrets/test_write.txt && rm /app/secrets/test_write.txt` should succeed

Script structure:
- #!/bin/bash at top
- Set -e (exit on error) for strict mode
- Define check_result() helper: takes expected and actual, prints pass/fail
- For each check, run docker exec command, capture output, compare, print result
- Print summary at end: "X/8 checks passed"
- Exit code 0 if all pass, 1 if any fail

Example structure:
```bash
#!/bin/bash
set -e

check_result() {
  local name=$1
  local expected=$2
  local actual=$3
  if [ "$expected" == "$actual" ]; then
    echo "✓ $name: PASS"
  else
    echo "✗ $name: FAIL (expected: $expected, got: $actual)"
    exit 1
  fi
}

echo "=== Checking Non-Root User Configuration ==="

# Check 1: Agent process UID
agent_uid=$(docker exec puppeteer-agent-1 ps -o uid=,comm= | grep python | awk '{print $1}')
check_result "Agent process UID" "1000" "$agent_uid"

# ... continue for other checks
```

Make script executable: chmod +x mop_validation/scripts/verify_nonroot.sh

Rationale: Standalone script for quick manual verification without pytest dependency; useful for ops/validation teams.
