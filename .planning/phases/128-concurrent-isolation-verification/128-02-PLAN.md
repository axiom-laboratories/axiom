## Reference Patterns

From Phase 126 validation (concurrent_isolation scenario):
- Memory hog: 512m limit, allocates >512m to trigger OOM kill (exit code 137)
- CPU burner: 1.0 cpu_limit, runs ~10 seconds
- Monitor: no limits, runs ~60 seconds, measures sleep drift
- All 3 dispatched concurrently, polled for completion

From existing stress test infrastructure:
- dispatch_job() returns job_id string
- poll_job(job_id, timeout_s) returns job dict with status, exit_code, result.stdout
- Monitor output: first stdout line is JSON with max_drift_s, mean_drift_s, pass fields

## New Behavior (5-Run Sequential with Same-Node Targeting)

1. **Node Selection**: Get all nodes via GET /nodes, filter to execution_mode='docker' AND status='ONLINE', pick first
2. **5-Run Loop**:
   - For each run (1-5):
     - Dispatch 3 jobs to target_node_id
     - Poll all 3 until completion (with timeout)
     - Verify assigned_node == target_node for all 3 (abort if mismatch)
     - Extract max_drift_s, pass from monitor JSON output
     - Extract exit_code from hog job (should be 137 for OOMKill)
     - Record run result: {run, pass, max_drift_s, mean_drift_s, hog_exit_code}
     - Wait 5 seconds for cleanup before next run
3. **Evaluation**: Count passes (pass_count = sum(1 for r in results if r['pass']))
4. **Overall Verdict**: overall_pass = pass_count >= 4 (4 out of 5 threshold)
5. **Report Generation**:
   - Markdown: summary table (run, pass/fail, max_drift, mean_drift, hog_exit), overall verdict, findings, environment metadata
   - JSON: full results including all 5 runs' 60-measurement arrays, job metadata, environment info
6. **Requirements Update**: If overall_pass, update REQUIREMENTS.md to mark ISOL-01 and ISOL-02 complete

## Error Handling

- If dispatch fails for any job: log error, skip run, continue (don't abort entire scenario)
- If poll timeout: log job status as "TIMEOUT", mark run as fail
- If assigned_node mismatch: log mismatch, mark run as fail, continue
- If monitor JSON unparseable: log parse error, mark run as fail
- If 0 runs complete successfully: report as "No successful runs" (don't block milestone completion per CONTEXT.md)
