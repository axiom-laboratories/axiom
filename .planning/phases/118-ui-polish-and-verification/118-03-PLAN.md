  <name>Task 1: Fix GH #20 — Queue page 500 error on status filter</name>
  <files>puppeteer/agent_service/main.py</files>
  <action>
    **Issue:** Jobs.tsx calls `GET /api/jobs?status=COMPLETED,FAILED,CANCELLED` to filter job queue, but endpoint returns 500 error instead of filtered results.

    1. **Locate the endpoint** in `puppeteer/agent_service/main.py`: Find the `GET /api/jobs` or `/jobs` route handler. Likely signature: `async def get_jobs(current_user, status: str = None, ...)`.

    2. **Debug the status filter logic:** The error is probably in:
       - Query parameter parsing: `status` may be comma-separated string (`"COMPLETED,FAILED,CANCELLED"`) that needs splitting
       - Invalid enum values: Status values in the request may not match database enum (check case sensitivity and exact string values)
       - Missing null check: If `status` is None or empty, filter logic may fail
       - SQL error: Filter clause may be malformed (e.g., `Job.status.in_([])` with empty list)

    3. **Fix approach:**
       ```python
       @app.get("/api/jobs")
       async def get_jobs(
           current_user = Depends(require_permission("jobs:read")),
           status: str = None,
           limit: int = 50,
           offset: int = 0,
       ):
           query = db.query(Job)

           # Parse status filter: accept comma-separated values like "COMPLETED,FAILED"
           if status:
               status_values = [s.strip().upper() for s in status.split(',') if s.strip()]
               if status_values:
                   try:
                       # Validate enum values before filtering
                       valid_statuses = [JobStatus[s] for s in status_values]
                       query = query.filter(Job.status.in_(valid_statuses))
                   except KeyError as e:
                       return {"error": f"Invalid status value: {e}"}, 400

           jobs = query.limit(limit).offset(offset).all()
           return [JobResponse.from_orm(job) for job in jobs]
       ```

    4. **Test the fix:** Make API call from Jobs.tsx or curl:
       ```bash
       curl "https://localhost:8001/api/jobs?status=COMPLETED,FAILED,CANCELLED" \
         -H "Authorization: Bearer <TOKEN>"
       ```
       Should return 200 with job list (possibly empty if no jobs with those statuses).

    5. **Error handling:** If invalid status provided, return 400 with error message. Do not return 500.
  </action>
