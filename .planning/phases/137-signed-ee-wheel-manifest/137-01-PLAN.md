<objective>
Verify Ed25519-signed wheel manifest (containing SHA256 hash) before installing EE wheel. Manifest verification is a cryptographic gate that prevents installation of unsigned, tampered, or corrupted wheels. Any verification failure raises RuntimeError, causing graceful degradation to CE mode with visible error feedback in the /admin/licence endpoint.

Purpose: Enforce EE wheel integrity and authenticity at install time; provide operators with visibility into activation failures.

Output: Manifest verifier function, updated wheel installer with gate logic, error propagation through activate_ee_live, and test suite covering all failure modes plus happy path.
</objective>

async def activate_ee_live(app: Any, engine: Any) -> EEContext | None:
    """Install EE wheel, remove stubs, and load real EE plugins.
    
    Stores error message in app.state.ee_activation_error if activation fails.
    """
    existing = getattr(app.state, "ee", None)
    if existing and existing.foundry:
        logger.info("EE already active, skipping activation")
        return existing
    
    # Install wheel with manifest verification
    try:
        if not _install_ee_wheel():
            return None
    except RuntimeError as e:
        error_msg = str(e)
        app.state.ee_activation_error = error_msg
        logger.error("EE activation failed: %s", error_msg)
        return None
    
    # Clear error on successful install
    app.state.ee_activation_error = None
    
    # ... rest of plugin loading logic (entry points, plugin registration) ...
    # If plugin loading also fails, log it but DON'T override ee_activation_error
