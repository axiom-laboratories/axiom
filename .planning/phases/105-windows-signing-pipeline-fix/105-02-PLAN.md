### Add PowerShell tab to Step 0 (keypair generation)

In `docs/docs/getting-started/first-job.md`, add a `=== "Windows (PowerShell)"` tab after the existing `=== "openssl"` tab in the Step 0 section (around line 39-44).

The PowerShell tab should use the Python cryptography library via a PowerShell here-string, since Ed25519 is not natively supported in PowerShell crypto APIs:

```markdown
=== "Windows (PowerShell)"

    ```powershell
    python -c @"
    from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PrivateKey
    from cryptography.hazmat.primitives import serialization

    key = Ed25519PrivateKey.generate()

    with open('signing.key', 'wb') as f:
        f.write(key.private_bytes(
            serialization.Encoding.PEM,
            serialization.PrivateFormat.PKCS8,
            serialization.NoEncryption()
        ))

    with open('verification.key', 'wb') as f:
        f.write(key.public_key().public_bytes(
            serialization.Encoding.PEM,
            serialization.PublicFormat.SubjectPublicKeyInfo
        ))

    print('Done. Upload verification.key to Axiom, keep signing.key private.')
    "@
    ```
```

Use the exact content recovered from the Phase 103 commit, but remove any CRLF normalization lines if present (server handles it now). Adapt the here-string quoting if the Phase 103 version used a different approach.
</task>
