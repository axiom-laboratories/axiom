# Plan 105-01: CRLF Countersign Fix + Cold-Start Forced Password Change

## Objective

Fix two backend gaps found in the v18.0 milestone audit:

1. **CRLF countersign asymmetry** (WIN-05): When a Windows user submits a script with CRLF line endings, the server signs the raw CRLF bytes but the node normalizes to LF before verifying -- causing SECURITY_REJECTED. Fix by normalizing CRLF to LF server-side before both user signature verification and countersigning.

2. **Cold-start forced password change** (WIN-03): When ADMIN_PASSWORD is explicitly set (e.g. in .env), the admin bootstrap sets `must_change_password=False`, so fresh deploys with a known password skip the forced change prompt. Fix by always forcing password change, with an opt-out env var for CI/automation.

Output: main.py patched with both fixes + a new unit test for CRLF countersign symmetry.

### Write CRLF countersign unit test

Create `puppeteer/tests/test_crlf_countersign.py` with a test that:

1. Generates an Ed25519 keypair (user key + server key)
2. Creates a script string with CRLF line endings
3. Normalizes it to LF (same logic as the fix)
4. Signs the LF-normalized script with the user key
5. Verifies the user signature against the LF-normalized script
6. Signs the LF-normalized script with the server key (countersign)
7. Verifies the server countersignature against the LF-normalized script
8. Asserts that signing the CRLF script directly would produce a DIFFERENT signature (proving normalization is necessary)

```python
"""Test that CRLF normalization produces symmetric signatures between server and node."""
import pytest
from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PrivateKey


def _normalize(text: str) -> str:
    return text.replace('\r\n', '\n').replace('\r', '\n')


def test_crlf_script_countersign_matches_lf():
    """A CRLF script, once normalized, produces the same signature as the LF version."""
    user_key = Ed25519PrivateKey.generate()
    server_key = Ed25519PrivateKey.generate()

    script_crlf = "print('hello')\r\nprint('world')\r\n"
    script_lf = "print('hello')\nprint('world')\n"

    # Normalize (as the server now does)
    normalized = _normalize(script_crlf)
    assert normalized == script_lf

    # User signs the normalized script
    user_sig = user_key.sign(normalized.encode("utf-8"))
    # Verification succeeds
    user_key.public_key().verify(user_sig, normalized.encode("utf-8"))

    # Server countersigns the normalized script
    server_sig = server_key.sign(normalized.encode("utf-8"))
    # Node verifies using LF bytes (which is what node.py does)
    server_key.public_key().verify(server_sig, script_lf.encode("utf-8"))


def test_crlf_without_normalization_fails():
    """Without normalization, CRLF and LF produce different signatures."""
    key = Ed25519PrivateKey.generate()

    script_crlf = "print('hello')\r\nprint('world')\r\n"
    script_lf = "print('hello')\nprint('world')\n"

    sig_crlf = key.sign(script_crlf.encode("utf-8"))

    # Verifying a CRLF signature against LF bytes must fail
    with pytest.raises(Exception):
        key.public_key().verify(sig_crlf, script_lf.encode("utf-8"))
```

Use the exact patterns above. The test file must be self-contained (no app imports needed -- pure cryptography test).
</task>
