# Plan 99-01: Diff-based sync and create_task() dispatcher

## Goal

Replace `sync_scheduler()`'s remove-all/re-add pattern with a diff-based algorithm that protects internal `__`-prefixed jobs, and wrap `execute_scheduled_job` in a synchronous `asyncio.create_task()` callback so APScheduler cron fires return immediately without blocking the event loop.

## Must-Haves

- `sync_scheduler()` never calls `remove_all_jobs()` — confirmed by source inspection test
- Internal jobs (`__prune_node_stats__`, `__prune_execution_history__`, `__dispatch_timeout_sweeper__`) survive any number of `sync_scheduler()` calls
- `add_job()` calls in `sync_scheduler()` use `replace_existing=True`
- Cron fire callback is a synchronous function that calls `asyncio.get_event_loop().create_task()` and returns immediately
- `execute_scheduled_job()` coroutine body is unchanged (all existing logic, skip checks, job creation retained)
- `ScheduledFireLog.status = 'failed'` written by done-callback when task raises an exception
- `get_scheduling_health()` counts `'failed'` fire_log rows in both the `fired` and `failed` aggregates
- All 9 phase tests pass

## Context

`scheduler_service.py:126-149` — `sync_scheduler()` currently calls `self.scheduler.remove_all_jobs()` which destroys all `__`-prefixed internal maintenance jobs. They are never re-added by `sync_scheduler()`, so any CRUD operation silently kills pruning and timeout sweeping.

`scheduler_service.py:151-258` — `execute_scheduled_job()` is a long async coroutine registered directly with APScheduler. Under cron burst load, APScheduler awaits each callback on the event loop, delaying heartbeat and WebSocket processing.

<description>
Replace the body of `sync_scheduler()` (lines 126-149) with a diff-based algorithm. The new implementation must:

1. **Never call `self.scheduler.remove_all_jobs()`** — instead compute a diff.

2. **Build desired set** — query DB for all active `ScheduledJob` rows where `is_active == True` and `schedule_cron` is not None/empty and has exactly 5 parts when split.

3. **Build current set** — enumerate `self.scheduler.get_jobs()` and collect IDs that do NOT start with `__`. Internal jobs are completely excluded from the diff.

4. **Compute diff**:
   - `to_remove`: job IDs in current set but not in desired IDs → call `self.scheduler.remove_job(job_id)` for each
   - `to_add_or_update`: all jobs in desired set → call `self.scheduler.add_job(...)` with `replace_existing=True`

5. **add_job() call** — use the synchronous wrapper `self._make_cron_callback(j.id)` as the func (created in task 99-01-02 below). Pass `replace_existing=True`. All existing cron parameters (minute, hour, day, month, day_of_week) unchanged from current implementation.

6. **Log line** — keep `logger.info(f"✅ Scheduler Synced: {count} jobs active.")` unchanged.

New sync_scheduler() skeleton:
```python
async def sync_scheduler(self):
    """Syncs DB ScheduledJobs with APScheduler using diff-based algorithm."""
    logger.info("🔄 Syncing Scheduler...")
    async with db_module.AsyncSessionLocal() as session:
        result = await session.execute(
            select(ScheduledJob).where(ScheduledJob.is_active == True)
        )
        db_jobs = result.scalars().all()

    # Build desired: {id -> ScheduledJob} for active jobs with valid cron
    desired: dict = {}
    for j in db_jobs:
        if j.schedule_cron:
            parts = j.schedule_cron.split()
            if len(parts) == 5:
                desired[j.id] = j

    # Build current: IDs in APScheduler excluding internal jobs
    current_ids = {
        job.id for job in self.scheduler.get_jobs()
        if not job.id.startswith('__')
    }

    # Remove jobs no longer desired
    to_remove = current_ids - set(desired.keys())
    for job_id in to_remove:
        try:
            self.scheduler.remove_job(job_id)
        except Exception as e:
            logger.warning(f"⚠️ Could not remove job {job_id}: {e}")

    # Add or update all desired jobs
    count = 0
    for j in desired.values():
        parts = j.schedule_cron.split()
        try:
            self.scheduler.add_job(
                self._make_cron_callback(j.id),
                'cron',
                minute=parts[0], hour=parts[1], day=parts[2],
                month=parts[3], day_of_week=parts[4],
                id=j.id,
                replace_existing=True,
            )
            count += 1
        except Exception as e:
            logger.error(f"❌ Failed to schedule {j.name}: {e}")

    logger.info(f"✅ Scheduler Synced: {count} jobs active.")
```

Note: `_make_cron_callback` is added in task 99-01-02.

**Method 1: `_make_cron_callback(scheduled_job_id: str) -> callable`**

Returns a synchronous function that, when called by APScheduler, creates a task for `execute_scheduled_job` and attaches a done-callback for failure tracking.

```python
def _make_cron_callback(self, scheduled_job_id: str):
    """Returns a synchronous APScheduler callback that spawns execute_scheduled_job as a task."""
    def _callback():
        try:
            loop = asyncio.get_event_loop()
            task = loop.create_task(self.execute_scheduled_job(scheduled_job_id))
            task.add_done_callback(
                lambda t: self._on_cron_task_done(t, scheduled_job_id)
            )
        except Exception as e:
            logger.error(f"❌ Failed to create task for scheduled job {scheduled_job_id}: {e}")
    return _callback
```

**Method 2: `_on_cron_task_done(task, scheduled_job_id: str) -> None`**

Synchronous done-callback. Schedules a DB update coroutine if the task raised an exception.

```python
def _on_cron_task_done(self, task: asyncio.Task, scheduled_job_id: str):
    """Done-callback: if task raised, update the most recent fire_log row to 'failed'."""
    if task.cancelled():
        return
    exc = task.exception()
    if exc is not None:
        logger.error(f"❌ Scheduled job task {scheduled_job_id} raised: {exc}")
        try:
            loop = asyncio.get_event_loop()
            loop.create_task(self._mark_latest_fire_failed(scheduled_job_id, str(exc)))
        except Exception as e:
            logger.error(f"❌ Could not schedule fire_log failure update: {e}")
```

**Method 3: `_mark_latest_fire_failed(scheduled_job_id: str, error_msg: str) -> None`**

Async coroutine that opens a fresh DB session and updates the most recent `ScheduledFireLog` row for the given job to `status='failed'`.

```python
async def _mark_latest_fire_failed(self, scheduled_job_id: str, error_msg: str):
    """Update the most recent fire_log row for this job to status='failed'."""
    from sqlalchemy import desc as _desc
    async with db_module.AsyncSessionLocal() as session:
        result = await session.execute(
            select(ScheduledFireLog)
            .where(ScheduledFireLog.scheduled_job_id == scheduled_job_id)
            .order_by(_desc(ScheduledFireLog.created_at))
            .limit(1)
        )
        fire_log = result.scalar_one_or_none()
        if fire_log and fire_log.status == 'fired':
            fire_log.status = 'failed'
            await session.commit()
            logger.warning(f"⚠️ Marked fire_log {fire_log.id} as failed for job {scheduled_job_id}: {error_msg}")
```

The `start()` method's internal job registrations do NOT use `_make_cron_callback` — they remain as direct coroutine references since they are interval-based and not subject to the cron burst problem.

```python
import time, asyncio
from unittest.mock import patch, AsyncMock

def test_cron_callback_returns_immediately():
    """Synchronous cron wrapper must return in < 50ms even if the coroutine is slow."""
    from agent_service.services.scheduler_service import SchedulerService

    async def slow_coroutine(job_id):
        await asyncio.sleep(0.5)

    svc = SchedulerService()
    callback = svc._make_cron_callback("test-job-timing")

    async def run():
        loop = asyncio.get_event_loop()
        with patch.object(svc, 'execute_scheduled_job', side_effect=slow_coroutine):
            start = time.monotonic()
            callback()
            elapsed = time.monotonic() - start
        assert elapsed < 0.05, f"Callback took {elapsed:.3f}s — must return in < 50ms"
        # Cancel any pending tasks to avoid warnings
        for task in asyncio.all_tasks():
            if not task.done() and task != asyncio.current_task():
                task.cancel()
                try:
                    await asyncio.wait_for(asyncio.shield(task), timeout=0.1)
                except (asyncio.CancelledError, asyncio.TimeoutError):
                    pass

    asyncio.run(run())
```
