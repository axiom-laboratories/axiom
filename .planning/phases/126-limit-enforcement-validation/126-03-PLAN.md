Investigate and resolve the Podman node enrollment failure (403 Forbidden during JOIN_TOKEN validation). The issue from Plan 01 was that the node could not complete enrollment, preventing it from sending heartbeats with execution_mode='podman'.

**Root Cause Analysis:**

From VERIFICATION.md, the failure manifests as:
- Node created but enrollment blocked with 403 Forbidden
- JOIN_TOKEN extraction and validation failing
- Likely cause: Token revocation tracking or token expiration

**Debugging steps:**

1. **Check Podman node logs:**
```bash
cd mop_validation/local_nodes
docker compose -f podman-node-compose.yaml logs puppet-podman --tail 50
# Look for error messages like:
# - "403 Forbidden" during enrollment
# - "JOIN_TOKEN invalid" or "certificate signing request rejected"
# - "AGENT_URL unreachable" (network issue)
```

2. **Verify JOIN_TOKEN environment variable:**
```bash
# Check if JOIN_TOKEN is set in podman-node-compose.yaml
grep -A 5 "JOIN_TOKEN" mop_validation/local_nodes/podman-node-compose.yaml
# If it references a variable like ${JOIN_TOKEN_PODMAN}, check if it exists:
echo ${JOIN_TOKEN_PODMAN:-"NOT SET"}
# If not set, extract from secrets.env:
grep JOIN_TOKEN mop_validation/secrets.env || grep JOIN_TOKEN ~/.mop_secrets.env
```

3. **Test JOIN_TOKEN validity:**
```bash
# Decode the JOIN_TOKEN to verify it's valid base64 JSON
TOKEN=$(cat ~/.mop_secrets.env | grep '^JOIN_TOKEN=' | cut -d= -f2)
echo "$TOKEN" | base64 -d | jq .
# Should output: { "root_ca_pem": "...", "api_url": "...", "agent_ca": "..." }
```

4. **Check if token was revoked:**
```bash
# Query orchestrator for revoked tokens (if endpoint exists)
curl -s -k -H "Authorization: Bearer $(cat ~/.mop_auth_token)" \
  https://localhost:8001/system/crl.pem | openssl crl -in /dev/stdin -text -noout | head -20
# If Podman node cert is listed, it was revoked — generate fresh token
```

**Fix strategies (choose based on root cause):**

**Strategy A: Fresh JOIN_TOKEN generation**
If token is revoked or expired:
```bash
# Generate fresh token via API or admin tool
# Option 1: Use API if available
curl -X POST -k -H "Authorization: Bearer $(cat ~/.mop_auth_token)" \
  https://localhost:8001/admin/tokens/generate \
  -H "Content-Type: application/json" \
  -d '{"purpose": "podman-node-test"}' | jq '.token' > /tmp/new_token.txt

# Option 2: Use admin CLI tool if available
python3 ~/Development/toms_home/.agents/tools/admin_signer.py --generate-token

# Update compose file with new token
sed -i 's/JOIN_TOKEN=.*/JOIN_TOKEN='"$(cat /tmp/new_token.txt)"'/' \
  mop_validation/local_nodes/podman-node-compose.yaml
```

**Strategy B: Fix AGENT_URL routing (if network issue)**
If node cannot reach Puppeteer:
```bash
# Verify Puppeteer is accessible from inside Docker
docker run --rm --entrypoint bash alpine:latest -c \
  "curl -s -k https://host.docker.internal:8001/nodes | head -20"

# If curl fails, update AGENT_URL in compose file:
# From: AGENT_URL=http://host.docker.internal:8001
# To: AGENT_URL=http://docker.for.mac.localhost:8001 (if on macOS)
# Or: AGENT_URL=http://172.17.0.1:8001 (if using custom bridge)
```

**Strategy C: Verify node certificate chain**
If enrollment cert signing failed:
```bash
# Check if node's CSR was signed; verify cert dates
if [ -f mop_validation/local_nodes/secrets/node-podman.crt ]; then
  openssl x509 -in mop_validation/local_nodes/secrets/node-podman.crt -text -noout | grep -A 2 "Not Before\|Not After"
fi
```

**After fix, redeploy Podman node:**
```bash
cd mop_validation/local_nodes
# Stop old container
docker compose -f podman-node-compose.yaml down

# Update compose file if needed (new token, AGENT_URL, etc.)
# nano podman-node-compose.yaml

# Restart with fresh deployment
docker compose -f podman-node-compose.yaml up -d

# Wait for enrollment and first heartbeat
sleep 30

# Verify enrollment succeeded
curl -s -k -H "Authorization: Bearer $(cat ~/.mop_auth_token)" \
  https://localhost:8001/nodes | jq '.[] | select(.execution_mode == "podman")'
# Expected output: Podman node with execution_mode='podman' and cgroup_version='v2'
```

**Success criteria for this task:**
- Podman node container is running: `docker ps | grep podman`
- Node has enrolled and sent heartbeat: `GET /nodes` includes a node with execution_mode='podman'
- Heartbeat includes cgroup_version field: `GET /nodes | jq '.[].cgroup_version'` shows 'v2'
- Node is responsive: can dispatch a job to it without errors
