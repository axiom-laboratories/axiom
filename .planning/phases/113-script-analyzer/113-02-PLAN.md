## Key Decisions from CONTEXT.md

**Suggestion presentation:**
- Grouped table by ecosystem (Python, APT, npm, etc.)
- Columns: package name, detected import/command, confidence, status badge (Approved/New/Pending)
- Approved packages: greyed out, show "Approved" badge, show which blueprints use them + node availability count
- New packages: checkboxes for bulk selection, "Select all new" toggle, "Approve Selected" button
- Stdlib modules silently excluded (never shown)

**Analysis trigger:**
- Explicit "Analyze Script" button (no auto-analyze on paste)
- Language auto-detected but dropdown allows override
- Results table appears below after analysis

**Approval workflow:**
- "Request Approval" button for operators without foundry:write
- "Approve Selected" button for operators with foundry:write (skip queue, direct approval)
- Separate "Review Queue" panel for admins in Admin.tsx

Create ScriptAnalyzerPanel.tsx React component (~450 lines) with:

1. **Component structure**:
   - Textarea for script paste (placeholder: "Paste your Python, Bash, or PowerShell script here...")
   - Language dropdown (auto-detected, user can override): Python | Bash | PowerShell
   - "Analyze Script" button (triggers analysis)
   - Results table (hidden until analysis completes)

2. **State management** (via useState):
   - scriptText: string (textarea content)
   - detectedLanguage: string (from auto-detection)
   - selectedLanguage: string (user override, defaults to detected)
   - analysisResults: SuggestionResult[] | null (null while loading or before analysis)
   - isAnalyzing: boolean (show spinner during fetch)
   - selectedSuggestions: Set<string> (package_name keys for checkboxes)
   - selectAllNew: boolean (bulk toggle state)

3. **Query/mutation hooks** (React Query):
   - useMutation for POST /api/analyzer/analyze-script
   - useMutation for POST /api/analyzer/requests (approval request)

4. **Analysis flow**:
   - User pastes script, dropdown shows detected language (can override)
   - Click "Analyze Script" → calls analyzeScript mutation with script + language
   - While loading: show spinner + "Analyzing..."
   - On success: display results table below textarea
   - On error: show toast error + clear results

5. **Results table** (grouped by ecosystem):
   - Columns: [Checkbox], Package Name, Import/Command, Confidence, Status Badge, Actions
   - Rows grouped by ecosystem headers (e.g., "Python (PyPI)", "Linux (APT)", "npm", "PowerShell")
   - For each suggestion:
     - Checkbox: disabled if status="approved", checked state in selectedSuggestions
     - Package name: clickable → shows detailed info (hover tooltip)
     - Import/command: grey text showing original detected text
     - Confidence: "High" | "Medium" | "Low" as small pill badge
     - Status badge:
       - Green "Approved" → greyed out row, not selectable, show blueprints + node count
       - Blue "New" → full color, selectable, highlight as new
       - Orange "Pending" → yellow/orange badge, not selectable
     - Actions column:
       - If status="approved": show "In {blueprint_name}, {blueprint_name}" + "Ready on {node_count} nodes"
       - If status="new": no action (selected via checkbox)
       - If status="pending": show "Awaiting admin review" label

6. **Bulk selection**:
   - Checkbox on table header "Select all new packages" → toggles all unchecked NEW rows
   - Count badge: "Selected {count} new packages"

7. **Action buttons** (below table):
   - If user has foundry:write permission: "Approve Selected" button (creates ApprovedIngredient immediately)
   - If user is operator/viewer (no foundry:write): "Request Approval" button (creates ScriptAnalysisRequest with PENDING status)
   - Both disabled if no rows selected
   - On click: show toast "Approving..." → on success "Approved {count} packages"

8. **Tooltip on mapped imports**:
   - cv2 → opencv-python: show subtle indicator (e.g., "mapped" badge or info icon)
   - Hover shows: "cv2 is the import name; package is opencv-python"

9. **Access control**:
   - Analyzer form visible to all authenticated users (analyze is read-only)
   - "Approve Selected" button visible only if user has foundry:write permission
   - "Request Approval" button visible if user is operator+ (default)

10. **Error handling**:
    - Empty script: show validation error "Please paste a script"
    - Analysis fails: show error toast + specific message from API
    - Malformed script: show warning "Script has syntax errors; analysis may be incomplete"
    - No suggestions found: show "No packages detected in script"

Pattern: Follow SmelterIngredientSelector.tsx component structure (tabs, forms, mutation handlers, permission checks). Use shadcn/ui components (Dialog, Button, Input, Select, Checkbox, Badge). Tailwind for spacing (space-y-4, p-4).

     - Status: badge color-coded (PENDING=blue, APPROVED=green, REJECTED=red)
     - Requested At: human-readable timestamp (e.g., "2 hours ago")
     - Actions:
       - If PENDING: "Approve" + "Reject" buttons
       - If APPROVED: show reviewed_by user + timestamp
       - If REJECTED: show "Rejected by {user}" + review_reason in tooltip

5. **Approve action**:
   - Click "Approve" → show confirmation toast "Approving {package}..."
   - API call POST /api/analyzer/requests/{id}/approve
   - On success: row status changes to green "APPROVED", timestamp updates, buttons disappear
   - On error: show error toast with specific message (e.g., "Already approved")
   - Auto-refresh table query (queryClient.invalidateQueries)

6. **Reject action**:
   - Click "Reject" → open dialog with:
     - Message: "Are you sure? Operator will see rejection reason if provided."
     - Textarea: "Rejection reason (optional)" placeholder
     - Cancel + Reject buttons
   - On Reject: POST /api/analyzer/requests/{id}/reject with reason
   - On success: row status changes to red "REJECTED", show reason in tooltip
   - Auto-refresh table

7. **Empty states**:
   - No PENDING requests: show "All caught up! No pending requests."
   - No requests at all (all tabs empty): show "No requests yet."

8. **Performance**:
   - Show skeleton loaders while querying
   - Pagination or infinite scroll for many requests (use existing dashboard patterns)
   - Limit initial query to 50 requests (newer first)

9. **Permission check**:
   - Component visible only if user has foundry:write permission
   - If viewer-only user lands on this component: show "Admin access required"

10. **WebSocket integration** (optional, nice-to-have):
    - Listen for WebSocket updates on script analysis request status changes
    - Auto-refresh table row when status changes from another admin's approval
    - Use useWebSocket hook pattern from existing codebase
