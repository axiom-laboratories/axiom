Output: Fully tested analyzer service with endpoints, DB schema for approval queue, and integration with existing ApprovedIngredient table.

    - Malformed Python (SyntaxError) → returns empty list with error flag, no crash

Create AnalyzerService class in puppeteer/agent_service/services/analyzer_service.py with:

1. **Static import mapping dict** (~200 entries):
```python
IMPORT_TO_PACKAGE = {
    "cv2": "opencv-python",
    "PIL": "Pillow",
    "yaml": "PyYAML",
    "sklearn": "scikit-learn",
    "bs4": "beautifulsoup4",
    "jwt": "PyJWT",
    "dateutil": "python-dateutil",
    # ... extend to ~200 total entries covering most common cases
}
```

2. **Language detection** (`_detect_language(script_text: str) -> str`):
   - Check shebang line (#! prefix) for python/bash/powershell keywords
   - Fallback to syntax patterns: "import " or "from " → python; "apt-get"/"yum"/"apk add" → bash; "Import-Module" → powershell
   - Default to "bash" if undetected
   - Return: "python" | "bash" | "powershell"

3. **Python analysis** (`_analyze_python(script_text: str) -> List[Dict[str, Any]]`):
   - Use `ast.parse()` to parse script into AST tree
   - Walk tree for ast.Import and ast.ImportFrom nodes
   - Extract module names (top-level only, split on "." to get root)
   - Filter out stdlib modules via `sys.stdlib_module_names` (Python 3.10+) or hardcoded fallback list
   - For each import, check IMPORT_TO_PACKAGE mapping, return resolved package name
   - Return: `[{"import_name": "cv2", "package_name": "opencv-python", "mapped": True}, {"import_name": "requests", "package_name": "requests", "mapped": False}, ...]`
   - On SyntaxError: catch and return empty list + log warning

4. **Bash analysis** (`_analyze_bash(script_text: str) -> List[Dict[str, Any]]`):
   - Regex pattern: `(?:apt-get|apt|yum|dnf|apk\s+add|pip\s+install)\s+([^\n;]+)`
   - For each match, extract the package list after the command
   - Split on whitespace, strip version specifiers (==1.0, >=2.0, etc.) via `re.sub(r'[<>=!~]+.*', '', pkg)`
   - Skip tokens starting with `-` (flags)
   - Return: `[{"import_name": "curl", "package_name": "curl", "ecosystem": "APT"}, ...]`

5. **PowerShell analysis** (`_analyze_powershell(script_text: str) -> List[Dict[str, Any]]`):
   - Regex patterns for: `Import-Module`, `Install-Module`, `Install-Package`
   - Extract module/package name from `-Name` parameter or positional arg
   - Return: `[{"import_name": "Az.Storage", "package_name": "Az.Storage", "ecosystem": "NUGET"}, ...]`

6. **Main async method** (`analyze_script(script_text: str, language: Optional[str] = None, db: AsyncSession = None) -> Dict[str, Any]`):
   - Auto-detect language if not provided
   - Dispatch to appropriate _analyze_* method
   - Return: `{"detected_language": "python", "suggestions": [...]}`

Patterns: Follow resolver_service.py and smelter_service.py structure (async methods, proper error handling, logging). Use regex for Bash/PowerShell (simple patterns), ast for Python (robust). No new dependencies required (all Python builtins + existing FastAPI/SQLAlchemy).

Create analyzer_router.py in puppeteer/agent_service/ee/routers/ with:

1. **Pydantic models** (in ee/interfaces/analyzer.py or inline):
   - `AnalyzeScriptRequest`: { script: str, language?: str }
   - `SuggestionResult`: { ecosystem: str, package_name: str, detected_import: str, confidence: str, status: str, is_approved: bool, is_stdlib: bool, blueprints_using?: List[str], nodes_ready?: int, mapped_from?: str, mapping_note?: str }
   - `AnalyzeScriptResponse`: { detected_language: str, suggestions: List[SuggestionResult] }
   - `ScriptAnalysisRequestModel`: { id: UUID, requester_id: UUID, package_name: str, ecosystem: str, detected_import: str, status: str, created_at: datetime, reviewed_at?: datetime, reviewed_by?: UUID, review_reason?: str }
   - `ApproveRequestPayload`: { reason?: str } (for rejection reason)

2. **POST /api/analyzer/analyze-script** (async):
   - Require auth (any authenticated user — viewers can analyze)
   - Accept AnalyzeScriptRequest
   - Call `AnalyzerService.analyze_script(script, language, db)`
   - For each suggestion, cross-reference against ApprovedIngredient:
     - Query ApprovedIngredient by package_name + ecosystem
     - If found: status="approved", show blueprints using it, show nodes with that capability
     - If not found: status="new"
     - If found in PENDING requests: status="pending"
   - Return AnalyzeScriptResponse with full enriched suggestions
   - No audit() call (read-only operation)

3. **GET /api/analyzer/requests** (async):
   - Require auth + foundry:read (can view own requests or all as admin)
   - If user is admin with foundry:write: return all PENDING requests
   - If user is non-admin: return only user's own requests
   - Return paginated list of ScriptAnalysisRequestModel
   - Filter by status (default PENDING) via query param ?status=pending

4. **POST /api/analyzer/requests** (async):
   - Require auth + operator permission (foundry:read or foundry:write)
   - Accept { package_name: str, ecosystem: str, detected_import: str, source_script_hash: str }
   - Check if already in ApprovedIngredient (same package + ecosystem)
   - If yes: return 422 "Package already approved"
   - Check if duplicate pending request (same user + package + ecosystem + script_hash)
   - If yes: return 422 "Request already pending"
   - Create ScriptAnalysisRequest with status=PENDING
   - audit("SCRIPT_ANALYSIS_REQUEST_CREATED", ...)
   - Return ScriptAnalysisRequestModel with 201
   - On success: call `resolver_service.resolve_ingredient_tree()` in background (asyncio.create_task) — don't wait

5. **POST /api/analyzer/requests/{request_id}/approve** (async):
   - Require auth + foundry:write permission
   - Load ScriptAnalysisRequest by id
   - If not found: 404
   - If not PENDING: 422 "Request already reviewed"
   - Create ApprovedIngredient with:
     - package_name from request
     - ecosystem from request
     - is_active=True
   - Update request: status=APPROVED, reviewed_at=now, reviewed_by=current_user.id
   - audit("SCRIPT_ANALYSIS_REQUEST_APPROVED", package_name=..., ecosystem=..., requester_id=...)
   - Trigger transitive resolution: `asyncio.create_task(resolver_service.resolve_ingredient_tree(approved_ingredient.id))`
   - Return 200 + updated ScriptAnalysisRequestModel
   - Broadcast WebSocket update (mirror pattern from license reload in Phase 116)

6. **POST /api/analyzer/requests/{request_id}/reject** (async):
   - Require auth + foundry:write permission
   - Accept { reason?: str }
   - Load request by id
   - If not PENDING: 422
   - Update request: status=REJECTED, reviewed_at=now, reviewed_by=current_user.id, review_reason=payload.reason
   - audit("SCRIPT_ANALYSIS_REQUEST_REJECTED", package_name=..., reason=...)
   - Return 200 + updated request
   - Broadcast WebSocket update

Pattern: Follow smelter_router.py structure (async, proper error codes, audit integration, permission decorators). Use `require_permission("foundry:write")` for approval/reject, `require_permission("foundry:read")` for viewing requests.
