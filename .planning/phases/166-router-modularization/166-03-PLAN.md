<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → System routes | `/system/health`, `/api/features`, `/api/licence` return public info; no auth required for most |
| WebSocket authentication | `/ws` endpoint validates JWT in query param; only authenticated clients receive broadcasts |
| Admin routes | `/admin/users`, `/admin/roles`, `/admin/audit-log` require `require_permission("admin:write")` or `require_permission("users:write")` |
| Smelter routes | `/api/smelter/...` require `require_permission("smelter:read")` or `require_permission("smelter:write")` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-166-09 | Information Disclosure | System health endpoint | mitigate | `/system/health` returns public metrics (CPU, memory) only; no secrets exposed |
| T-166-10 | Elevation of Privilege | Admin routes use require_permission | mitigate | All admin routes preserve `require_permission()` checks from original main.py; FastAPI `Depends()` ensures per-request evaluation |
| T-166-11 | Spoofing | WebSocket token validation | mitigate | `/ws` validates JWT signature and token_version before accepting connection; expired or invalid tokens rejected with 1008 close code |
| T-166-12 | Tampering | Audit log extraction | mitigate | Audit logs are read-only from API (`GET /admin/audit-log`); no write operations on logs from routers |

**Security assessment:** No new vulnerabilities introduced. All auth/permission/audit logic preserved from original main.py. WebSocket token validation remains unchanged. Smelter routes maintain same CVE scanning RBAC.
