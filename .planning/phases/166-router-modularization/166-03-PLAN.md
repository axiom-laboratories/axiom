## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-166-01 | I (Information Disclosure) | Router imports in main.py | mitigate | All routers imported at module level; no dynamic imports; source code control via git |
| T-166-02 | T (Tampering) | Route handlers moved to routers | mitigate | All handlers preserve original auth/permission decorators; no changes to logic |
| T-166-03 | R (Repudiation) | Audit logging distributed across routers | mitigate | All routers call audit() before db.commit(); same signature as original |
| T-166-04 | D (Denial of Service) | Middleware ordering unchanged | accept | Middleware applied at app level before routing; no per-router middleware yet (added in Phase 167) |
| T-166-05 | A (Authentication) | JWT/mTLS auth in routers | mitigate | All routers use Depends(get_current_user) or mTLS checks; no auth logic changed |
| T-166-06 | E (Elevation of Privilege) | Permission checks in routers | mitigate | require_permission() factory applied consistently; no permissions escalated or removed |
