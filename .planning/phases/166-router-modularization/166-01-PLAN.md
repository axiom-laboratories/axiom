## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client→API | HTTP requests to FastAPI app; must be validated by routers using `Depends(get_current_user)` and `Depends(require_permission(...))`  |
| Router→Database | AsyncSession from `get_db()` dependency; all SQL is parameterized via SQLAlchemy ORM |
| Router→WebSocket broadcast | `ws_manager` singleton exported from main.py; used by routers to broadcast events to connected clients |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-166-01 | Elevation of Privilege | `require_permission()` calls in routers | mitigate | Per CONTEXT.md D-03, all permission checks preserved exactly as-is during extraction; FastAPI's `Depends()` ensures per-request evaluation |
| T-166-02 | Spoofing | JWT verification in auth routes | mitigate | JWT validation in auth_router identical to original main.py routes; no changes to token generation or verification logic |
| T-166-03 | Tampering | Audit logging order (audit before commit) | mitigate | All `audit()` calls preserved before `await db.commit()`; verified during extraction |
| T-166-04 | Repudiation | WebSocket broadcast to unauthenticated clients | accept | `/ws` endpoint validates JWT in query param (`?token=...`); only authenticated clients receive broadcasts. No change in security posture — this is existing behavior. |

**Security assessment:** Router refactoring is a pure internal reorganization. No authentication, permission, or audit logic is modified — only relocated from main.py to routers via imports from deps.py. STRIDE threat posture is unchanged.
