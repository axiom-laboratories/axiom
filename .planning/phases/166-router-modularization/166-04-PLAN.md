## Test Client Pattern (conftest.py)

All tests use the FastAPI TestClient:
```python
from fastapi.testclient import TestClient
from agent_service.main import app

client = TestClient(app)

def test_example():
    response = client.get("/api/health")
    assert response.status_code == 200
```

The `app` must be importable from agent_service.main. If router wiring is broken, app initialization will fail and conftest.py will crash.

## Async Test Support

pytest.ini should have:
```ini
[pytest]

```

This allows `async def test_*()` functions to work with @pytest.mark.asyncio.

## What to Watch For

Common breakage patterns:
- **Circular imports:** ws_manager in main.py is imported by routers inside handlers (not at module level)
- **Missing imports in routers:** forgot to import a Pydantic model, dependency, or DB model
- **Wrong FastAPI dependency syntax:** using `Depends()` incorrectly, or missing it entirely
- **Audit ordering:** audit() called AFTER db.commit() instead of BEFORE
- **WebSocket auth:** `/ws` endpoint not wired or auth check missing
- **mTLS endpoints:** `/api/enroll`, `/work/pull`, `/heartbeat` expecting cert instead of JWT

If tests fail, the error message will pinpoint the issue (import error, missing endpoint, wrong status code, etc.).
