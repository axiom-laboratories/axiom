<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | HTTP requests to FastAPI app; must be validated by routers using `Depends(get_current_user)` and `Depends(require_permission(...))`  |
| Unauthenticated agent endpoints | `/api/enroll`, `/work/pull`, `/heartbeat` use mTLS client certs and node secrets, not JWT |
| Router → Database | AsyncSession from `get_db()` dependency; all SQL is parameterized via SQLAlchemy ORM |
| Router → WebSocket broadcast | `ws_manager` singleton exported from main.py; used by routers to broadcast events to connected clients |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-166-05 | Elevation of Privilege | `require_permission()` calls in nodes/workflows routers | mitigate | All permission checks preserved exactly as-is during extraction; FastAPI's `Depends()` ensures per-request evaluation |
| T-166-06 | Spoofing | mTLS verification in agent endpoints | mitigate | `/api/enroll`, `/work/pull`, `/heartbeat` use mTLS client certs and node secrets (not JWT); extracted without modification |
| T-166-07 | Tampering | Audit logging order (audit before commit) | mitigate | All `audit()` calls preserved before `await db.commit()`; verified during extraction |
| T-166-08 | Repudiation | WebSocket broadcast to unauthenticated clients | accept | `/ws` endpoint validates JWT in query param (`?token=...`); only authenticated clients receive broadcasts. No change in security posture — this is existing behavior. |

**Security assessment:** Router refactoring is a pure internal reorganization. No authentication, permission, or audit logic is modified — only relocated from main.py to routers via imports from deps.py. STRIDE threat posture is unchanged.
