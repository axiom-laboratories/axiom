  <name>Task 4: Fix WorkflowDetail.test.tsx — ensure mock resolves before duration assertion</name>
  <files>puppeteer/dashboard/src/views/__tests__/WorkflowDetail.test.tsx</files>
  <action>
The test "calculates and displays run duration" (line 196 in WorkflowDetail.test.tsx) fails because the assertion `expect(screen.getByText('300.0s')).toBeInTheDocument()` runs before the mocked fetch completes.

Root cause: The test uses `setTimeout(..., 100)` which is arbitrary and may not give enough time for React Query to fetch and render.

Fix: Replace the arbitrary setTimeout with a proper `waitFor()` call that waits for the duration text to appear:

```javascript
it('calculates and displays run duration', async () => {
  const mockFetch = vi
    .spyOn(authModule, 'authenticatedFetch')
    .mockResolvedValueOnce({
      ok: true,
      json: async () => sampleWorkflowDetail,
    } as any)
    .mockResolvedValueOnce({
      ok: true,
      json: async () => ({
        runs: sampleRunHistory,
        total: 2,
        skip: 0,
        limit: 10,
      }),
    } as any);

  renderWorkflowDetail();

  // Wait for the duration to appear in the document
  await waitFor(() => {
    expect(screen.getByText('300.0s')).toBeInTheDocument();
  }, { timeout: 5000 });

  mockFetch.mockRestore();
});
```

This ensures the test waits for React Query to complete both fetch calls, render the run history table, and display the calculated duration before asserting.

Import `waitFor` from '@testing-library/react' if not already imported (line 2).

Root cause: Arbitrary setTimeout doesn't guarantee mock resolution before assertion.
