## Backend Mirror Operations (APT + apk)

### 1. Start the Docker stack with EE mirrors enabled
```bash
cd puppeteer
docker compose -f compose.server.yaml -f compose.ee.yaml up -d
```

Wait for all services to be healthy (~30s). Verify mirrors are running:
```bash
docker compose -f compose.server.yaml -f compose.ee.yaml ps | grep -E "pypi|mirror"
```

Expected: `pypi` and `mirror` services show as "Up".

### 2. Test APT mirror backend
Via API (use curl or Postman with JWT token):
```bash
# Get a valid JWT token from login
TOKEN=$(curl -s -X POST http://localhost:8001/api/auth/login \
  -d "username=admin&password=YOUR_PASSWORD" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  | jq -r '.access_token')

# Approve an APT ingredient (curl, nano, git — common Debian packages)
curl -X POST http://localhost:8001/api/smelter/ingredients \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "curl",
    "ecosystem": "APT",
    "version_constraint": "==7.81.0",
    "cve_status": "OK"
  }'
```

Expected response: 201 Created, ingredient.id returned.

Verify the APT mirror backend ran:
```bash
# Check mirror-data/apt/ for downloaded .deb files
docker exec puppeteer-agent-1 ls -lah /app/mirror_data/apt/

# Check for Packages.gz index
docker exec puppeteer-agent-1 file /app/mirror_data/apt/Packages.gz
```

Expected: Packages.gz should be a gzip-compressed file (file returns "gzip compressed data").

Test health check:
```bash
curl http://localhost:8001/api/system/health | jq '.mirrors_available'
```

Expected: `true` (both mirrors reachable).

### 3. Test apk (Alpine) mirror backend
```bash
# Approve an Alpine package
curl -X POST http://localhost:8001/api/smelter/ingredients \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "curl",
    "ecosystem": "APK",
    "version_constraint": "==8.0.1",
    "cve_status": "OK"
  }'
```

Verify apk mirror:
```bash
# Check mirror-data/apk/v*/main/ for .apk files
docker exec puppeteer-agent-1 ls -lah /app/mirror_data/apk/v*/main/

# Check for APKINDEX.tar.gz
docker exec puppeteer-agent-1 file /app/mirror_data/apk/v3.20/main/APKINDEX.tar.gz
```

Expected: APKINDEX.tar.gz should be a gzip-compressed tar archive.

## Compose CE/EE Separation

### 4. Verify CE deployment excludes mirrors
Stop the stack and restart with CE-only config:
```bash
docker compose -f compose.server.yaml -f compose.ee.yaml down
docker compose -f compose.server.yaml up -d
```

Check services:
```bash
docker compose -f compose.server.yaml ps
```

Expected: `pypi` and `mirror` services should NOT be listed. Only agent, registry, caddy (if any other services), database should be present.

### 5. Verify EE deployment includes mirrors
```bash
docker compose -f compose.server.yaml -f compose.ee.yaml up -d
docker compose -f compose.server.yaml -f compose.ee.yaml ps | grep -E "pypi|mirror"
```

Expected: `pypi` and `mirror` services show as "Up".

Test Caddy routing:
```bash
# Test APT path
curl http://localhost:8081/apt/Packages.gz -I

# Test apk path
curl http://localhost:8081/apk/v3.20/main/APKINDEX.tar.gz -I

# Test PyPI path
curl http://localhost:8081/simple/ -I
```

Expected: All return HTTP 200 or 404 (depends on whether files exist). No 502/503 Bad Gateway errors.

## Foundry Integration

### 6. Build a Debian image with APT packages
Via Dashboard (Foundry > Templates):
1. Create a new blueprint with:
   - Base OS: `debian:12-slim`
   - Name: `test-debian-mirror`
   - Approved packages: curl (APT)
2. Click Build
3. Check build log for `sources.list` injection

Expected: Build succeeds, Dockerfile contains `COPY sources.list /etc/apt/sources.list` and `apt-get install --no-install-recommends curl`.

Verify image was built:
```bash
docker images | grep test-debian-mirror
```

Expected: Image is listed and tagged correctly.

### 7. Build an Alpine image with apk packages
Via Dashboard:
1. Create a new blueprint with:
   - Base OS: `alpine:3.20`
   - Name: `test-alpine-mirror`
   - Approved packages: curl (APK)
2. Click Build
3. Check build log for `repositories` injection and `--allow-untrusted`

Expected: Build succeeds, Dockerfile contains:
- `COPY repositories /etc/apk/repositories`
- `apk add --no-cache --allow-untrusted curl`

Verify image was built:
```bash
docker images | grep test-alpine-mirror
```

Expected: Image is listed and tagged correctly.

## Dashboard Health Status

### 8. Test MirrorHealthBanner visibility
1. Open the Admin dashboard (`/admin`)
2. Verify the amber "Mirror services not running" banner is NOT shown (mirrors are running)
3. Stop mirrors:
   ```bash
   docker compose -f compose.server.yaml -f compose.ee.yaml stop pypi mirror
   ```
4. Wait ~10s for health check to detect unreachable mirrors
5. Refresh Admin page
6. Verify amber banner appears with:
   - Title: "Mirror services not running"
   - Compose command: `docker compose -f compose.server.yaml -f compose.ee.yaml up -d`
   - Dismiss button (X) works
7. Restart mirrors:
   ```bash
   docker compose -f compose.server.yaml -f compose.ee.yaml start pypi mirror
   ```
8. Refresh page
9. Verify banner disappears

Expected: Banner shows only when mirrors are unreachable, disappears when mirrors are healthy.

### 9. Test mirror health check endpoint
```bash
# When mirrors are healthy
curl http://localhost:8001/api/system/health | jq '.mirrors_available'
# Expected: true

# Stop mirrors
docker compose -f compose.server.yaml -f compose.ee.yaml stop pypi mirror

# Wait ~65s (one health check cycle)
sleep 65

# Check again
curl http://localhost:8001/api/system/health | jq '.mirrors_available'
# Expected: false
```

Expected: `mirrors_available` field toggles based on mirror service health.
