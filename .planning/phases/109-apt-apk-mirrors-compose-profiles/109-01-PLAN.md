<task type="auto">
  <name>Task 2: Implement _mirror_apk() and get_apk_repos_content() in mirror_service.py</name>
  <files>puppeteer/agent_service/services/mirror_service.py</files>
  <action>
    In mirror_service.py, add two new methods:

    1. `async def _mirror_apk(db: AsyncSession, ingredient: ApprovedIngredient)`:
       - Parses ingredient.version_constraint to form pkg_spec (e.g., "package=1.0.0")
       - Determines Alpine version: if ingredient has a base_os tag (e.g., "alpine:3.20"), extract "v3.20"; else use DEFAULT_ALPINE_VERSION env var (default "v3.20")
       - Creates mirror-data/apk/v{version}/main/ directory
       - Runs `docker run --rm -v {apk_dir}:/mirror alpine:3.20 sh -c "apk fetch -o /mirror {pkg_spec}"` using asyncio.to_thread
       - On success:
         - Calls `_regenerate_apk_index(apk_dir)` helper (see below)
         - Sets ingredient.mirror_status = "MIRRORED"
         - Sets ingredient.mirror_log with success message
       - On failure:
         - Sets ingredient.mirror_status = "FAILED"
         - Sets ingredient.mirror_log to stderr
       - Commits to database

    2. `def get_apk_repos_content(base_os: str = None) -> str`:
       - A **synchronous** static method (not async) that generates /etc/apk/repositories file content
       - Parses base_os (e.g., "alpine:3.20" → "v3.20"; "alpine:latest" → fallback to DEFAULT_ALPINE_VERSION)
       - Gets APK_MIRROR_URL from environment (default "http://mirror/apk")
       - Returns multi-line content: `{APK_MIRROR_URL}/v{ver}/main\n{APK_MIRROR_URL}/v{ver}/community\n`

    3. `async def _regenerate_apk_index(apk_dir: str)`:
       - Runs `apk index -o APKINDEX.tar.gz -d {apk_dir}` using asyncio.to_thread with cwd=apk_dir
       - Note: `-d` flag tells apk index to output to /apk_dir, not current directory
       - Handles errors gracefully

    Pattern: Follow _mirror_apt() pattern above. Alpine container should be "alpine:3.20" or match the version being downloaded.

    Timeout: 120 seconds.
  </action>

  </verify>
