    ```yaml
    version: "3.8"

    services:
      agent:
        # Override the volumes block — compose merges this with compose.server.yaml,
        # replacing the entire volumes list (not appending).
        # Important: Keep this in sync with compose.server.yaml agent.volumes
        volumes:
          - certs-volume:/app/global_certs:ro

          - MIRROR_DATA_PATH=/app/mirror_data

      pypi:
        image: pypiserver/pypiserver:latest
        restart: always
        command: -P . -a . /data/packages
        volumes:
          - mirror-data:/data/packages
        ports:
          - "8080:8080"

      mirror:
        image: caddy:latest
        restart: always
        volumes:
          - ./mirror/Caddyfile:/etc/caddy/Caddyfile
          - mirror-data:/data
        ports:
          - "8081:80"

    volumes:
      mirror-data:
    ```

  <action>
    Edit puppeteer/mirror/Caddyfile to use `handle` directives for multi-path routing. Replace or update the file to:

    ```caddy
    :80 {
        # APT (Debian) packages at /apt/
        handle /apt/* {
            root /data/apt
            file_server browse
        }

        # Alpine packages at /apk/
        handle /apk/* {
            root /data/apk
            file_server browse
        }

        # PyPI packages at /simple/ (existing)
        handle /simple/* {
            root /data/pypi
            file_server browse
        }

        # Default 404 for unmapped paths
        handle {
            respond 404
        }
    }
    ```

    Notes:
    - Each `handle` block matches a path pattern and sets a root directory
    - `/apt/*` routes to /data/apt (where Packages.gz lives)
    - `/apk/*` routes to /data/apk (where versioned APKINDEX.tar.gz files live)
    - `/simple/*` routes to /data/pypi (existing PyPI mirror)
    - `file_server browse` enables directory listing (optional, convenient for debugging)
    - Catch-all `handle` at the end returns 404 for unmapped paths

    Verify the existing Caddyfile structure — if it has a different layout, preserve its existing functionality and add the new /apt/ and /apk/ blocks.
