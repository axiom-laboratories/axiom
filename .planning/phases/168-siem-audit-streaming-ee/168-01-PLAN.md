# Phase 168 Plan 01 — SIEM Service Core Implementation

## Goal
Implement the core SIEMService class with asyncio.Queue-based batching, CEF formatting with field masking, and exponential backoff retry logic for webhook and syslog delivery.

## Context
This plan builds the heart of the SIEM audit streaming feature (D-01 through D-15). It creates the stateless, fire-and-forget service that auditor events queue into, formats as CEF, masks sensitive fields at transmission time (never modifying the audit_log DB), and handles failed deliveries with intelligent retry. The service follows the VaultService pattern (Phase 167) for non-blocking startup and graceful degradation.

## Threat Model

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-168-01 | Information Disclosure | Sensitive field masking | mitigate | Mask password, secret, token, api_key, *_key, *_secret at format time BEFORE transmission (D-11, D-12); never modify audit_log DB rows |
| T-168-02 | Denial of Service | Queue overflow | mitigate | Drop oldest events on overflow (FIFO); log structured warning with dropped count exposed in status endpoint (D-10) |
| T-168-03 | Tampering | CEF field injection | mitigate | Use syslogcef library which escapes special chars (\|, \=) in extension dict; avoid DIY formatting |
| T-168-04 | Elevation of Privilege | Status endpoint leaks errors | mitigate | Status endpoint (later plan) requires permission gate; error_detail only shown to admins |

### Task 4: Create SIEMService class in puppeteer/ee/services/siem_service.py

**Files:** `puppeteer/ee/services/siem_service.py` (NEW)

**Action:** CREATE new file with complete SIEMService implementation. Use RESEARCH.md skeleton and PATTERNS.md structure exactly.

Key implementation sections (order matters):

1. **Imports** — asyncio, json, logging, datetime, typing, httpx, AsyncSession, select, APScheduler, uuid, socket, logging.handlers, timedelta
2. **Module-level constants** — SENSITIVE_KEYS set, logger setup, _siem_service singleton variable
3. **SIEMService class definition**
   - `__init__(self, config: Optional[SIEMConfig], db: AsyncSession, scheduler: AsyncIOScheduler)` — initialize queue, status fields, scheduler reference
   - `async def startup(self)` — non-blocking startup (D-02): test connection if enabled, register flush job with APScheduler, set status
   - `def enqueue(self, event: dict) -> None` — sync, fire-and-forget (D-03, D-09): explicit drop-oldest overflow — `put_nowait()` raises `QueueFull`; catch it, call `get_nowait()` to drop the oldest, then `put_nowait()` again, increment `dropped_events`:
     ```python
     def enqueue(self, event: dict) -> None:
         try:
             self.queue.put_nowait(event)
         except asyncio.QueueFull:
             try:
                 self.queue.get_nowait()
                 self.queue.put_nowait(event)
                 self.dropped_events += 1
                 logger.warning("siem_queue_overflow", extra={"dropped_total": self.dropped_events})
             except Exception:
                 pass
     ```
   - `async def _flush_periodically(self)` — APScheduler job: read up to 100 events from queue, call flush_batch()
   - `async def flush_batch(self, batch: list[dict])` — format to CEF, attempt delivery with retry (D-13, D-14)
   - `def _format_cef(self, event: dict) -> str` — apply masking, build CEF header + extensions (D-11, D-12)
   - `def _map_severity(self, action: str) -> int` — map audit action to CEF severity (1-10)
   - `async def _deliver(self, payload: str)` — webhook (httpx POST) or syslog (SysLogHandler via asyncio.to_thread)
   - `async def _test_connection(self)` — validate destination reachability
   - `async def status(self)` → Literal["healthy", "degraded", "disabled"]
   - `def status_detail(self) -> dict` — return detailed status for admin UI
4. **Module-level singleton accessors** — `get_siem_service()` and `set_active(instance)`

**Implementation notes:**
- Queue maxsize=10,000 (D-08)
- Flush triggers: 100 events OR 5s interval (D-09)
- Masking keywords: password, secret, token, api_key, secret_id, role_id, encryption_key, access_token, refresh_token, *_key, *_secret (case-insensitive) (D-11 + review addendum: access_token/refresh_token added as they appear in agent_service audit events)
- Retry backoff: 5s → 10s → 20s, max 3 attempts (D-13)
- On 3 consecutive failures, transition to DEGRADED (D-14)
- Successful delivery resets consecutive_failures counter to 0
- APScheduler job IDs: `__siem_flush__` (periodic), `siem_retry_{uuid}_{attempt}` (retries) with `replace_existing=True` for periodic job
- For syslog, parse `destination` as "host:port" or just "host" (use port from config.syslog_port if not in destination)
- Webhook: Content-Type: application/cef, POST body is CEF text, timeout 10s
- Syslog: use logging.handlers.SysLogHandler with socktype SOCK_DGRAM (UDP) or SOCK_STREAM (TCP)
- Run SysLogHandler in asyncio.to_thread() to avoid blocking event loop
