# Phase 168 Plan 03 — Audit Integration

## Goal
Hook the SIEMService into the existing `audit()` function so that every audit event is enqueued for SIEM transmission. Implement env var bootstrap so admins can configure SIEM at startup without UI access. Ensure the integration is non-blocking and CE-compatible.

## Context
This plan connects SIEMService (Plan 01) to the audit stream. Every call to `deps.py:audit()` will now fire-and-forget an event to SIEMService.enqueue(), which pushes it into the in-memory queue for batch processing and transmission. The hook must be synchronous (never await), never fail, and never block the caller. Plan 03 also implements environment variable bootstrap so admins can set SIEM_BACKEND, SIEM_DESTINATION, SIEM_ENABLED at startup to seed a default configuration.

## Threat Model

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-168-09 | Denial of Service | audit() blocked by SIEM failure | mitigate | enqueue() is synchronous, fire-and-forget; wrapped in try/except to never propagate errors (D-03, D-09) |
| T-168-10 | Information Disclosure | Audit log inadvertently masked at storage | mitigate | Masking is only at transmission time (D-12); audit_log DB rows are always unmasked raw data; verified by schema inspection |
| T-168-11 | Tampering | SIEM queue overflow causes silent data loss | mitigate | Drop oldest on overflow (D-10); log structured warning with count; alert via status endpoint |

def audit(
    db: Session,
    user: str,
    action: str,
    resource_id: str,
    detail: Dict[str, Any]
) -> None:
    """
    Schedule an audit log entry (non-blocking).
    
    Logs to local audit_log table and enqueues to SIEM service (if enabled).
    Never blocks the caller; never propagates errors.
    Per D-03, D-09.
    """
    loop = asyncio.get_event_loop()
    loop.create_task(_insert(db, user, action, resource_id, detail))
    
    # Fire-and-forget SIEM enqueue (Phase 168 — D-03, D-09)

        from ee.services.siem_service import get_siem_service
        
        siem = get_siem_service()
        if siem:
            event = {
                "username": user,
                "action": action,
                "resource_id": resource_id,
                "detail": detail,
                "timestamp": datetime.utcnow().isoformat(),
            }
            siem.enqueue(event)
    except Exception:
        # Never block audit path due to SIEM errors (D-03)
        pass

### Task 2: Bootstrap SIEMConfig from environment variables in main.py

**Files:** `puppeteer/agent_service/main.py`

**Action:** MODIFY main.py startup (in the lifespan, after DB init) to check for SIEM_* env vars and create a default SIEMConfig if none exists. This allows admins to configure SIEM without UI access on first startup.

Add this block in the lifespan startup, before SIEM service initialization (around line ~90, after vault config init):

```python
# Bootstrap SIEM config from env vars if not present (Phase 168 — D-18)
try:
    from agent_service.db import SIEMConfig
    from sqlalchemy import select
    import os
    
    existing_result = await db.execute(select(SIEMConfig).limit(1))
    existing_config = existing_result.scalars().first()
    
    if not existing_config:
        siem_backend = os.getenv("SIEM_BACKEND")
        siem_destination = os.getenv("SIEM_DESTINATION")
        siem_enabled = os.getenv("SIEM_ENABLED", "false").lower() in ("true", "1")
        
        if siem_backend and siem_destination:
            # Create bootstrap config from env vars
            siem_config = SIEMConfig(
                id=str(uuid.uuid4()),
                backend=siem_backend,
                destination=siem_destination,
                syslog_port=int(os.getenv("SIEM_SYSLOG_PORT", "514")),
                syslog_protocol=os.getenv("SIEM_SYSLOG_PROTOCOL", "UDP"),
                cef_device_vendor=os.getenv("SIEM_CEF_DEVICE_VENDOR", "Axiom"),
                cef_device_product=os.getenv("SIEM_CEF_DEVICE_PRODUCT", "MasterOfPuppets"),
                enabled=siem_enabled,
            )
            db.add(siem_config)
            await db.commit()
            logger.info(f"Bootstrapped SIEM config from env vars: backend={siem_backend}")
except Exception as e:
    logger.warning(f"Failed to bootstrap SIEM config from env vars: {e}")
```

**Environment Variables Reference:**

| Variable | Required | Default | Purpose |
|----------|----------|---------|---------|
| SIEM_BACKEND | Yes (if SIEM_DESTINATION set) | None | "webhook" or "syslog" |
| SIEM_DESTINATION | Yes (if SIEM_BACKEND set) | None | webhook URL or syslog host |
| SIEM_SYSLOG_PORT | No | 514 | Syslog port for syslog backend |
| SIEM_SYSLOG_PROTOCOL | No | UDP | "UDP" or "TCP" |
| SIEM_CEF_DEVICE_VENDOR | No | Axiom | CEF header vendor field |
| SIEM_CEF_DEVICE_PRODUCT | No | MasterOfPuppets | CEF header product field |
| SIEM_ENABLED | No | false | "true"/"1" to enable on bootstrap |

### Task 3: Verify audit integration end-to-end

**Files:** (read-only verification)

**Action:** Write a simple integration test that verifies:

1. audit() function successfully calls SIEMService.enqueue() without blocking
2. Event payload has correct structure (username, action, resource_id, detail, timestamp)
3. SIEM enqueue never raises an exception that escapes audit()
4. CE mode (get_siem_service() returns None) doesn't break audit()

```python
import asyncio
from unittest.mock import Mock, AsyncMock, patch
from datetime import datetime
from agent_service.deps import audit
from agent_service.db import Base, engine
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

# Create in-memory test DB
test_engine = create_engine("sqlite:///:memory:")
Base.metadata.create_all(test_engine)
TestSession = sessionmaker(bind=test_engine)

def test_audit_integration():
    # Test 1: audit() works with SIEM service active
    with patch('agent_service.deps.get_siem_service') as mock_get_siem:
        mock_siem = Mock()
        mock_siem.enqueue = Mock()
        mock_get_siem.return_value = mock_siem
        
        db = TestSession()
        audit(db, "alice", "job_execute", "job_123", {"result": "success"})
        
        # Verify enqueue was called with correct event structure
        assert mock_siem.enqueue.called, "enqueue() should be called"
        event = mock_siem.enqueue.call_args[0][0]
        assert event["username"] == "alice", f"Expected username='alice', got {event['username']}"
        assert event["action"] == "job_execute", f"Expected action='job_execute', got {event['action']}"
        assert event["resource_id"] == "job_123", f"Expected resource_id='job_123', got {event['resource_id']}"
        assert "timestamp" in event, "Event should have timestamp"
        print("✓ Test 1 passed: audit() integrates with SIEM service")
    
    # Test 2: audit() works in CE mode (SIEM service is None)
    with patch('agent_service.deps.get_siem_service') as mock_get_siem:
        mock_get_siem.return_value = None
        
        db = TestSession()
        # Should not raise
        audit(db, "bob", "user_create", "user_456", {"username": "newuser"})
        print("✓ Test 2 passed: audit() works in CE mode (SIEM=None)")
    
    # Test 3: audit() never raises even if SIEM.enqueue() fails
    with patch('agent_service.deps.get_siem_service') as mock_get_siem:
        mock_siem = Mock()
        mock_siem.enqueue = Mock(side_effect=Exception("Queue overflow"))
        mock_get_siem.return_value = mock_siem
        
        db = TestSession()
        # Should not raise despite enqueue() failure
        audit(db, "charlie", "vault_access", "secret_123", {"field": "accessed"})
        print("✓ Test 3 passed: audit() never propagates SIEM errors")

if __name__ == "__main__":
    test_audit_integration()
    print("\n✓ All audit integration tests passed")
```
