# Phase 168 — UI Design Contract: SIEM Audit Streaming (EE)

> Visual and interaction contract for the SIEM configuration and status dashboard components. Covers admin UI forms, status badges, and system health integration. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS + custom components) |
| Preset | not applicable |
| Component library | Radix UI (via custom components) |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | background (hsl(var(--background))) | Page background, card backgrounds |
| Secondary (30%) | card (hsl(var(--card))) | Form fields, panels, dividers |
| Accent (10%) | primary (hsl(var(--primary))) | Buttons, interactive elements, focus states |
| Destructive | destructive (hsl(var(--destructive))) | Delete buttons, error states only |
| Status — Healthy | emerald-500 (var(--chart-1)) | Health badge, success indicators |
| Status — Degraded | amber-500 (var(--chart-2)) | Warning badge, degraded indicators |
| Status — Disabled | muted (hsl(var(--muted))) | Disabled state badge |

**Source:** Existing Admin.tsx color scheme (Phase 127, 167) — HSL CSS variables (--background, --card, --primary, etc.) defined in globals.css; status colors from chart palette used in existing health badges (Vault status, System Health).

**Accent reserved for:**
- Primary action button (e.g., "Save Config", "Test Connection") — primary-foreground text on primary background
- Form field focus state (ring color)
- Tab indicator (active tab border/background)
- Toggle switch active state

**NOT accent:** Disabled buttons, secondary buttons, labels, borders.

## Interaction States

### Form Field States

| State | Style | Behavior |
|-------|-------|----------|
| **Default (empty)** | Border: border-input; Background: input | Placeholder text visible; cursor inactive |
| **Focused** | Border: ring (2px outline in primary color); Box-shadow: ring shadow | Text cursor active |
| **Filled** | Border: border-input; Background: input; Text: foreground | Value visible; no placeholder |
| **Error (validation)** | Border: destructive; Background: input; Text: foreground | Error message below field in red (destructive-foreground) |
| **Disabled** | Border: border; Background: muted; Text: muted-foreground; Cursor: not-allowed | Non-interactive; grayed out |

### Button States

| Button | State | Style | Behavior |
|--------|-------|-------|----------|
| **Test Connection** | Default | bg-secondary text-secondary-foreground; border | Hover: opacity increase; Click: disabled state |
| **Test Connection** | Loading | Disabled; Loader2 icon spinning | Tooltip: "Testing..." |
| **Test Connection** | Success | Temporarily fade to green (emerald-500/20 badge + CheckCircle2 icon) for 2s, then revert | Toast notification: "Connection successful" |
| **Test Connection** | Error | Red badge (destructive-foreground) + AlertCircle icon for 5s | Toast notification: "Connection failed: {error}" |
| **Save Config** | Default | bg-primary text-primary-foreground | Hover: opacity increase; Click: disabled state |
| **Save Config** | Disabled | bg-primary/50 text-primary-foreground/50; Cursor: not-allowed | Shown when no changes made or validation errors |
| **Save Config** | Loading | Disabled; Loader2 icon spinning | Tooltip: "Saving..." |
| **Save Config** | Success | Form persists; toast: "SIEM configuration saved" | Status badge updates to reflect new state |

### Select Dropdowns

| Dropdown | Options | Default |
|----------|---------|---------|
| **Backend** | "Webhook (HTTP POST)", "Syslog (UDP/TCP)" | "Webhook" |
| **Protocol** | "UDP (fire-and-forget)", "TCP (connection-oriented)" | "UDP" |
| **Syslog Port** | User-editable (1–65535) | "514" (standard syslog) |

**Conditional visibility:**
- Port input: Visible only when Backend = "Syslog"
- Protocol selector: Visible only when Backend = "Syslog"

### Toggle Switch

| Toggle | Label | Default | Behavior |
|--------|-------|---------|----------|
| **Enable SIEM Streaming** | Left of toggle | Off (disabled) | Click to toggle; disables form fields when off |

**Disabled form state (when toggle off):**
- All input fields (Address, Port, Vendor, Product) become disabled
- Backend and Protocol selectors become disabled
- Test Connection and Save buttons become disabled
- Status badge remains visible (shows "Disabled")

## Responsive Design

**Breakpoints (Tailwind default):**
- Mobile (< 640px): Single-column form; buttons stack vertically
- Tablet (640px–1024px): Two-column form (Backend + Address on row 1; Port + Protocol on row 2)
- Desktop (> 1024px): Form fields in logical pairs or single column with max-width constraint

**Touchscreen considerations:**
- Button minimum height: 44px (touch target)
- Form field padding: md (16px) minimum
- Select dropdown: Full-width on mobile, auto-width on desktop

## Error Handling & Validation

| Error Scenario | User Feedback | Action Path |
|----------------|---------------|-------------|
| **Missing address** | Inline error text below field: "Address is required"; field border turns destructive | User must fill address, retry Test Connection |
| **Invalid webhook URL** | Inline error: "Must be valid HTTPS URL"; border destructive | User corrects URL format (https://...) |
| **Invalid hostname/port** | Inline error: "Hostname and port must be valid"; border destructive | User corrects format (hostname:port) |
| **Port out of range** | Inline error: "Port must be 1–65535"; border destructive | User enters valid port number |
| **Test connection timeout** | Toast error: "Test connection failed: timeout (exceeded 10s)"; status badge → Degraded | User checks network connectivity, retries |
| **Test connection refused** | Toast error: "Test connection failed: connection refused"; status badge → Degraded | User verifies SIEM destination is running |
| **Network unreachable** | Toast error: "Test connection failed: network unreachable"; status badge → Degraded | User checks firewall rules, DNS |
| **Config save conflict** | Toast error: "Failed to save: configuration changed by another admin. Refresh and try again."; form reverts | User refreshes page, reapplies changes |

**Error boundary:** Wrap entire SIEM tab in React error boundary to catch unexpected exceptions; show fallback UI: "An error occurred loading SIEM settings. Please refresh."

## Copy Patterns

All copy follows existing Admin.tsx voice/tone:

- **Headings:** Title case, no period. Example: "SIEM Audit Streaming"
- **Labels:** Sentence case, no period. Example: "Backend type", "Device vendor"
- **Help text:** Sentence case, ends with period. Example: "Webhook uses HTTP POST with CEF formatting."
- **Error messages:** Sentence case, ends with period, includes actionable guidance. Example: "Address must be a valid HTTPS URL. Check your syslog server is reachable."
- **Button labels:** Title case, imperative verb. Example: "Test Connection", "Save Configuration"
- **Status labels:** Title case. Example: "Healthy", "Degraded", "Disabled"

## Accessibility (WCAG 2.1 AA)

| Requirement | Implementation |
|-------------|-----------------|
| **Contrast ratio** | All text ≥ 4.5:1 against background (Tailwind default colors meet AA) |
| **Form labels** | All inputs have associated `<Label>` component with `htmlFor` linking to input `id` |
| **Error messages** | Associated with form field via `aria-describedby` |
| **Focus management** | Tab order follows DOM; focus visible (browser default or custom ring style) |
| **ARIA roles** | Badge: role="status" for status indicators; AlertDialog: role="alertdialog" for confirmation |
| **Keyboard navigation** | All buttons keyboard-accessible; Select dropdowns keyboard-navigable |
| **Loading state** | Disabled button + spinner icon communicates async state; no silent hangs |
| **Status badge** | `aria-live="polite"` so screen readers announce status changes |
