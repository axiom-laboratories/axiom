# Plan 12-07: Automated Security Scanning

**Objective**: Integrate `pip-audit` into the Smelter Registry to automatically identify and flag vulnerable packages in the approved catalog.

## Context
A vetted catalog is only effective if it's regularly audited for new CVEs. This plan implements the automated scanning logic and exposes it to the admin.

## Tasks
1. **Update `SmelterService.scan_vulnerabilities` in `puppeteer/agent_service/services/smelter_service.py`**:
    - Fetch all approved ingredients from the DB.
    - Format ingredients into a temporary requirements list.
    - Execute `.venv/bin/pip-audit --format json` as a subprocess.
    - Parse the JSON results.
    - Update each ingredient's `is_vulnerable` and `vulnerability_report` fields.
2. **Expose Scan API in `puppeteer/agent_service/main.py`**:
    - Add `POST /api/smelter/scan`: Triggers the vulnerability scan.
3. **Update `Admin.tsx` Dashboard**:
    - Add a "Scan for Vulnerabilities" button to the Smelter Registry tab.
    - Show a loading state/toast during the scan.
4. **Verification**:
    - Add a known vulnerable package (e.g., `requests==2.20.0`) to the catalog.
    - Run the scan and verify it is flagged as "Vulnerable" in the UI.

## Success Criteria
- Scan correctly identifies known vulnerable packages.
- Catalog records are updated with security metadata.
- Admin can trigger the scan from the UI.
