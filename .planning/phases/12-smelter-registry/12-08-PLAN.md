# Plan 12-08: Phase Verification

**Objective**: Perform final end-to-end verification of the Smelter Registry features (SMLT-01..05).

## Context
All technical components are implemented. This plan ensures they work together as expected before concluding the phase.

## Tasks
1. **Verify SMLT-01 (Catalog CRUD)**:
    - Add an ingredient via API/UI.
    - Delete an ingredient via API/UI.
    - List ingredients and verify fields.
2. **Verify SMLT-02 (Security Scanning)**:
    - Add `requests==2.20.0` (known vulnerable).
    - Trigger a scan.
    - Verify `is_vulnerable` is True and report is populated.
3. **Verify SMLT-03 (STRICT Enforcement)**:
    - Set enforcement to `STRICT`.
    - Attempt a build with a blueprint containing an unapproved package.
    - Verify 403 error.
4. **Verify SMLT-04 (WARNING Enforcement)**:
    - Set enforcement to `WARNING`.
    - Attempt the same build.
    - Verify build proceeds.
5. **Verify SMLT-05 (Dashboard Badging)**:
    - Check the resulting template in the dashboard.
    - Verify the "Non-Compliant" badge is visible.

## Success Criteria
- All 5 manual/automated verification steps pass.
- Documentation is updated if necessary.

## Output
Create `.planning/phases/12-smelter-registry/12-08-SUMMARY.md` after completion.
Update `ROADMAP.md` if Milestone progress is significant.
Update `STATE.md` to mark Phase 12 as complete.
