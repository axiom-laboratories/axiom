# Plan 12-03: SmelterService Core Logic

**Objective**: Implement the backend service for managing the vetted ingredient catalog and validating blueprints.

## Context
Phase 12 requires a dedicated service to handle the "Smelter" logic — managing approved packages and ensuring blueprints only use these approved ingredients.

## Tasks
1. **Create `puppeteer/agent_service/services/smelter_service.py`**:
    - Implement `SmelterService` class.
    - Implement `add_ingredient(db, ingredient_in)`: Creates a new approved ingredient.
    - Implement `list_ingredients(db)`: Returns all approved ingredients.
    - Implement `delete_ingredient(db, ingredient_id)`: Removes an ingredient.
    - Implement `scan_vulnerabilities(db)`: Placeholder for `pip-audit` integration.
    - Implement `validate_blueprint(db, rt_def, os_family)`: Checks if all packages in a blueprint definition are approved for the target OS family.
2. **Update `puppeteer/tests/test_smelter.py`**:
    - Update test stubs to import and call the real service instead of just checking for its existence.
3. **Verification**:
    - Run `python3 -m pytest puppeteer/tests/test_smelter.py::test_smelter_service_exists_stub` and `test_vulnerability_scan_integration_stub`.

## Success Criteria
- `SmelterService` is implemented with all required methods.
- Basic CRUD for ingredients works.
- Blueprint validation logic correctly identifies approved vs. unapproved packages.
- Wave 1 tests for service existence pass.
