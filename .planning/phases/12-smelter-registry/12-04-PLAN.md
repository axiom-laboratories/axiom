# Plan 12-04: Foundry Build Enforcement

**Objective**: Integrate Smelter Registry validation into the Foundry build pipeline with STRICT/WARNING enforcement.

## Context
Foundry must now consult the Smelter Registry before building an image. Based on the system configuration, it should either block unapproved builds or mark them as non-compliant.

## Tasks
1. **Update `puppeteer/agent_service/services/foundry_service.py`**:
    - Import `SmelterService` and `Config`.
    - At the start of `build_template`, fetch `smelter_enforcement_mode` from the `Config` table (default: "WARNING").
    - Call `SmelterService.validate_blueprint(db, rt_def, os_family)`.
    - If unapproved packages exist:
        - In `STRICT` mode: Raise `HTTPException(status_code=403, detail=f"Unapproved packages: {unapproved}")`.
        - In `WARNING` mode: Proceed but set `tmpl.is_compliant = False` on the template record.
    - If no unapproved packages exist, ensure `tmpl.is_compliant = True`.
    - Commit the template compliance status.
2. **Verification**:
    - Run `python3 -m pytest puppeteer/tests/test_smelter.py::test_foundry_enforcement_strict_stub` and confirm it now PASSES.
    - Add a new functional test `test_foundry_enforcement_functional` to verify the logic with a mock DB and RT definition.

## Success Criteria
- Builds are blocked when `STRICT` mode is active and unapproved packages are present.
- Builds are allowed but marked non-compliant when `WARNING` mode is active and unapproved packages are present.
- Integration test passes.
