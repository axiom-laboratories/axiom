# Plan 12-05: Smelter Catalog & Config UI

**Objective**: Implement the administrative interface for the Smelter Registry and expose required API endpoints.

## Context
Admins need a way to manage approved ingredients and toggle enforcement modes (STRICT vs WARNING) via the dashboard.

## Tasks
1. **Expose Smelter API in `puppeteer/agent_service/main.py`**:
    - `GET /api/smelter/ingredients`: List all approved ingredients.
    - `POST /api/smelter/ingredients`: Add a new approved ingredient.
    - `DELETE /api/smelter/ingredients/{id}`: Remove an ingredient.
    - `GET /api/smelter/config`: Get current `smelter_enforcement_mode`.
    - `PATCH /api/smelter/config`: Update `smelter_enforcement_mode`.
2. **Update `puppeteer/dashboard/src/views/Admin.tsx`**:
    - Add `SmelterRegistryManager` sub-component.
    - Add "Smelter Registry" tab to the main `Admin` view.
    - Implement CRUD UI for ingredients (Table + Dialog).
    - Implement enforcement mode toggle (STRICT/WARNING).
3. **Verification**:
    - Manually verify that adding an ingredient through the UI reflects in the DB.
    - Manually verify that changing enforcement mode through the UI updates the `config` table.

## Success Criteria
- Smelter API endpoints are functional and authenticated.
- Admin can manage the ingredient catalog from the dashboard.
- Enforcement mode can be toggled without server restart.
