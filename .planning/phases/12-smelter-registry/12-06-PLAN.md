# Plan 12-06: Dashboard Template Badging

**Objective**: Surface the compliance status of Puppet templates on the Foundry dashboard.

## Context
Templates built with unapproved ingredients in WARNING mode are marked as non-compliant in the database. Users need to see this status clearly on the dashboard.

## Tasks
1. **Update `Template` interface in `puppeteer/dashboard/src/views/Templates.tsx`**:
    - Add `is_compliant: boolean;`.
2. **Update `TemplateCard` component**:
    - Render a "Non-Compliant" badge if `template.is_compliant` is false.
    - Use `ShieldAlert` icon and Amber/Red styling.
    - Place the badge alongside existing status indicators (Stale, Canonical ID).
3. **Verification**:
    - Run `cd puppeteer/dashboard && npx tsc` to ensure no type errors.
    - Manually verify the UI appearance (simulated if necessary).

## Success Criteria
- Templates view displays a clear visual indicator for non-compliant images.
- The UI accurately reflects the `is_compliant` field from the backend.

## Output
Create `.planning/phases/12-smelter-registry/12-06-SUMMARY.md` after completion.
Update `STATE.md` with progress.
