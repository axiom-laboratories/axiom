# Plan 12-01: Wave 0 — Smelter Registry Test Stubs

**Objective**: Create the automated test harness for Phase 12 (SMLT-01..05) using the Nyquist source-inspection pattern.

## Context
Following the established project pattern, we begin with Wave 0 test stubs that reflect the requirements before any implementation. These tests will use `pytest` and `inspect` to verify the existence and logic of the proposed `SmelterService` and its integration.

## Tasks
1. **Create `puppeteer/tests/test_smelter.py`**:
    - `test_ingredient_catalog_crud_stubs`: SMLT-01 (Verify `SmelterService.add_ingredient`, `list_ingredients`, `delete_ingredient` exist and follow expected signatures).
    - `test_vulnerability_scan_integration_stub`: SMLT-02 (Verify `SmelterService.scan_vulnerabilities` exists and integrates with a mock security scanner).
    - `test_foundry_enforcement_strict_stub`: SMLT-03 (Verify `FoundryService.build_template` calls `SmelterService.validate_blueprint` and raises `HTTPException(403)` if unauthorized in STRICT mode).
    - `test_smelter_enforcement_config_stub`: SMLT-04 (Verify `Config` table can store `smelter_enforcement_mode` with values `STRICT` or `WARNING`).
    - `test_template_compliance_badging_stub`: SMLT-05 (Verify `PuppetTemplate` model has a `is_compliant` boolean field).
2. **Verify Wave 0 Failures**:
    - Run `pytest puppeteer/tests/test_smelter.py` and confirm all 5 tests fail (missing symbols).

## Success Criteria
- Test file exists in `puppeteer/tests/`.
- All 5 tests are collected by `pytest`.
- All 5 tests fail with `AttributeError` or similar due to missing implementation.
