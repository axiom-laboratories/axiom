### Existing Licence Patterns

From puppeteer/agent_service/main.py:
```python
from .services.licence_service import require_ee

# Usage in route:
async def route(current_user = Depends(require_ee())):
    # Route body
```

The `require_ee()` function returns a dependency that:
- Checks if current user has EE licence
- Raises HTTPException(status_code=403) if licence invalid/missing
- Returns current_user if valid

| Unauthenticated → Vault routes | Public routes (health, etc.) don't expose Vault details |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-167-19 | A (Authorization) | Vault admin routes | mitigate | All routes have require_ee() dependency; CE users get 403 |
| T-167-20 | S (Spoofing) | Licence verification | mitigate | Licence service check happens at route time, not global; fresh check for each request |
| T-167-21 | I (Information Disclosure) | Health endpoint | mitigate | Vault status visible only if configured; CE users see "disabled" or None (no credentials leaked) |
| T-167-22 | R (Repudiation) | CE access attempt | mitigate | Audit log records all 403 errors (already logged by licence service) |
