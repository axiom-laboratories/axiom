Write `docs/docs/workflows/concepts.md` with sections:

1. **Header & intro** (5 lines):
   - "# Workflow Concepts"
   - Brief intro: "Learn about the building blocks of workflows: steps, edges, and gates"

2. **Data Model** (15-20 lines):
   - "## Data Model"
   - Explain: "A Workflow is a directed acyclic graph (DAG) of Steps connected by Edges. Each Step is either a SCRIPT (executes a ScheduledJob) or a GATE (controls flow based on conditions or topology)."
   - Show one concrete example: "A 3-step pipeline: Extract Data → [IF quality_check=PASS: Transform | IF quality_check=FAIL: Rollback] → Load"
   - Link to developer-guide.md for the full 7-table ERD

3. **Step Types** (40-50 lines):
   - "## Step Node Types"
   - For each of the 6 types (SCRIPT, IF_GATE, AND_JOIN, OR_GATE, PARALLEL, SIGNAL_WAIT):
     - **Name & short description** (1-2 lines)
     - **When to use this** (2-3 lines of rationale, per CONTEXT.md)
     - **Example** (1-2 lines)
   
   Example structure for each:
   ```markdown
   ### SCRIPT
   Executes a ScheduledJob; the most common step type. When any upstream dependency completes, the job is dispatched.
   
   **When to use:** Any time you need to run a job (data extraction, transformation, cleanup, API call, etc.).
   
   **Example:** Extract data from a database, validate it, load it elsewhere.
   ```

4. **Gate Types** (40-50 lines):
   - "## Gate Node Types"
   - Subsections for each gate type (IF_GATE, AND_JOIN, OR_GATE, PARALLEL, SIGNAL_WAIT) with same structure:
     - Name, description, when to use, example
   
   Key rationales (from RESEARCH.md pitfalls):
   - IF_GATE: "Routes to one of two branches based on conditions from the previous step's output"
   - AND_JOIN: "Merges multiple incoming branches; waits for ALL to complete"
   - OR_GATE: "Merges multiple incoming branches; releases as soon as ANY completes"
   - PARALLEL: "Fans out to multiple independent branches; all execute concurrently"
   - SIGNAL_WAIT: "Pauses execution until an external signal is posted via the Signal mechanism"

5. **Execution Lifecycle** (15-20 lines):
   - "## Execution Lifecycle"
   - High-level state machine: RUNNING → (COMPLETED | PARTIAL | FAILED | CANCELLED)
   - Explain each status:
     - RUNNING: At least one step is pending/assigned/running
     - COMPLETED: All steps succeeded
     - PARTIAL: Some branches failed but failures were isolated by gates
     - FAILED: Critical failure; downstream steps cancelled
     - CANCELLED: User cancelled the run
   - Note: "See Operator Guide for detailed state transitions"

6. **DAG Constraints** (10-15 lines):
   - "## DAG Constraints"
   - "Workflows must be acyclic (no loops)"
   - "Maximum depth: 30 levels (vs. 10 for regular jobs)"
   - "Cycles detected at save time; system rejects invalid DAGs"

Total target: ~100-130 lines

Use Markdown formatting (headers, bullet points, inline code). Include conceptual diagrams if natural (can use mermaid if helpful).
