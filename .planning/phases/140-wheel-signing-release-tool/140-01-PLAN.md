# Key cryptography patterns
- Ed25519 keypair generation: `cryptography.hazmat.primitives.asymmetric.ed25519.Ed25519PrivateKey.generate()`
- PEM serialization: `cryptography.hazmat.primitives.serialization` with `NoEncryption()`
- SHA256 chunked hashing: `hashlib.sha256()` with 64KB chunks
- Base64 encoding: stdlib `base64.b64encode()` (standard, not urlsafe)
- Manifest format (locked): `{"sha256": "<hex>", "signature": "<base64>"}`
- Signing message: UTF-8 encoded hex SHA256 string (not raw bytes)

       - Print to stdout: Python bytes literal of public key (format: b"""...\n...\n""")

       - Decode PEM bytes to string: public_pem.decode()

  <name>Task 2: Implement sign_wheels.py signing and verification script</name>
  <files>
    axiom-licenses/tools/sign_wheels.py
  </files>
  <action>
    Implement sign_wheels.py (release-time wheel signing and verification):

    1. Structure:
       - Shebang: #!/usr/bin/env python3
       - Module docstring: EE wheel manifest signing and verification
       - Imports: argparse, sys, os, json, base64, hashlib, glob, Path, cryptography.hazmat.primitives.serialization, cryptography.hazmat.primitives.asymmetric.ed25519
       - Sections: (a) helper functions, (b) key resolution, (c) signing logic, (d) verification logic, (e) argparse, (f) main

    2. Helper function: hash_wheel(wheel_path: str) → str
       - Compute SHA256 of wheel file in 64KB chunks (matching Phase 137 pattern)
       - Return hexdigest as lowercase string
       - Error handling: catch OSError, print clear error to stderr

    3. Helper function: resolve_key(args, mode: str) → (Ed25519PrivateKey | Ed25519PublicKey)
       - mode="private" for signing, mode="public" for verification
       - Check args.key or os.getenv("AXIOM_WHEEL_SIGNING_KEY")
       - If neither: sys.exit("Error: no signing key provided.\nSet AXIOM_WHEEL_SIGNING_KEY env var or pass --key <path>.")
       - Load file: Path(key_source).read_bytes()
       - If mode="private": serialization.load_pem_private_key(bytes, password=None)
       - If mode="public": Try private first, fall back to public (flexible loading)
       - Error on any load failure: sys.exit(f"Error: failed to load key from {key_source}: {e}")
       - Reference: axiom-licenses/tools/issue_licence.py resolve_key pattern

    4. Function: sign_wheels(wheels_dir: Path, private_key, deploy_name: bool, quiet: bool)
       - Find all .whl files: sorted(wheels_dir.glob("*.whl"))
       - If no wheels: sys.exit(f"Error: no .whl files found in {wheels_dir}")
       - For each wheel:
         a. Compute SHA256: sha256_hex = hash_wheel(str(wheel_path))
         b. Sign the UTF-8 hex string (not raw bytes):
            - message = sha256_hex.encode('utf-8')
            - signature_bytes = private_key.sign(message)
            - signature_b64 = base64.b64encode(signature_bytes).decode('ascii')
         c. Build manifest dict: {"sha256": sha256_hex, "signature": signature_b64}
         d. Write versioned manifest: wheel_path.with_stem(wheel_path.stem + ".manifest").write_text(json.dumps(manifest, indent=2))
            - Alternative: manifest_path = wheels_dir / f"{wheel_path.stem}.manifest.json"
         e. If deploy_name: also write axiom_ee.manifest.json to same dir
         f. If not quiet: print to stderr: f"Signed: {manifest_path.name} (sha256: {sha256_hex[:16]}...)"

    5. Function: verify_manifests(wheels_dir: Path, public_key)
       - Find all .whl files in wheels_dir
       - For each wheel:
         a. Look for corresponding .manifest.json file
         b. Load manifest JSON, extract sha256 and signature
         c. Compute SHA256 of wheel
         d. If computed != manifest: print to stdout: f"FAIL: {manifest_name} — SHA256 mismatch"
         e. Decode signature from base64
         f. Verify Ed25519: public_key.verify(signature_bytes, sha256_hex.encode('utf-8'))
         g. If verify succeeds: print to stdout: f"OK: {manifest_name}"
         h. If verify fails: print to stdout: f"FAIL: {manifest_name} — signature verification failed: {e}"
       - Return True if all passed, False if any failed

    6. Argparse setup:
       - Description: "Sign EE wheel manifests with Ed25519 key"
       - Argument: --wheels-dir (required, help="Directory containing .whl files")
       - Argument: --key (optional, help="Path to private key PEM file (or set AXIOM_WHEEL_SIGNING_KEY)")
       - Argument: --deploy-name (action="store_true", help="Also write axiom_ee.manifest.json (fixed name for /tmp/ deployment)")
       - Argument: --verify (action="store_true", help="Verify all wheel+manifest pairs (requires --key to be public PEM)")
       - Argument: --quiet (action="store_true", help="Suppress per-wheel summary output")

    7. Main logic:
       - Parse args
       - If args.verify:
         a. Resolve public key: resolve_key(args, mode="public")
         b. Call verify_manifests(Path(args.wheels_dir), public_key)
         c. Exit 0 if all verified, exit 1 if any failed
       - Else (signing mode):
         a. Resolve private key: resolve_key(args, mode="private")
         b. Call sign_wheels(Path(args.wheels_dir), private_key, args.deploy_name, args.quiet)
         c. Exit 0

    8. Error handling:
       - All file operations wrapped in try/except with clear error messages to stderr
       - Invalid JSON manifests: catch json.JSONDecodeError, print FAIL message
       - Base64 decode errors: catch Exception, print FAIL message
       - Signature verification: wrap in try/except, print FAIL with reason

    Reference: Phase 137 _verify_wheel_manifest() for manifest format and verification patterns.
  </action>
