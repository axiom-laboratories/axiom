# Plan 18-02: Wave 2 — OAuth Device Flow Implementation

**Objective**: Implement the `mop-push login` command using the RFC 8628 Device Authorization Flow.

## Context
Operators need to authenticate from the terminal without using passwords or transmitting private keys. The backend already supports device flow (Phase 17).

## Tasks
1. **Implement `mop_sdk/auth.py`**:
   - `DeviceFlowHandler` class: handle `/auth/device` request and `/auth/device/token` polling.
   - `CredentialStore` class: handle saving and loading `~/.mop/credentials.json` with appropriate file permissions (0600).
2. **Implement `mop-push login`**:
   - Call `DeviceFlowHandler`.
   - Display user code.
   - Open browser automatically.
   - Poll for token.
   - Save to `CredentialStore`.
3. **Unit Tests**:
   - Verify `CredentialStore` encryption/serialization (if applicable) and file permissions.
   - Mock backend device flow endpoints to test polling logic and error handling (`authorization_pending`, `slow_down`, etc.).

## Success Criteria
- `mop-push login` successfully follows the device flow and persists a JWT.
- Subsequent calls to `CredentialStore.load()` return the saved JWT.
- File permissions on `credentials.json` are strictly `0600`.
