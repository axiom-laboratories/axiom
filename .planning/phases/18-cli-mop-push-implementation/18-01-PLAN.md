# Plan 18-01: Wave 1 — Packaging & CLI Skeleton

**Objective**: Scaffolding the `mop-push` CLI within the `mop_sdk` package and ensuring it can be installed via `pip`.

## Context
Phase 18 requires a standalone CLI tool. Wave 1 focuses on the packaging structure and the basic command-line entry point.

## Tasks
1. **Create `pyproject.toml`**: Define the `mop_sdk` package metadata and the `mop-push` entry point.
2. **Create `mop_sdk/cli.py`**: Implement a basic `argparse` skeleton with `login` and `job` subcommands.
3. **Initialize `mop_sdk/tests/`**: Create a basic unit test to verify that the CLI help command executes without error.

## Success Criteria
- Running `pip install -e ./mop_sdk` succeeds.
- Running `mop-push --help` displays the expected subcommands (`login`, `job`).
- `pytest mop_sdk/tests/test_cli.py` passes.
