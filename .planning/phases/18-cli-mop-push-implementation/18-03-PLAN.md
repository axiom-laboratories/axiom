# Plan 18-03: Wave 3 — Job Signing & Staging

**Objective**: Implement local Ed25519 signing and the job push/create commands.

## Context
This is the core functional requirement of the CLI: allowing operators to sign and publish jobs from their local environment.

## Tasks
1. **Implement `mop_sdk/signer.py`**:
   - Helper to load Ed25519 PEM keys and sign script payloads.
2. **Update `mop_sdk/client.py`**:
   - Add `push_job` (calls `/api/jobs/push`) and `create_job` (calls existing scheduler API) methods to `MOPClient`.
3. **Implement `mop-push job push`**:
   - Load key, sign script, push to backend. Support `--name` (DRAFT) and `--id` (update).
4. **Implement `mop-push job create`**:
   - Direct creation of ACTIVE jobs with schedule and tags.
5. **Unit Tests**:
   - Verify signing matches backend verification logic.
   - Verify `MOPClient` correctly injects the Bearer token from the `CredentialStore`.

## Success Criteria
- `mop-push job push` returns a successful Job ID.
- Private key files are read but never sent in the HTTP request payload.
- All Ed25519 signing logic is covered by unit tests.
