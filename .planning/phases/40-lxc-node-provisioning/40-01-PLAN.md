**Helper functions:**

`load_env(path)` — reads KEY=VALUE pairs from a file, returns dict (same pattern as verify_ce_install.py).

`get_incusbr0_ip()` — runs `ip -json addr show incusbr0`, parses JSON, returns first inet address. Raises RuntimeError if not found.

`get_jwt(base_url, admin_password)` — POSTs to `/auth/login` with `{"username": "admin", "password": admin_password}`, returns JWT string.

`api_generate_token(base_url, jwt)` — POSTs to `/admin/generate-token`, returns token string.

`is_container_running(name)` — runs `incus list <name> --format json`, returns True if status == "Running".

`get_container_ip(name)` — polls `incus list <name> --format json` for global inet address on any network interface; retries up to 30 times with 2s sleep; raises RuntimeError on timeout.

`run_in_lxc(name, cmd_str)` — runs `incus exec <name> -- bash -c cmd_str`, prints output, raises on non-zero exit.

`push_file_to_lxc(name, local_path, remote_path)` — runs `incus file push <local_path> <name><remote_path>`. Note: incus file push syntax is `incus file push src dst` where dst is `container/path`.

**Main provisioning flow:**

1. Load secrets from SECRETS_ENV (`load_env`). Get `AGENT_URL` (or use default `https://localhost:8001`) and `ADMIN_PASSWORD`.
2. `bridge_ip = get_incusbr0_ip()`. Print it.
3. Push and tag the node image to local registry:
   ```
   docker tag localhost/master-of-puppets-node:latest localhost:5000/puppet-node:latest
   docker push localhost:5000/puppet-node:latest
   ```
   Run via subprocess. Fail fast if this fails.
4. Get admin JWT: `jwt = get_jwt("https://localhost:8001", admin_password)`.
5. For each node in NODE_CONFIGS:
   - Check if NODES_SECRETS_DIR / f"{node['name']}.env" already exists. If yes, print `[SKIP] Token file exists for {name} — reusing` and load it. If no, call `api_generate_token`, write the .env file:
     ```
     JOIN_TOKEN=<token>
     ENV_TAG=<env_tag>
     NODE_TAGS=<tags>
     AGENT_URL=https://<bridge_ip>:8001
     ```
     Print `[TOKEN] Generated and saved token for {name}`.
6. For each node in NODE_CONFIGS:
   - If `is_container_running(name)`: print `[SKIP] {name} already running — re-deploying compose stack only`.
   - Else:
     a. `incus launch ubuntu:24.04 <name> --config security.nesting=true` — wait up to 30s for launch.
     b. Wait for IP: `get_container_ip(name)`.
     c. Install Docker and compose plugin: `run_in_lxc(name, "apt-get update -qq && apt-get install -y docker.io docker-compose-plugin")`. This is the preferred method (faster than convenience script, docker-compose-plugin is in Ubuntu 24.04 repos and provides `docker compose` v2).
     d. Configure insecure registry: `run_in_lxc(name, f'mkdir -p /etc/docker && echo \'{{"insecure-registries":["{bridge_ip}:5000"]}}\' > /etc/docker/daemon.json && systemctl restart docker')`.
     e. Wait for Docker daemon: retry `run_in_lxc(name, "docker info")` up to 15 times with 2s sleep.
   - In all cases (running or freshly launched):
     f. Read LXC_COMPOSE_TEMPLATE, replace `__REGISTRY_IP__` with `bridge_ip`, write to a temp file.
     g. Push compose file: `incus file push /tmp/lxc-compose-<name>.yaml <name>/home/ubuntu/docker-compose.yaml`.
     h. Push .env file: `incus file push NODES_SECRETS_DIR/<name>.env <name>/home/ubuntu/.env`.
     i. Pull image: `run_in_lxc(name, f"docker pull {bridge_ip}:5000/puppet-node:latest")`.
     j. Deploy: `run_in_lxc(name, "cd /home/ubuntu && docker compose down 2>/dev/null || true && docker compose up -d")`.
     k. Print `[PASS] {name} deployed`.
7. After all nodes deployed, poll `GET /api/nodes` (via requests with admin JWT) up to 60 times with 3s sleep until all 4 names appear in the response with status HEALTHY. Print node count progress.
8. Print final summary: `[DONE] {N}/4 nodes HEALTHY`.

Exit 0 on success, exit 1 if any node fails to reach HEALTHY status.

**Error handling:**
- Use try/except around each node's provisioning block. On exception, print `[FAIL] {name}: {error}` and continue to next node (don't abort). Track failures.
- Fail fast only for: stack connectivity check (step 4) and incusbr0 discovery (step 2).
