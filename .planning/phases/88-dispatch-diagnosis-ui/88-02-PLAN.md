**4. Add fetchDiagnoses callback** inside the Jobs component, after the diagnosisCache state declaration.

Use `useCallback` with dependency on the pending GUIDs derived from the current `jobs` state. The callback must:
- Compute `pendingGuids = jobs.filter(j => j.status === 'PENDING' || j.status === 'ASSIGNED').map(j => j.guid)`
- Return early if `pendingGuids.length === 0`
- POST to `/jobs/dispatch-diagnosis/bulk` with body `{ guids: pendingGuids }`
- On success, spread-merge results into diagnosisCache: `setDiagnosisCache(prev => ({ ...prev, ...data.results }))`
- Silently swallow errors (non-critical)

```typescript
const fetchDiagnoses = useCallback(async () => {
    const pendingGuids = jobs
        .filter(j => j.status === 'PENDING' || j.status === 'ASSIGNED')
        .map(j => j.guid);
    if (pendingGuids.length === 0) return;
    try {
        const r = await authenticatedFetch('/jobs/dispatch-diagnosis/bulk', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ guids: pendingGuids }),
        });
        if (!r.ok) return;
        const data = await r.json();
        setDiagnosisCache(prev => ({ ...prev, ...data.results }));
    } catch {}
}, [jobs]);
```
