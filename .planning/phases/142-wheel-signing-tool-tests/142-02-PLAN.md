Key implementation patterns:

**Pattern 1: Key arg priority**
- Write private_pem to temp file
- Create args = Namespace(key=str(temp_path))
- Call resolve_key(args, mode="private")
- Assert returned object is an Ed25519PrivateKey instance (has .sign method)

**Pattern 2: Env var fallback**
- Write private_pem to temp file
- monkeypatch.setenv("AXIOM_WHEEL_SIGNING_KEY", str(temp_path))
- Create args = Namespace(key=None)  # No --key arg
- Call resolve_key(args, mode="private")
- Assert returned object is not None

**Pattern 3: Missing key error**
- monkeypatch.delenv("AXIOM_WHEEL_SIGNING_KEY", raising=False)
- Create args = Namespace(key=None)
- Expect pytest.raises(SystemExit) with message containing "no signing key"

**Pattern 4: File not found error**
- Create args = Namespace(key="/nonexistent/path/key.pem")
- Expect pytest.raises(SystemExit) with message containing "not found"

**Pattern 5: Malformed PEM error**
- Write invalid PEM bytes to temp file: key_path.write_bytes(b"not a valid PEM\n")
- Create args = Namespace(key=str(key_path))
- Expect pytest.raises(SystemExit) with message containing "failed to load" or similar

**Pattern 6: Private-to-public fallback**
- Write private_pem (not public) to temp file
- Create args = Namespace(key=str(temp_path))
- Call resolve_key(args, mode="public")
- Expect returned object is Ed25519PrivateKey (the private key was accepted as fallback)
- Alternatively, if passing public_pem, expect Ed25519PublicKey

From sign_waves.py lines 87-91:
```python
if mode == "public":
    try:
        return serialization.load_pem_private_key(key_bytes, password=None)
    except Exception:
        return serialization.load_pem_public_key(key_bytes)
```

So mode="public" tries private first, then public. Tests should validate this behavior by:
- Providing a private key file → expect return type is Ed25519PrivateKey
- Verifying the returned key can be used for verification (has .verify method)
