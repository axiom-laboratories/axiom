  <action>
In puppeteer/compose.server.yaml:

1. Add `pypi` service after the `agent` service block:
```yaml
  pypi:
    image: pypiserver/pypiserver:latest
    restart: always
    command: -P . -a . /data/packages
    volumes:
      - mirror-data:/data/packages
    ports:
      - "8080:8080"
```

2. Add `mirror` service (Caddy serving APT static files):
```yaml
  mirror:
    image: caddy:latest
    restart: always
    volumes:
      - mirror-data:/data
      - ./mirror/Caddyfile:/etc/caddy/Caddyfile:ro
    ports:
      - "8081:80"
```

3. Add `mirror-data` to the top-level `volumes:` block (alongside pgdata, certs-volume, etc.).

4. Add to the `agent` service environment and volumes:
   - Environment: `MIRROR_DATA_PATH=/app/mirror_data`
   - Volume: `mirror-data:/app/mirror_data`

Create puppeteer/mirror/Caddyfile:
```
:80 {
    root * /data/apt
    file_server browse
}
```

Create the directory `puppeteer/mirror/` if it does not exist.

Key constraints:
- pypiserver command must be `-P . -a . /data/packages` — the trailing path argument is where pypiserver serves packages from. Do NOT use just `/data` as pypiserver would serve the entire volume root.
- The `agent` volume mount must use `/app/mirror_data` to match the `MIRROR_DATA_PATH` env var.
- Do NOT remove any existing services or volumes.
