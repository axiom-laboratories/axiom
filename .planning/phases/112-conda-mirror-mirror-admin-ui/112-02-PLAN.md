## Service Provisioning Specifics

**Gating mechanism:** If ALLOW_CONTAINER_MANAGEMENT != "true", provisioning toggles show read-only status with manual docker compose command.

**Container configs (hardcoded):** For each ecosystem, store image name, ports, volumes, network. Examples:
- PyPI: `pypiserver:latest` (port 8080, volume: `/mirror-data/packages`)
- Conda: `mirror-sidecar:latest` (port 8081, volume: `/mirror-data/conda`)
- npm: `verdaccio:latest` (port 4873, volume: `/verdaccio/storage`)
- NuGet: `bagetter:latest` (port 5555, volume: `/data`)

**Auto-pull:** If container image not available locally, docker-py auto-pulls before create.

**Status indicators:** Running (green), Stopped (gray), Error (red) — fetched from docker.container.get().status.
