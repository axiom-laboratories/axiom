## Conda ToS Decision (from CONTEXT.md)

- Selecting the Anaconda `defaults` channel triggers a blocking modal dialog explaining Anaconda's commercial ToS
- Dialog recommends conda-forge as the free alternative; operator must acknowledge before proceeding
- Default channel for new Conda ingredients is conda-forge (pre-selected in UI)
- ToS acknowledgment persisted per-user — once acknowledged, that user won't see it again; other users still see it on first encounter

## Smelter UI Pattern

From existing Smelter.tsx:
- Ingredient selector with ecosystem dropdown, channel dropdown, version constraint field
- "Approve Ingredient" button submits the selection
- Modal/dialog pattern already established in codebase

## Config DB Pattern

From Plan 02:
- Config table stores per-user ToS acknowledgment: `CONDA_DEFAULTS_TOS_ACKNOWLEDGED_BY_{user_id}`
- Endpoint: POST /api/admin/conda-defaults-acknowledge accepts { "channel": "defaults" } (locked)
- GET /api/admin/mirror-config returns `conda_defaults_acknowledged_by_current_user: bool`
