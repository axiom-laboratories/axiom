  <how-to-verify>
    Open the dashboard at https://localhost:8001 (or the Cloudflare tunnel URL) and navigate to Admin → Automation tab.

    **1. Empty state**
    - If no triggers exist: confirm "No triggers yet." message displays with the description text and a "+ Create Trigger" button inside the table body

    **2. Create a trigger**
    - Click "+ Create Trigger" (either in card header or empty state)
    - Fill in: Name = "Test CI", Slug = "test-ci", select any job definition
    - Click "Register Trigger"
    - Confirm the trigger appears in the table with an "Active" green badge

    **3. Status badge**
    - The new Status column shows a green "Active" badge for the trigger just created

    **4. Copy Token**
    - Click "Copy Token" on the row — confirm a toast "Token copied" appears
    - (Optional) Paste into a text editor to verify it starts with "trg_")

    **5. Disable confirmation**
    - Click "Disable" on the active trigger
    - Confirm a dialog opens: "Disable this trigger? ... Disabling this trigger will prevent new jobs from being fired."
    - Click Cancel — trigger stays Active
    - Click Disable again, then click "Disable Trigger" in the dialog
    - Confirm the badge changes to grey "Inactive"

    **6. Re-enable without confirmation**
    - Click "Enable" on the now-inactive trigger
    - Confirm it immediately flips to Active with no confirmation dialog

    **7. Rotate Key**
    - Click "Rotate Key" on any trigger
    - Confirm a confirmation dialog: "Rotate trigger token? ... This will invalidate the current token."
    - Click Cancel — nothing changes
    - Click "Rotate Key" again, then "Rotate Token"
    - Confirm a one-time reveal modal opens with: amber warning banner, a token input field starting with "trg_", a Copy button, and "I have saved the token" button
    - Click the Copy button — confirm "Token copied" toast
    - Click "I have saved the token" — modal closes

    **8. Delete**
    - Click the red trash icon on any trigger
    - Confirm the trigger is removed from the table

    **9. Empty state after deletion**
    - If all triggers deleted: confirm empty state reappears
