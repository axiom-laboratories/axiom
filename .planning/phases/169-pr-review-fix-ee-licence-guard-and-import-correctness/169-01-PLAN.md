```python
    await test_service.startup()

```

Wrap the startup() and status() calls in a try/finally block that unconditionally calls test_service.shutdown():

```python
async with AsyncSessionLocal() as test_db:
    test_service = SIEMService(test_config, test_db, scheduler_service.scheduler)
    try:
        await test_service.startup()
        status = await test_service.status()
    finally:
        await test_service.shutdown()
```

The finally block MUST call `await test_service.shutdown()` with no conditions — this prevents APScheduler jobs registered during startup() from leaking if status() raises an exception.

Keep all other logic (audit calls, response building) in the outer try/except block — only the startup() and status() calls go inside the inner try/finally.
