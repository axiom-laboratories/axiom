       - Output: {status: "deleted", id: id}

    6. POST /api/admin/bundles/{id}/items
       - Input: CuratedBundleItemCreate
       - Output: CuratedBundleItemResponse
       - Logic: Create item, generate UUID id, link to bundle
       - Error: 404 if bundle not found, 422 if ecosystem invalid (enum validation via Pydantic)

    7. PATCH /api/admin/bundles/{id}/items/{item_id}
       - Input: CuratedBundleItemCreate (all fields optional)
       - Output: CuratedBundleItemResponse
       - Logic: Update only provided fields
       - Errors: 404 if bundle or item not found

    8. DELETE /api/admin/bundles/{id}/items/{item_id}
       - Output: {status: "deleted", item_id: item_id}
       - Logic: Delete item
       - Errors: 404 if bundle or item not found

    **Bundle Application Endpoint (operator can call, require foundry:write):**

    9. POST /api/foundry/apply-bundle/{bundle_id}
       - Output: ApplyBundleResult
       - Logic:
         a. Query bundle and all items
         b. Loop through items:
            - Check if ingredient already approved (query ApprovedIngredient by name + ecosystem)
            - If yes: increment skipped_count
            - If no: call SmelterService.add_ingredient(db, ApprovedIngredientCreate(...))
              - This auto-triggers ResolverService + MirrorService
              - Catch exceptions, log error, continue
            - Increment approved_count
         c. audit(db, current_user, "bundle:applied", f"{bundle.name} ({approved_count} new, {skipped_count} skipped)")
         d. Return ApplyBundleResult with all counts
       - Errors: 404 if bundle not found, 403 if user lacks foundry:write
       - Important: Toast feedback shows both approved + skipped counts (CONTEXT decision)

    **Registration (in main.py):**
    - Import bundles_router and include with `app.include_router(bundles_router.router, tags=["Bundles"])`

    Why: Endpoints follow existing Axiom patterns (require_permission, audit, Pydantic models). Loop + add_ingredient is transparent and observable (resolver/mirror runs per item). Already-approved skipping prevents duplicate approvals (idempotent design).
