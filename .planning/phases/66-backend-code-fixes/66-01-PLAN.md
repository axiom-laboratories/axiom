  <action>
Edit `puppets/Containerfile.node` to replace the hardcoded `_amd64.deb` PowerShell download
with a TARGETARCH-conditional block.

Exact changes:
1. Add `ARG TARGETARCH` as a new line DIRECTLY BEFORE the `RUN apt-get update` block (the
   block that installs system deps and PowerShell). BuildKit sets this arg automatically —
   no `--build-arg` is needed. It MUST appear before the RUN block that uses it.

2. Replace the existing wget line:
   ```
   && wget -q -O /tmp/powershell.deb \
      "https://github.com/PowerShell/PowerShell/releases/download/v7.6.0/powershell-lts_7.6.0-1.deb_amd64.deb" \
   ```
   With:
   ```
   && if [ "$TARGETARCH" = "arm64" ] || [ "$TARGETARCH" = "aarch64" ]; then \
        PWSH_ARCH=arm64; \
      else \
        PWSH_ARCH=amd64; \
      fi \
   && wget -q -O /tmp/powershell.deb \
        "https://github.com/PowerShell/PowerShell/releases/download/v7.6.0/powershell-lts_7.6.0-1.deb_${PWSH_ARCH}.deb" \
   ```

3. Keep everything else in the Containerfile unchanged: Docker CLI COPY on line 9 stays,
   ENV DOTNET_SYSTEM_GLOBALIZATION_INVARIANT=1 stays, requirements.txt install stays,
   node.py/runtime.py COPY stays.

4. Do NOT use multi-stage build — single-stage Containerfile as per CONTEXT.md decision.
5. PowerShell version stays at 7.6.0 — do not change the version string.

The resulting RUN block structure (within the single RUN layer):
```
apt-get update && apt-get install -y --no-install-recommends \
    curl wget apt-transport-https gnupg podman krb5-user iptables \
&& if [ "$TARGETARCH" = "arm64" ] || [ "$TARGETARCH" = "aarch64" ]; then \
     PWSH_ARCH=arm64; \
   else \
     PWSH_ARCH=amd64; \
   fi \
&& wget -q -O /tmp/powershell.deb \
     "https://github.com/.../powershell-lts_7.6.0-1.deb_${PWSH_ARCH}.deb" \
&& apt-get install -y /tmp/powershell.deb \
&& rm /tmp/powershell.deb \
&& rm -rf /var/lib/apt/lists/*
```
