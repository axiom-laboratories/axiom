## What Was Delivered

### 1. Workflow ORM Extended (db.py)
- **schedule_cron**: TEXT column storing cron expressions (e.g., "0 9 * * MON")
- **is_paused**: Confirmed existing; gates cron activation (fires when `schedule_cron IS NOT NULL AND is_paused = false`)
- **webhooks relationship**: Added reverse relationship to WorkflowWebhook for bidirectional ORM traversal

### 2. WorkflowRun ORM Extended (db.py)
- **trigger_type**: VARCHAR column for trigger source (MANUAL, CRON, WEBHOOK)
- **triggered_by**: VARCHAR column for human-readable audit trail (username, "scheduler", webhook_name)
- **parameters_json**: TEXT column storing resolved parameters as JSON string at run creation time
- Ensures all steps in a run see identical parameter values even if defaults change mid-run

### 3. WorkflowWebhook ORM Class (db.py)
- **id**: String PK (UUID, generated by endpoint)
- **workflow_id**: FK to workflows.id with ON DELETE CASCADE
- **name**: String label (e.g., "github-push")
- **secret_hash**: Bcrypt hash of plaintext secret (plaintext never stored; returned once at creation only)
- **created_at**: Timestamp, default current_time
- **workflow relationship**: Bidirectional to Workflow.webhooks

### 5. Migration SQL (migration_v55.sql)

Idempotent SQL for existing deployments:

```sql
-- Workflow columns
ALTER TABLE workflows ADD COLUMN IF NOT EXISTS schedule_cron TEXT;

-- WorkflowRun columns
ALTER TABLE workflow_runs ADD COLUMN IF NOT EXISTS trigger_type VARCHAR(50);
ALTER TABLE workflow_runs ADD COLUMN IF NOT EXISTS triggered_by VARCHAR(255);
ALTER TABLE workflow_runs ADD COLUMN IF NOT EXISTS parameters_json TEXT;

-- WorkflowWebhook table
CREATE TABLE IF NOT EXISTS workflow_webhooks (
    id VARCHAR(36) PRIMARY KEY,
    workflow_id VARCHAR(36) NOT NULL,
    name VARCHAR(255) NOT NULL,
    secret_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (workflow_id) REFERENCES workflows(id) ON DELETE CASCADE
);

-- Index for fast webhook queries by workflow_id
CREATE INDEX IF NOT EXISTS idx_workflow_webhooks_workflow_id
ON workflow_webhooks(workflow_id);
```

## Architecture Alignment

- **Fresh deployments**: `create_all` in `init_db()` handles WorkflowWebhook table + all new columns via SQLAlchemy ORM definitions

- **Naming patterns**: Follows Phase 146-148 conventions (schedule_cron matches ScheduledJob pattern; Text for long strings; Optional[str] for nullable ORM fields)
