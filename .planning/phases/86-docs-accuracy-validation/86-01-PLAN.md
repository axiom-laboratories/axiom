# Plan 86-01: Docs Validation Scripts

## Goal

Write `tools/generate_openapi.py` (snapshot generator) and `tools/validate_docs.py` (validator), run the generator against the live Docker stack to populate `docs/docs/api-reference/openapi.json`, then run the validator to verify the committed snapshot is clean.

## Context

- **CONTEXT.md decisions** govern all design choices: two-script strategy, exit codes, PASS/WARN/FAIL semantics, env var WARN (not FAIL), strict CI gate (any WARN or FAIL exits 1)
- `tools/capture_screenshots.py` is the structural template for `generate_openapi.py`
- `mop_sdk/cli.py` registered subcommands: `login`, `job push`, `job create`, `key generate`, `init` — parse statically with regex, do not import
- Env var search scope: `puppeteer/`, `puppets/`, `mop_sdk/`, `tools/example-jobs/` — widen to `tools/example-jobs/` to avoid false positives on `AXIOM_VOLUME_PATH` / `AXIOM_BLOCKED_HOST`
- Default URL for both scripts: `http://localhost:8080` (Caddy proxy front-end)
- `validate_docs.py` must exit 2 if `openapi.json` has no paths (stub detection)

### Task 1: Write tools/generate_openapi.py

Write `tools/generate_openapi.py`. Requirements:

- **No auth required** — `/openapi.json` is unauthenticated
- **Stdlib only** plus `requests` (already in `puppeteer/requirements.txt`)
- `REPO_ROOT` anchored at two levels above `tools/` (same pattern as `capture_screenshots.py`)
- Accepts `--url` flag (default: `http://localhost:8080`)
- Fetches `{url}/openapi.json`, writes result to `docs/docs/api-reference/openapi.json`
- Prints route count on success: `Written to {path}\nRoutes: {N}`
- Exits 1 with clear error if fetch fails or response is not valid JSON

Script structure:
```python
#!/usr/bin/env python3
"""
Axiom OpenAPI Snapshot Generator

Fetches the OpenAPI spec from a running Docker stack and commits it to
docs/docs/api-reference/openapi.json as the source-of-truth snapshot
for docs accuracy validation.

Usage:
    python tools/generate_openapi.py [--url URL]

Options:
    --url URL    Base URL of the running stack (default: http://localhost:8080)
"""
import argparse
import json
import sys
from pathlib import Path

import requests

REPO_ROOT = Path(__file__).resolve().parents[1]


def main():
    parser = argparse.ArgumentParser(
        description="Fetch OpenAPI spec from running stack and save snapshot."
    )
    parser.add_argument(
        "--url",
        default="http://localhost:8080",
        help="Base URL of the running stack (default: http://localhost:8080)",
    )
    args = parser.parse_args()

    print(f"Fetching OpenAPI spec from {args.url}/openapi.json ...")
    try:
        r = requests.get(f"{args.url}/openapi.json", timeout=10)
        r.raise_for_status()
    except requests.exceptions.ConnectionError:
        print(f"[ERROR] Cannot connect to {args.url} — is the stack running?")
        sys.exit(1)
    except requests.exceptions.HTTPError as e:
        print(f"[ERROR] HTTP error: {e}")
        sys.exit(1)
    except requests.exceptions.Timeout:
        print(f"[ERROR] Connection timed out to {args.url}")
        sys.exit(1)

    try:
        spec = r.json()
    except Exception:
        print("[ERROR] Response is not valid JSON")
        sys.exit(1)

    out_path = REPO_ROOT / "docs" / "docs" / "api-reference" / "openapi.json"
    out_path.write_text(json.dumps(spec, indent=2))

    route_count = len(spec.get("paths", {}))
    print(f"Written to {out_path}")
    print(f"Routes: {route_count}")

    if route_count == 0:
        print("[WARN] Snapshot has no routes — is this a stub?")


if __name__ == "__main__":
    main()
```

### Task 2: Write tools/validate_docs.py

Write `tools/validate_docs.py`. This is the main deliverable of the phase. Full requirements:

**Exit codes:**
- `0` — all items PASS
- `1` — any WARN or FAIL
- `2` — `openapi.json` is missing or has no paths (stub detection)

**Stub check (must run first):**
```python
spec_path = REPO_ROOT / "docs" / "docs" / "api-reference" / "openapi.json"
if not spec_path.exists():
    print("ERROR: OpenAPI snapshot not found. Run tools/generate_openapi.py first.", file=sys.stderr)
    sys.exit(2)
spec = json.loads(spec_path.read_text())
if not spec.get("paths"):
    print("ERROR: OpenAPI snapshot is empty. Run tools/generate_openapi.py first.", file=sys.stderr)
    sys.exit(2)
```

**Extraction regexes:**

Route extraction (both inline prose and code blocks):
```python
ROUTE_RE = re.compile(
    r'\b(GET|POST|PUT|DELETE|PATCH)\s+(/(?:api|admin|auth|system|work|jobs|nodes|'
    r'signatures|job-definitions|config|triggers|heartbeat|verification-key)[^\s`\'"]*)',
    re.IGNORECASE
)
```

CLI subcommand extraction:
```python
CLI_RE = re.compile(r'\baxiom-push\s+([\w-]+(?:\s+[\w-]+)?)')
```

Env var extraction (backtick-wrapped, all-caps, 3+ chars):
```python
ENV_RE = re.compile(r'`([A-Z][A-Z0-9_]{3,})`')
```

**CLI subcommand validation from mop_sdk/cli.py (static parse, no import):**
```python
def get_registered_commands() -> set:
    cli_path = REPO_ROOT / "mop_sdk" / "cli.py"
    source = cli_path.read_text()
    names = re.findall(r'add_parser\(["\'](\w[\w-]*)["\']', source)
    return set(names)
```

**Env var source search:**
```python
SEARCH_DIRS = ["puppeteer", "puppets", "mop_sdk", "tools"]
EXCLUDE_PATTERNS = {"venv", ".venv", "node_modules", ".git", "__pycache__", "dist", "build"}

def var_in_source(var_name: str) -> bool:
    for base_name in SEARCH_DIRS:
        base = REPO_ROOT / base_name
        if not base.exists():
            continue
        for py_file in base.rglob("*.py"):
            if any(ex in py_file.parts for ex in EXCLUDE_PATTERNS):
                continue
            if var_name in py_file.read_text():
                return True
    return False
```

**Route matching against openapi.json:**
- Strip query string before matching: `path = re.sub(r'\?.*', '', path)`
- Normalize trailing slashes: `path = path.rstrip('/')`
- Check `spec["paths"]` for the path key, then check the method under it
- OpenAPI paths use `{param}` syntax — do a best-effort match that converts `{node_id}` etc. to wildcard when no exact match

**Output format (one line per item):**
```
PASS   GET /api/features              docs/docs/licensing.md:14
FAIL   GET /api/verification-key      docs/docs/runbooks/jobs.md:87  [not in openapi.json]
WARN   AXIOM_CUSTOM_VAR               docs/docs/runbooks/custom.md:22  [not found in source]

Summary: 14 PASS, 1 WARN, 1 FAIL
```

**Deduplication:** If the same route/var/command appears in multiple docs files, report each occurrence separately (file:line is the key info).

**Exclusions:** Skip `docs/docs/api-reference/openapi.json` (not markdown) and any non-`.md` files.

### Task 4: Run validator and fix any real FAILs

Run `validate_docs.py` against the populated snapshot. Expected outcomes:

- **PASS items:** All documented routes that exist in the spec
- **WARN items:** Env vars like `AXIOM_VOLUME_PATH` / `AXIOM_BLOCKED_HOST` may still WARN if not found in `tools/example-jobs/*.py` — check if widening search to `tools/` resolves this
- **FAIL items:** Any route in docs not found in spec — these are real accuracy issues to fix

For any FAIL result:
1. Check if the route exists in `puppeteer/agent_service/main.py` with a different path format
2. If docs are wrong: fix the docs file (update route path)
3. If route was removed from the API: update docs to remove or correct the reference

For WARNs that are job-script env vars: verify they appear in `tools/example-jobs/` scripts. If not, they are legitimate warnings — document them as known.
