# Plan 86-02: CI Integration

## Goal

Add a `docs-validate` job to `.github/workflows/ci.yml` that runs `python tools/validate_docs.py` on every push to main and every PR. The job requires only the committed `openapi.json` snapshot and Python stdlib + `requests` — no live stack needed.

## Context

- Plan 86-01 must be complete: `tools/validate_docs.py` must exist and be verified working
- The existing `ci.yml` already has `backend`, `frontend-lint`, `frontend-test`, `docker-validate`, `docs`, and `secret-scan` jobs
- New job follows the same pattern as the `docs` job: checkout + Python setup + install + run
- `validate_docs.py` requires only `requests` beyond stdlib — add a minimal pip install step
- CONTEXT.md decision: "Trigger: every push to main + every PR (no path filter — runs unconditionally)"
- CONTEXT.md decision: "Script installed as a GitHub Actions step: `python tools/validate_docs.py`"

## Tasks

<task id="86-02-01" wave="1">
### Task 1: Add docs-validate job to ci.yml

Read the current `.github/workflows/ci.yml` and append the `docs-validate` job. The job must:

1. Run on `ubuntu-latest`
2. Use Python 3.12
3. Install `requests` only (no full requirements.txt needed)
4. Run `python tools/validate_docs.py` from the repo root
5. Exit non-zero (fail CI) if the validator exits 1 or 2

Add this job block to `.github/workflows/ci.yml`:

```yaml
  docs-validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python 3.12
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install dependencies
        run: pip install requests

      - name: Validate docs accuracy
        run: python tools/validate_docs.py
```

Add this as the last job in the `jobs:` block (after `secret-scan`).

**Verification:**
