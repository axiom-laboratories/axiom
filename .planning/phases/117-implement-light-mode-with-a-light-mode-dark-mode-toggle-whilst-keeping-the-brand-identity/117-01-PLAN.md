<task type="auto">
  <name>Task 3: Add FOWT prevention inline script to index.html</name>
  <files>puppeteer/dashboard/index.html</files>
  <action>
Add an inline script in the `<head>` of index.html that runs before React hydrates. This script prevents the flash-of-wrong-theme (FOWT) by setting the correct theme class on <html> synchronously:

```html
<head>
  <!-- ... existing head content ... -->
  <script>
    // FOWT prevention: read localStorage and set theme class before React mounts
    (function() {
      const stored = localStorage.getItem('mop_theme');
      const theme = stored === 'light' ? 'light' : 'dark';
      if (theme === 'light') {
        document.documentElement.classList.add('light');
      } else {
        document.documentElement.classList.remove('light');
      }
    })();
  </script>
  <!-- Rest of head -->
</head>
```

Place this script as early as possible in the `<head>` tag — before any stylesheets or preload directives. The default is 'dark' (even if localStorage is empty or missing), so the login page (which always stays dark) will render correctly.

The `<html>` tag will have:
- No `dark` class (the Tailwind darkMode config uses .dark prefix on parent, but we're using .light on <html>)
- `.light` class when localStorage.mop_theme === 'light'
- Neither class when the theme is dark (default)

Do not alter the existing body or other elements — just add this one IIFE script to the head.
