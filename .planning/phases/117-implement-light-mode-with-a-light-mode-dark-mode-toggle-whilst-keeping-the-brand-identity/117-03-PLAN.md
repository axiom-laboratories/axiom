<objective>
Migrate all hardcoded dark-mode Tailwind classes throughout the dashboard to theme-aware alternatives. This plan replaces component-level styling with CSS variable-backed utilities and conditional classes that respond to the light/dark theme toggle.

Purpose: Without this migration, the UI will stay dark-colored even when the toggle switches to light mode. This is the bulk of the work needed to make light mode visually complete.
Output: All dashboard components and views updated for light/dark theme support
</objective>
