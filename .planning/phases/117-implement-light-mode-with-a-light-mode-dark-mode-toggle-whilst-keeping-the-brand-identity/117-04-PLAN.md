<checkpoint>

<what-built>
Light and dark theme infrastructure with complete CSS variable layer, theme provider hook, sidebar toggle component, and full component styling migration across all dashboard views. The system switches between light (warm stone palette) and dark (existing) modes via a segmented sun/moon slider in the sidebar footer. Theme persists to localStorage and is restored on page reload.
</what-built>

<how-to-verify>

**Setup:**
1. Navigate to https://localhost:5173 (or the dashboard URL in your Docker stack)
2. Log in with admin credentials

**Test 1: Theme Toggle Visibility and Functionality**
1. Look at the sidebar footer — you should see a segmented sun/moon toggle (pink slider dot between two icons)
2. Click the toggle to switch from dark to light mode
3. Verify the UI changes colors smoothly (200ms transition):
   - Page background should change from dark gray/black to warm off-white (stone-50)
   - Text should change from white to dark brown (stone-900)
   - Cards should change from darker gray (zinc-900) to warm gray (stone-100)
   - Input fields should change from dark to white background

**Test 2: Light Mode Color Scheme**
Verify that light mode displays the correct colors as specified:
1. Main page background: warm off-white (stone-50) ✓
2. Sidebar background: warm gray (stone-100) ✓
3. Card backgrounds: warm gray (stone-100) ✓
4. Input fields: white background with stone-300 border ✓
5. Primary buttons: pink background with white text (unchanged) ✓
6. Text colors: dark brown/stone-900 for primary text ✓
7. Secondary text: lighter brown (stone-600) ✓
8. Status badges: soft colored backgrounds with matching text
   - Success: emerald-50 background with emerald-700 text ✓
   - Error: red-50 background with red-700 text ✓
   - Warning: amber-50 background with amber-700 text ✓

**Test 3: Dark Mode Unchanged**
1. Toggle back to dark mode
2. Verify all colors match the original dark theme (no regressions)
3. Page background should return to dark gray/black
4. Text should return to white
5. All UI elements should look identical to before phase 117

**Test 4: Theme Persistence**
1. Toggle to light mode
2. Refresh the page (F5 or Cmd+R)
3. Verify light mode is still active (no flash of dark theme on reload)
4. Close the browser and reopen the dashboard
5. Verify light mode is still persisted
6. Toggle to dark mode and repeat the persistence checks

**Test 5: Login Page Always Dark**
1. Log out by clicking the logout button in the sidebar
2. Verify the login page is always dark, regardless of the theme toggle
3. Log back in
4. The theme setting should be remembered (not reset by login)

**Test 6: Component Styling (Navigate to Each View)**
Go through each dashboard view and verify light mode styling:

- **Dashboard.tsx**: Cards, stats, metrics should all render in light colors
- **Nodes.tsx**:
  - Node cards and status badges should show correct colors
  - Sparkline charts should have light-themed grid lines and tooltips
  - Hover states should darken slightly (stone-100 → stone-200)
- **Jobs.tsx**: Job list and status indicators should be light-themed
- **JobDefinitions.tsx**: Scheduled job cards and toggles should be light-themed
- **Templates.tsx**: Template cards and buttons should render correctly
- **Signatures.tsx**: Signature table rows should have subtle alternating stripes (stone-50/white)
- **Users.tsx**: User table and role panels should be light-themed
- **AuditLog.tsx**: Log table should have light styling with alternating row stripes
- **Admin.tsx**: Admin panels and configuration fields should be light-themed

**Test 7: Interactive Elements in Light Mode**
1. Hover over buttons, cards, and interactive elements
2. Verify hover states (darken on hover) work smoothly
3. Click input fields and verify the pink focus ring appears
4. Check dropdowns, modals, and dialogs open with light mode styling
5. Modals should have a lighter backdrop (black/30 vs black/60)

**Test 8: Charts and Complex Components**
If any views have Recharts charts:
1. Verify chart grid lines are light (stone-200)
2. Verify axis labels are readable (stone-500)
3. Verify tooltip backgrounds are white with shadow in light mode
4. Verify chart fills are softer/less vibrant in light mode

**Test 9: Toast Notifications**
If any operations trigger toast notifications:
1. Perform an action that shows a toast (e.g., create/update something)
2. Verify the toast background and text colors are appropriate for light mode
3. Verify the toast is readable and not glaring

**Test 10: Accessibility**
1. Use browser DevTools color contrast checker or accessibility tool
2. Verify text contrast is sufficient in both light and dark modes (WCAG AA minimum)
3. Verify focus rings are visible and the brand pink color is used

**Expected behavior for all tests:**
- Smooth 200ms transitions when toggling
- No flash of wrong theme on page reload (FOWT prevention working)
- Light mode colors match the decision document exactly
- Dark mode looks identical to the original
- All interactive elements respond correctly in both modes

</how-to-verify>
