<decisions_made>

- Signature placement: `signature`, `signature_payload`, `signature_id` go INSIDE `payload` dict
  (not top-level of JobCreate). Server stores them in the payload JSON; node reads `payload.get("signature")`.
  The dispatch_job() function at line 1075 of main.py is the canonical example.
- Server signing key: The server auto-generates an Ed25519 key pair at startup via
  `pki_service.ca_authority.ensure_signing_key("secrets")`. This key (not any host-side key) is what
  nodes trust via `/verification-key`. Tests must use this key.
- Test node runtime: EXECUTION_MODE=podman with JOB_IMAGE=docker.io/library/python:3.12-slim is the
  working combination for running the test node inside Docker with host networking.
- `direct` execution mode removed from node.py: confirms the "node enrollment fix" referenced in the
  session start message. Nodes must use docker/podman/auto.
