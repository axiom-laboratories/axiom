    ```python
    from agent_service.db import Base
    ...
    target_metadata = Base.metadata
    ```
    
    Change to:
    ```python
    from agent_service.db import Base, EE_Base
    ...
    target_metadata = [Base.metadata, EE_Base.metadata]
    ```
    
    **Rationale:** Alembic's `autogenerate` feature scans `target_metadata` to detect schema changes. By including both, Alembic will generate migrations for both CE and EE table changes. This is safe because:
    1. EE tables only exist in EE deployments (init_db() conditionally creates them)
    2. Alembic migrations are applied sequentially; if an EE table doesn't exist in CE mode, the migration is skipped at runtime
    3. In EE mode, both CE and EE tables exist, so all migrations apply
  </action>
