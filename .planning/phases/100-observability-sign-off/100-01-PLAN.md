# Plan 100-01: Scale Health Endpoint + Pydantic Model

## Goal

Implement `GET /api/health/scale` — a new FastAPI endpoint that returns live pool stats (Postgres) or null-safe defaults (SQLite), APScheduler job count, and current pending job depth. Add the `ScaleHealthResponse` Pydantic model and a Wave 0 test file covering the endpoint and model schema.

## Context

All prerequisites are done:
- `IS_POSTGRES` exported from `db.py` — gates pool stats path
- `engine` available in `db.py` — `engine.pool` exposes `QueuePool.size()`, `.checkedout()`, `.checkedin()`, `.overflow()`
- `scheduler_service.scheduler` is the `AsyncIOScheduler` instance — `get_jobs()` returns all registered jobs
- Existing health endpoint pattern at `main.py:757` (`/api/health/scheduling`) is the reference implementation
- `ix_jobs_status_created_at` index (Phase 98) makes `COUNT(*) WHERE status='PENDING'` efficient

## must_haves

- `GET /api/health/scale` returns HTTP 200 with JSON body
- Response includes: `is_postgres`, `pool_size`, `checked_out`, `available`, `overflow`, `apscheduler_jobs`, `pending_job_depth`
- On SQLite: `is_postgres: false`, all four pool fields are `null`, apscheduler_jobs and pending_job_depth are integers >= 0
- On Postgres: all pool fields are non-null integers
- Auth: requires valid JWT (require_auth), no specific permission needed
- Endpoint placed immediately after `/api/health/scheduling` in main.py

### Implement GET /api/health/scale endpoint in main.py

Add the endpoint immediately after `get_scheduling_health_endpoint` (main.py ~line 766):

```python
@app.get("/api/health/scale", response_model=ScaleHealthResponse, tags=["Health"])
async def get_scale_health_endpoint(
    current_user: User = Depends(require_auth),
    db: AsyncSession = Depends(get_db),
):
    """Return live pool and scheduler health metrics."""
    from .db import engine, IS_POSTGRES
    from .services.scheduler_service import scheduler_service
    from sqlalchemy import func, select as sa_select

    # APScheduler job count
    apscheduler_jobs = len(scheduler_service.scheduler.get_jobs())

    # Pending job depth
    result = await db.execute(
        sa_select(func.count(Job.guid)).where(Job.status == "PENDING")
    )
    pending_depth = result.scalar() or 0

    if not IS_POSTGRES:
        return ScaleHealthResponse(
            is_postgres=False,
            pool_size=None,
            checked_out=None,
            available=None,
            overflow=None,
            apscheduler_jobs=apscheduler_jobs,
            pending_job_depth=pending_depth,
        )

    pool = engine.pool
    return ScaleHealthResponse(
        is_postgres=True,
        pool_size=pool.size(),
        checked_out=pool.checkedout(),
        available=pool.checkedin(),
        overflow=pool.overflow(),
        apscheduler_jobs=apscheduler_jobs,
        pending_job_depth=pending_depth,
    )
```

Import note: `require_auth` is already imported in main.py. `Job` is already accessible. No new imports needed at module level.
