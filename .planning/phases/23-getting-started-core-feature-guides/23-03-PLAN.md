Replace the temporary stub with the full Foundry guide. Structure:

1. **Title and intro**: "# Foundry" — intro paragraph explaining that Foundry builds custom Docker images for Puppet nodes. Blueprints define what goes into an image; templates combine blueprints into a buildable image.

2. **Concepts section**: "## Concepts"
   - Brief definition table:
     | Concept | Description |
     |---------|-------------|
     | Blueprint | Reusable definition of a runtime environment (packages) or network config |
     | Template | Combines a RUNTIME + NETWORK blueprint into a Docker image build |
     | Smelter | Built-in package scanner that validates ingredients against the Smelter registry |
     | Image Lifecycle | Status tracking for built images (ACTIVE / DEPRECATED / REVOKED) |

3. **Blueprints section**: "## Blueprints"
   - Two types: RUNTIME (Python packages, execution env) and NETWORK (network config)
   - How to create a blueprint: navigate to **Foundry → Blueprints → New Blueprint**
   - The Blueprint Wizard opens — walk through it step by step (see wizard steps in interfaces)
   - `!!! danger "Package format"` admonition: packages **must** use the dict format `{"python": ["requests", "numpy"]}` — a plain list like `["requests", "numpy"]` is rejected. This is the most common source of blueprint creation failures.
   - Cloning: Step 1 offers "Clone Existing" to base a new blueprint on an existing one
   - Advanced mode: "Advanced (JSON)" toggle in the wizard bypasses the form and lets you paste raw JSON — useful for importing blueprints from another environment
   - After submission: blueprint appears in the Blueprints list with its OS family and package count

4. **Templates section**: "## Templates"
   - A template combines one RUNTIME blueprint and one NETWORK blueprint
   - Navigate to **Foundry → Templates → New Template**
   - Select the runtime blueprint and network blueprint
   - Click **Build** — Foundry copies the environment_service code into a temp build context and runs `docker build`
   - Build progress is visible in the dashboard; failures surface the Docker build log
   - On success: template shows **Last Built** timestamp and is ready for node enrollment

5. **Smelter integration**: "## Smelter"
   - Smelter is the built-in package validation engine. It checks packages against the Smelter registry during blueprint Step 3 (Ingredients).
   - Two enforcement modes:
     | Mode | Behaviour |
     |------|-----------|
     | STRICT | Build fails if any package fails the CVE scan |
     | WARNING | Build proceeds; template shows an amber warning badge |
   - The enforcement mode is set per-template at build time
   - `!!! info "Deep CVE configuration"` admonition linking to the Security section (Phase 24): "Configuring custom enforcement policies and adding packages to the Smelter registry is covered in [Security → Smelter Configuration](../security/index.md)."

6. **Image lifecycle**: "## Image Lifecycle"
   - Every built template has a lifecycle status. Use the **Foundry → Templates** view to change it.
   - Status table:
     | Status | Meaning | How to change |
     |--------|---------|---------------|
     | ACTIVE | Normal operation — nodes can enroll and use this image | Default after build |
     | DEPRECATED | New node enrollments with this template are blocked | Click **Deprecate** in the template row |
     | REVOKED | All enrollment with this template blocked (403) | Click **Revoke** in the template row |
   - `!!! warning "Revoke is irreversible"` admonition: revoking a template cannot be undone from the dashboard. Use DEPRECATED if you want to phase out a template without immediately blocking all nodes.
   - For enforcement mechanics (how REVOKED blocks `/api/enroll`), see the [Architecture guide](../developer/architecture.md).

7. **Quick reference** at the bottom — one-sentence descriptions of each workflow action:
   - Create a blueprint: Foundry → Blueprints → New Blueprint → complete the wizard
   - Build a template: Foundry → Templates → New Template → select blueprints → Build
   - Rebuild after base image update: Foundry → Templates → template row → Rebuild
   - Change lifecycle status: Foundry → Templates → template row → status dropdown
