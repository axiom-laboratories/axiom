# Plan 85-01: Screenshot Capture Script

## Objective

Write `tools/capture_screenshots.py` — an operator-invoked Playwright script that seeds demo data against a live Docker stack and captures 11 named PNG screenshots at 1440×900, writing them to both `docs/docs/assets/screenshots/` and `homepage/assets/screenshots/`.

## Context

- **Auth pattern**: `POST /api/auth/login` with form-encoded `username`/`password`, inject JWT into `localStorage['mop_auth_token']`
- **Playwright launch**: `p.chromium.launch(headless=True, args=['--no-sandbox'])` (required on Linux per CLAUDE.md)
- **Credentials**: Read `ADMIN_PASSWORD` from `puppeteer/secrets.env` using simple line-by-line parser
- **Signing**: Generate ephemeral Ed25519 keypair inline — no external key file dependency
- **Viewport**: 1440×900 exactly (SCR-01 spec)
- **Wait strategy**: `wait_for_load_state("networkidle")` preferred over `sleep`
- **Reference**: `mop_validation/scripts/test_playwright.py` has the correct working pattern

## must_haves

- `python tools/capture_screenshots.py --check` exits 0 on a live stack with an enrolled node
- `python tools/capture_screenshots.py --check` exits 1 with a clear error if stack is unreachable or no nodes enrolled
- Running the full script produces 11 PNG files in `docs/docs/assets/screenshots/` and `homepage/assets/screenshots/`
- All screenshots are at 1440×900 (not fullPage)
- Script has zero hardcoded credentials — reads from `puppeteer/secrets.env`

### Task 1: Script skeleton with argument parsing and pre-flight check

**File:** `tools/capture_screenshots.py`

Create the script with:

1. **Argument parser**:
   - `--url URL` — base URL of the running stack, defaults to `http://localhost:8080`
   - `--check` — run pre-flight only (no screenshots), then exit

2. **`load_secrets()`** — reads `puppeteer/secrets.env` (relative to repo root, i.e., same-level as `tools/`). Line-by-line parse: skip blank lines and `#` comments, split on first `=`. Return dict. Raise with clear message if file not found.

3. **`preflight_check(base_url, admin_password)`** — checks:
   - Stack reachable: `GET {base_url}/` → 200
   - Admin credentials valid: `POST {base_url}/api/auth/login` with form-encoded body → get JWT
   - At least one enrolled node: `GET {base_url}/api/nodes` with `Authorization: Bearer {jwt}` → `len(nodes) > 0`
   - Print result of each check. On any failure: print clear error and return `False`. On success: return JWT.

4. **`main()`** function:
   - Load secrets
   - If `--check`: run `preflight_check()`, print "Pre-flight OK. Ready to capture." and `sys.exit(0)` on success, `sys.exit(1)` on failure
   - Otherwise: run `preflight_check()` first, abort with exit 1 if failed

5. **Output directory setup**: `os.makedirs("docs/docs/assets/screenshots", exist_ok=True)` and `os.makedirs("homepage/assets/screenshots", exist_ok=True)` — paths relative to `REPO_ROOT` (computed as `Path(__file__).resolve().parents[1]`)

**Dependencies**: `requests`, `pathlib`, `argparse`, `sys`, `os` — all stdlib except `requests`

### Task 2: Data seeding — ephemeral keypair + job dispatch

**File:** `tools/capture_screenshots.py` (extend)

Add `seed_demo_data(base_url, jwt)` function:

1. **Generate ephemeral Ed25519 keypair** (inline, no file write needed):
   ```python
   from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PrivateKey
   from cryptography.hazmat.primitives import serialization
   priv = Ed25519PrivateKey.generate()
   pub_pem = priv.public_key().public_bytes(
       serialization.Encoding.PEM,
       serialization.PublicFormat.SubjectPublicKeyInfo
   ).decode()
   ```

2. **Register public key** via `POST {base_url}/api/signatures` with body `{"name": "screenshot-seed-key", "public_key": pub_pem}`. If a key named "screenshot-seed-key" already exists (409/400), list signatures and find its ID.

3. **Sign and dispatch 4 jobs** (mix of outcomes):
   - **Job 1** — Python hello: `print("hello from axiom")` → should complete quickly
   - **Job 2** — Python with output: `import platform; print(f"node: {platform.node()}")` → completes
   - **Job 3** — Python that raises: `raise RuntimeError("intentional failure for demo")` → FAILED status
   - **Job 4** — Python sleep: `import time; time.sleep(2); print("done")` → completes after brief wait

   For each job:
   ```python
   import base64
   sig = base64.b64encode(priv.sign(script.encode("utf-8"))).decode()
   payload = {
       "task_type": "script",
       "runtime": "python",
       "script": script,
       "signature_id": sig_id,
       "signature": sig
   }
   requests.post(f"{base_url}/api/jobs", json=payload, headers={"Authorization": f"Bearer {jwt}"})
   ```

4. **Wait for terminal state** — poll `GET {base_url}/api/jobs` for up to 30 seconds until at least 2 jobs are in COMPLETED or FAILED status (1-second polling interval). Print progress dots during wait.

5. Return `{"sig_id": sig_id}` for reference (not currently needed but clean interface).

**Note on node targeting**: Do NOT set `target_node_id` — let the scheduler route naturally to any available node. If no node is online, `preflight_check` will have already caught this.

### Task 3: Screenshot capture — 11 views

**File:** `tools/capture_screenshots.py` (extend)

Add `capture_screenshots(base_url, jwt, out_dirs)` function where `out_dirs` is a list of two `Path` objects: `docs/docs/assets/screenshots/` and `homepage/assets/screenshots/`.

**Playwright setup:**
```python
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch(headless=True, args=["--no-sandbox"])
    ctx = browser.new_context(viewport={"width": 1440, "height": 900})
    page = ctx.new_page()
```

**Auth injection helper** (call before any authenticated page):
```python
def auth_page(page, base_url, jwt, route):
    page.goto(f"{base_url}/login", wait_until="domcontentloaded")
    page.evaluate(f"localStorage.setItem('mop_auth_token', '{jwt}')")
    page.goto(f"{base_url}{route}")
    page.wait_for_load_state("networkidle")
```

**Capture sequence:**

| # | Filename | Route | Action |
|---|----------|-------|--------|
| 1 | `login.png` | `/login` | No auth — navigate directly, wait for networkidle |
| 2 | `dashboard.png` | `/` | `auth_page(page, base_url, jwt, "/")` |
| 3 | `nodes.png` | `/nodes` | `auth_page(page, base_url, jwt, "/nodes")` |
| 4 | `node_detail.png` | `/nodes` | Reuse loaded nodes page — `page.locator("table tbody tr").first.click()` + `wait_for_timeout(800)` |
| 5 | `jobs.png` | `/jobs` | `auth_page(page, base_url, jwt, "/jobs")` |
| 6 | `job_detail.png` | `/jobs` | Reuse loaded jobs page — click a completed job row: `page.locator("tr").filter(has_text="COMPLETED").first.click()` + `wait_for_timeout(800)` |
| 7 | `queue.png` | `/queue` | `auth_page(page, base_url, jwt, "/queue")` |
| 8 | `history.png` | `/history` | `auth_page(page, base_url, jwt, "/history")` |
| 9 | `scheduled_jobs.png` | `/scheduled-jobs` | `auth_page(page, base_url, jwt, "/scheduled-jobs")` |
| 10 | `foundry.png` | `/templates` | `auth_page(page, base_url, jwt, "/templates")` |
| 11 | `audit.png` | `/audit` | `auth_page(page, base_url, jwt, "/audit")` |

**Save helper** — for each screenshot, write to all `out_dirs`:
```python
def save_screenshot(page, name, out_dirs):
    tmp = page.screenshot()
    for d in out_dirs:
        (d / name).write_bytes(tmp)
    print(f"  ✓ {name}")
```

**Error handling**: Wrap each capture in try/except — on failure, print warning and continue (partial output is better than crash).

**After browser.close()**: Print summary of how many PNGs were written and to which directories.

**main() update**: Wire together — call `seed_demo_data()` then `capture_screenshots()` with the two output dirs.

## Notes

- The ephemeral signing keypair is regenerated on each run — the "screenshot-seed-key" signature will accumulate in the DB across runs. This is acceptable for an operator tool.
- Do NOT attempt to start or stop Docker containers — the stack must already be running.
