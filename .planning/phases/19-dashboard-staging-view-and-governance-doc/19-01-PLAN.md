# Plan 19-01: Wave 1 — UI Foundation

**Objective**: Update the core job definition UI components to support the new `status` and `pushed_by` fields, and implement a tabbed view.

## Context
The dashboard needs to handle multiple job lifecycle states and provide a dedicated area for staged (DRAFT) jobs.

## Tasks
1. **Update `JobDefinition` interfaces**:
   - Add `status: string` and `pushed_by?: string` to interfaces in `JobDefinitions.tsx` and `JobDefinitionList.tsx`.
2. **Implement Status Badges in `JobDefinitionList.tsx`**:
   - Create a status-aware badge renderer (DRAFT: yellow, ACTIVE: green, DEPRECATED: zinc, REVOKED: red).
   - Add a "Status" column to the table.
3. **Implement Tabbed View in `JobDefinitions.tsx`**:
   - Introduce a state for the active tab ("active" vs "staging").
   - Filter the `definitions` list based on the active tab (Active tab shows ACTIVE/DEPRECATED/REVOKED; Staging tab shows DRAFT).
   - Use `Tabs` component if available, or a simple button-based toggle.

## Success Criteria
- Job list correctly displays the lifecycle status of every job.
- Switching tabs correctly filters between live and staged jobs.
- The UI remains consistent with the established design system.
