# Plan 19-02: Wave 2 — Staging Features & Publishing

**Objective**: Implement specific features for managing staged jobs, including script inspection and the transition to ACTIVE.

## Context
Staged jobs need to be reviewed and finalized before being published.

## Tasks
1. **Implement Script Inspection**:
   - Add a "View Script" action to `JobDefinitionList.tsx` (visible for all jobs, but highlighted for DRAFTs).
   - Implement a read-only modal or expandable row to display the `script_content` with proper formatting (monospaced).
2. **Implement "Publish" Logic**:
   - Add a "Publish" button to `JobDefinitionList.tsx` (only visible/enabled for DRAFT jobs).
   - This button should trigger a `PATCH` request to `/jobs/definitions/{id}` setting `status` to `ACTIVE`.
   - Ensure the UI refreshes after a successful publish.
3. **Enhance Job Metadata Display**:
   - Show `pushed_by` information in the job list or detail view to provide attribution.

## Success Criteria
- Operators can view the full script of a staged job without entering edit mode.
- One-click publishing from Staging to Active is functional and verified.
- Attribution (`pushed_by`) is visible in the UI.

## Output
Create `.planning/phases/19-dashboard-staging-view-and-governance-doc/19-02-SUMMARY.md` after completion.
