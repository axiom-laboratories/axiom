<objective>
Fix the status filter in the Jobs view to operate server-side so it works correctly across all pages, not just the 50 jobs currently in memory.

Purpose: The UAT found that "you have to jump through to the right page for the filter to work." The root cause is that `filteredJobs` is a `.filter()` over the in-memory array. If the user is on page 2 and selects FAILED, they only see failed jobs on page 2 — they cannot see failed jobs from page 1. The fix is server-side filtering: append `status=X` to the fetch URL, add an optional status param to the backend route and service, and wire the dependency correctly so changing the filter re-fetches from page 0.

Output: Backend accepts status filter in GET /jobs and GET /jobs/count. Frontend passes filter in URL and resets to page 0 on change.
</objective>
