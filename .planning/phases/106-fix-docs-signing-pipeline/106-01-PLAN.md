# Plan 106-01: Fix Docs Signing Pipeline (3 Gaps)

## Objective

Fix 3 documentation bugs in `docs/docs/getting-started/first-job.md` that break the signing/submission flow for users following the published docs. All 3 gaps were identified by the v18.0 milestone audit.

1. **Field name mismatch** (LNX-04 + WIN-05): Replace `signature_key_id` with `signature_id` in both the Linux curl snippet (line 289) and the PowerShell submission snippet (line 338). The server reads `signature_id` at main.py:1098 -- the wrong field name means the countersign block never fires.

2. **TrustAll replacement** (WIN-05): Replace the deprecated TrustAll .NET class pattern (lines 297-304) with `-SkipCertificateCheck` on the Invoke-RestMethod call. Consistent with enroll-node.md which already uses this pattern.

3. **CRLF normalization**: NOT needed in docs. Phase 105 added server-side CRLF normalization (main.py normalizes before both user sig verify and countersign). Per Phase 105 decision, no client-side CRLF normalization is documented.

Output: single file (`first-job.md`) patched with all 3 fixes.

<task id="3">
### Replace TrustAll with -SkipCertificateCheck and fix PowerShell signature_key_id

Two changes in the PowerShell section:

**3a. Replace TrustAll (lines 296-304):**

Remove the entire TrustAll class block:
```powershell
# Disable TLS validation for self-signed cert
add-type @"
    using System.Net;
    using System.Security.Cryptography.X509Certificates;
    public class TrustAll : ICertificatePolicy {
        public bool CheckValidationResult(ServicePoint sp, X509Certificate cert, WebRequest req, int problem) { return true; }
    }
"@
[System.Net.ServicePointManager]::CertificatePolicy = New-Object TrustAll
```

Replace with a comment noting SkipCertificateCheck will be used on each call:
```powershell
# For self-signed certs, use -SkipCertificateCheck on each request below
```

Then add `-SkipCertificateCheck` to the Invoke-RestMethod login call that follows:
```powershell
$response = Invoke-RestMethod -Method POST `
    -Uri "https://<your-orchestrator>:8001/auth/login" `
    -ContentType "application/x-www-form-urlencoded" `
    -Body "username=admin&password=<your-password>" `
    -SkipCertificateCheck
```

**3b. Fix signature_key_id in PowerShell submission (around original line 338):**

Change `signature_key_id` to `signature_id` in the `$body` hashtable:
```powershell
$body = @{
    script_content = $scriptContent
    signature = $SIG
    signature_id = "<your-key-id>"
} | ConvertTo-Json
```

Also add `-SkipCertificateCheck` to the Invoke-RestMethod job submission call:
```powershell
Invoke-RestMethod -Method POST `
    -Uri "https://<your-orchestrator>:8001/jobs" `
    -Headers @{Authorization = "Bearer $TOKEN"} `
    -ContentType "application/json" `
    -Body $body `
    -SkipCertificateCheck
```
