## Tech Debt by Phase

### Phase 132 — Non-Root User Foundation
- **Containerfile.server privilege drop approach:** Uses `entrypoint.py` runtime privilege drop (`os.setuid`) rather than a `USER appuser` Dockerfile directive. Functionally correct — all processes verified running as UID 1000. Entrypoint approach is more flexible (allows volume ownership fix before dropping privileges). Acceptable deviation; low risk.

### Phase 133 — Network & Security Capabilities
- **Documentation imprecision:** Plan and SUMMARY reference "all 9 services" but `compose.server.yaml` contains 7 services (tunnel and ddns-updater removed prior to this phase). Implementation is correct; only documentation is imprecise. No code impact.

### Integration Amber Flags (Design Observations — Not Blockers)
1. **Job container cap_drop scope** — Ephemeral job containers spawned by `runtime.py` don't explicitly set `cap_drop: ALL`; they rely on Docker/Podman runtime defaults. CONT-03 requirement scopes hardening to "orchestrator services", not job containers — by design. Recommend adding a comment to CONT-03 clarifying this scope boundary.
2. **Podman socket UID variance** — `node-compose.podman.yaml` defaults to `/run/user/1000/podman/podman.sock`; path is UID-specific. Operator comment in compose file guides verification. Non-blocking.
3. **Docker GID assumption** — `node-compose.yaml` hardcodes `group_add: ["999"]` for Docker group access. GID varies by Linux distro. Operator comment present. Non-blocking.
