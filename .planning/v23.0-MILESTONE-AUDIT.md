## Integration Gap: Missing Single-Run Endpoint

**Severity:** HIGH  
**Affected requirements:** UI-02 (live status overlay), UI-04 (step drill-in)  
**Affected file:** `puppeteer/dashboard/src/views/WorkflowRunDetail.tsx:67`

```typescript
// WorkflowRunDetail.tsx:67 — this returns 404 in production
const res = await authenticatedFetch(`/api/workflows/${workflowId}/runs/${runId}`);
```

**Root cause:** `GET /api/workflows/{workflow_id}/runs/{run_id}` was never added to `main.py`. The list endpoint (`GET /api/workflows/{workflow_id}/runs`) exists and returns all runs with nested `step_runs`, but there is no per-run detail route.

**Why it slipped through:** Phase 150's verification used mocked `authenticatedFetch` in Vitest. `WorkflowRunDetail.test.tsx` mocks the fetch call and never hits the real API. No E2E Playwright test exercised `WorkflowRunDetail` against the live Docker stack.

**Suggested fix (~20 lines in main.py):**

```python
@app.get("/api/workflows/{workflow_id}/runs/{run_id}", tags=["workflows"], response_model=WorkflowRunResponse)
async def get_workflow_run(
    workflow_id: str,
    run_id: str,
    current_user: User = Depends(require_permission("workflows:read")),
    db: AsyncSession = Depends(get_db)
) -> WorkflowRunResponse:
    """Get a single WorkflowRun with full step detail."""
    run = await db.get(WorkflowRun, run_id)
    if run is None or run.workflow_id != workflow_id:
        raise HTTPException(status_code=404, detail="Run not found")
    from .services.workflow_service import WorkflowService
    return await WorkflowService()._run_to_response(db, run)
```
