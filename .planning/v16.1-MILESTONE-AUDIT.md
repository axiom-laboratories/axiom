### Phase 92: Signing UX

1. **test_signing_ux.py missing from main** — The test file was authored on `feat/usp-signing-ux` (commits 8c02a00 and 6ec4294) but was not included in the squash merge commit `2d6cad8`. The file tests signing error paths (`POST /jobs/definitions` with missing/invalid signatures).

2. **Status code mismatch in test** — The test asserts `status_code == 403` for bad-signature scenarios, but `main.py` raises `422 Unprocessable Entity`. If porting the test to main, the assertion must be updated to `422`.

3. **SIGN_CMD hardcodes `hello.py`** — The `SIGN_CMD` constant in `Signatures.tsx` uses `open("hello.py", "r")` as the script filename rather than a named `YOUR_SCRIPT` placeholder. A fix commit (142e303) was written on the feature branch but not included in the squash merge. Minor UX friction only.

**Recommended action before next release:** Port `test_signing_ux.py` to main with the 403→422 assertion fix; update `SIGN_CMD` placeholder text.
