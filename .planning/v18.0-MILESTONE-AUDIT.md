## Integration Findings

### HIGH: first-job.md PowerShell tabs lost during rebase

**Affected:** WIN-05
**Root cause:** During Phase 104-02 (PR #18 rebase), conflicts on `first-job.md` were resolved with `--ours`, keeping PR #19's Python/curl version. The PowerShell tabs added in Phase 103 (keypair generation, CRLF-normalized signing, Invoke-RestMethod dispatch) were dropped.
**Evidence:** `first-job.md` has zero occurrences of "PowerShell", "Windows", or "Invoke-RestMethod".
**Fix:** Restore PowerShell tabs from Phase 103 commits (103-01 and 103-04 SUMMARYs describe the content).

### MEDIUM: CRLF countersign asymmetry

**Affected:** WIN-05, LNX-04 (CRLF scripts only)
**Root cause:** Server countersign (`main.py` line 1123) signs `script_content.encode("utf-8")` without CRLF normalization. Node verification (`node.py` line 585) normalizes CRLF to LF before `verify()`. Different byte sequences → `SECURITY_REJECTED`.
**Fix:** Add `script_content = script_content.replace('\r\n', '\n').replace('\r', '\n')` before `_sk.sign()` in `main.py`.

### LOW: No confirmed Dwight E2E run

**Affected:** WIN-01, WIN-06 (validation gap, not code defect)
**Root cause:** Dwight was offline during Phase 103-04. All code fixes committed, GHCR images pushed.
**Fix:** Run `python3 mop_validation/scripts/run_windows_e2e.py` when Dwight is available.
