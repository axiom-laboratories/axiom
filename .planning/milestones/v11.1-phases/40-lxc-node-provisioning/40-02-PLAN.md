**Section 5: NODE-05 — Revoke → confirm blocked → reinstate → re-enroll → HEALTHY, new cert serial**

This test operates on axiom-node-dev. Run only if NODE-03 passed (all nodes HEALTHY). If NODE-03 failed, skip NODE-05 with a [FAIL] noting the prerequisite.

Flow:
1. Get node list; find axiom-node-dev entry by env_tag=="DEV". Record `node_id` and `client_cert_pem`.
2. Extract original serial: `from cryptography import x509; orig_cert = x509.load_pem_x509_certificate(client_cert_pem.encode()); orig_serial = orig_cert.serial_number`. If client_cert_pem is None/empty, record `orig_serial = None` and note it for fallback comparison.
3. Revoke: `POST /nodes/{node_id}/revoke`.
4. Confirm REVOKED: poll `GET /api/nodes` until the node's status == "REVOKED" (max 10s / 5 polls). This is the authoritative check — we verify the orchestrator set REVOKED status, which is what blocks /work/pull.
5. Reinstate: `POST /nodes/{node_id}/reinstate`. Confirm 200.
6. Generate fresh token: `POST /admin/generate-token`. Get new token string.
7. Update NODES_SECRETS_DIR/axiom-node-dev.env: write new JOIN_TOKEN= line (keep ENV_TAG, NODE_TAGS, AGENT_URL).
8. Push updated .env to LXC: `subprocess.run(["incus", "file", "push", str(env_file), "axiom-node-dev/home/ubuntu/.env"], check=True)`.
9. Restart compose on LXC: `subprocess.run(["incus", "exec", "axiom-node-dev", "--", "bash", "-c", "cd /home/ubuntu && docker compose down && docker volume rm ubuntu_node_secrets 2>/dev/null || true && docker compose up -d"], check=True)`. Volume removal clears the old cert so the node generates a fresh identity.
10. Poll `GET /api/nodes` until a node with env_tag=="DEV" appears with status=="HEALTHY" (max 120s / 40 polls with 3s sleep). Record `new_node_id` and `new_client_cert_pem`.
11. Assert HEALTHY: check(NODE-05, "Re-enrolled node HEALTHY", new_node.status == "HEALTHY")
12. Assert new cert serial differs from original (or new node_id differs if cert PEM unavailable):
    - If orig_serial is not None and new_client_cert_pem: parse new cert, compare serial numbers
    - Else: compare node IDs (new enrollment creates new node record with different ID if volume was cleared)
    - check("NODE-05", "New cert serial differs from original", serials_differ, detail)
