<task type="auto">
  <name>Task 1: Launch CE install scenario with checkpoint monitoring</name>
  <files></files>
  <action>
Execute the CE install scenario. This task has an interactive monitoring component.

**Before launching Gemini — instruct operator to open a second terminal and run:**
```bash
cd /home/thomas/Development/mop_validation && python3 scripts/monitor_checkpoint.py
```
The monitor polls `/workspace/checkpoint/PROMPT.md` every 30 seconds and prompts the operator when Gemini hits a checkpoint. `monitor_checkpoint.py` requires an interactive terminal — do not automate it.

**Push scenario script to LXC:**
```bash
incus file push \
  /home/thomas/Development/mop_validation/scenarios/ce-install.md \
  axiom-coldstart/workspace/ce-install.md
```

**Launch Gemini with HOME isolation:**
```bash
incus exec axiom-coldstart -- bash -c \
  "source /etc/environment && HOME=/root/validation-home gemini -p \"\$(cat /workspace/ce-install.md)\""
```
The `source /etc/environment` ensures `GEMINI_API_KEY` is available even in a non-login shell. Gemini will run the full scenario. This will take 10-30 minutes.

**Checkpoint intervention protocol (if Gemini raises a checkpoint during the run):**
Gemini writes `/workspace/checkpoint/PROMPT.md`. `monitor_checkpoint.py` surfaces it. For each intervention:
1. Read the PROMPT.md content
2. Draft a steering response — keep it minimal, direct Gemini back to the documented path without solving the problem for it
3. Print the draft to the operator for approval
4. Only after operator types "approved" (or provides edits): push via `incus file push <local_response> axiom-coldstart/workspace/checkpoint/RESPONSE.md`
5. Increment intervention counter

If a 4th checkpoint is requested: do NOT push RESPONSE.md. Instead, `incus exec axiom-coldstart -- bash -c "echo 'Abort: 4th checkpoint reached. Please write /workspace/FRICTION-CE-INSTALL.md as a partial report noting this as a BLOCKER, then exit.' > /workspace/checkpoint/RESPONSE.md"`. Pull the partial FRICTION file if written, and note the BLOCKER in the plan completion notes.

**After Gemini exits:**
Verify the FRICTION file was written:
```bash
incus exec axiom-coldstart -- bash -c "ls -la /workspace/FRICTION-CE-INSTALL.md"
```
If missing: Gemini may have written to a different path. Check: `incus exec axiom-coldstart -- bash -c "ls /workspace/*.md 2>/dev/null"`.
