Read the current `puppets/Containerfile.node`. Replace the entire PowerShell `RUN` block

that contains the MS packages.microsoft.com wget and the `|| echo "PowerShell install skipped"` fallback)
with the following new `RUN` block:

```dockerfile
# Install system deps + PowerShell Core via direct .deb from GitHub releases
# (The Microsoft Debian 12 apt repo uses SHA1 keys which are rejected by Debian — use direct .deb instead)
RUN apt-get update && apt-get install -y --no-install-recommends \
    curl wget apt-transport-https gnupg podman krb5-user iptables docker.io \
    && rm -rf /var/lib/apt/lists/* \
    && wget -q -O /tmp/powershell.deb \
       "https://github.com/PowerShell/PowerShell/releases/download/v7.6.0/powershell-lts_7.6.0-1.deb_amd64.deb" \
    && apt-get install -y /tmp/powershell.deb \
    && rm /tmp/powershell.deb \
    && rm -rf /var/lib/apt/lists/*
```

Key difference from the old block:
- Old: wget MS .deb, `dpkg -i`, then `apt-get install powershell` from the MS repo (fails silently)
- New: wget PowerShell .deb directly from GitHub releases, `apt-get install /tmp/powershell.deb`
  (apt-get resolves dependencies like libicu72 automatically; no repo key needed)

Do NOT change any other part of Containerfile.node (the FROM line, WORKDIR, COPY, requirements
