## Security — CVE Remediation

- [x] **SEC-03**: Axiom ships with `cryptography >= 46.0.7`, resolving the HIGH buffer-overflow CVE flagged on the v23.0 tag *(Phase 165)*
- [x] **SEC-04**: All Dependabot HIGH and MODERATE alerts on the v23.0 tag are resolved and tests pass *(Phase 165)*

## Architecture — Router Modularization

- [x] **ARCH-01**: Backend routes are split into 7 domain APIRouter modules (auth, jobs, nodes, workflows, admin, system, smelter) *(Phase 166)*
- [x] **ARCH-02**: All existing API endpoints function identically after the refactor (zero behavior change, same paths/responses) *(Phase 166)*
- [x] **ARCH-03**: Domain routers support per-domain middleware injection via FastAPI `Depends()` *(Phase 166)*
- [x] **ARCH-04**: Full backend test suite passes with unchanged coverage after the refactor *(Phase 166)*

## Vault Integration (EE)

- [x] **VAULT-01**: EE administrator can configure Vault connection (address + AppRole credentials) via admin UI or env vars *(Phase 167)*
- [x] **VAULT-02**: Platform fetches secrets from Vault at startup with automatic fallback to env vars when Vault is unreachable *(Phase 167)*
- [x] **VAULT-03**: Job dispatch injects Vault-sourced secrets into the execution context without embedding them in the job definition *(Phase 167)*
- [x] **VAULT-04**: Platform actively renews secret leases before expiry during long-running jobs *(Phase 167)*
- [x] **VAULT-05**: Admin dashboard shows Vault connectivity status (healthy / degraded / disabled) *(Phase 167)*
- [x] **VAULT-06**: Platform starts successfully and degrades gracefully when Vault is offline at boot *(Phase 167)*

## SIEM Audit Streaming (EE)

- [x] **SIEM-01**: EE administrator can configure a SIEM destination (webhook URL or syslog host + format) via admin UI *(Phase 168)*
- [x] **SIEM-02**: Audit events are streamed in batches (flush at 100 events or every 5 seconds, whichever comes first) *(Phase 168)*
- [x] **SIEM-03**: SIEM webhook payloads are formatted as CEF *(Phase 168)*
- [x] **SIEM-04**: Sensitive fields (secrets, tokens, passwords) are masked before transmission to the SIEM *(Phase 168)*
- [x] **SIEM-05**: Failed webhook deliveries are retried with exponential backoff and surface an admin alert *(Phase 168)*
- [x] **SIEM-06**: SIEM streaming can be disabled without affecting the local audit log *(Phase 168)*

## Out of Scope (v24.0)

| Feature | Reason |
|---------|--------|
| TPM-based node identity | Axiom nodes run in containers — hardware attestation is impractical |
| Vault agent sidecar | Unnecessary complexity; native hvac AppRole auth is sufficient |
| Per-job TPM attestation | TPM support removed entirely from v24.0 |
| Plugin hot-reload | Memory leak / dangling reference risk; deferred with Plugin SDK |
| Splunk HEC native format | CEF covers Splunk via HTTP Event Collector adapter; native HEC deferred |
