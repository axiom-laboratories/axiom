<task type="auto">
  <name>Task 1: Install build tools and run cibuildwheel</name>
  <files>~/Development/axiom-ee/wheelhouse/</files>
  <action>
Install required build tools and run cibuildwheel to produce compiled wheels.

**Step 1 — Install build tools in the axiom-ee venv (or system pip):**
```bash
pip install cibuildwheel==3.4.0 cython==3.2.4 build twine devpi-client
```

**Step 2 — Verify QEMU is registered (should already be active):**
```bash
ls /proc/sys/fs/binfmt_misc/qemu-aarch64
# If missing, register: docker run --rm --privileged tonistiigi/binfmt --install all
```

**Step 3 — Run the build from the axiom-ee directory:**
```bash
cd ~/Development/axiom-ee
make build
# Equivalent to: cibuildwheel --platform linux .
```

cibuildwheel will:
1. Pull manylinux_2_28 containers (x86_64 and aarch64 via QEMU)
2. Pull musllinux containers for Alpine compatibility
3. Install cython, run setup.py build_ext inside each container
4. Produce tagged wheels in wheelhouse/

**Expected output in wheelhouse/ (12 wheels minimum — manylinux + musllinux × 3 Python × 2 arch):**
```
axiom_ee-0.1.0-cp311-cp311-manylinux_2_28_x86_64.whl
axiom_ee-0.1.0-cp311-cp311-manylinux_2_28_aarch64.whl
axiom_ee-0.1.0-cp311-cp311-musllinux_1_2_x86_64.whl
axiom_ee-0.1.0-cp311-cp311-musllinux_1_2_aarch64.whl
axiom_ee-0.1.0-cp312-cp312-manylinux_2_28_x86_64.whl
... (12 total)
```

**If build fails for a specific platform/Python combination:**
- Check the cibuildwheel log output for the specific error
- `from __future__ import annotations` errors: confirm `language_level=3` is in setup.py compiler_directives
- SQLAlchemy `Mapped[T]` errors: confirm `language_level=3` is set (lazy annotations fix this)
- Module naming errors: confirm `src.replace("/", ".").removesuffix(".py")` in Extension name

**Do NOT skip any platform** — musllinux is required for Alpine-based Containerfile.server.
