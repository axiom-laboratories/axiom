  <name>Task 1: Update docs-deploy.yml — replace mkdocs gh-deploy with ghp-import prefix deploy</name>
  <files>.github/workflows/docs-deploy.yml</files>
  <action>
Replace the final step of `.github/workflows/docs-deploy.yml`. The existing step is:

```yaml
- name: Deploy to GitHub Pages
  working-directory: docs
  run: mkdocs gh-deploy --force
```

Replace it with two steps:

```yaml
- name: Build docs
  working-directory: docs
  run: mkdocs build

- name: Deploy docs to gh-pages/docs/
  working-directory: docs
  run: ghp-import -n -p -f -x docs site
```

Keep all other steps in the workflow unchanged (checkout, git config, setup-python, cache, pip install). `ghp-import` is installed transitively by `pip install -r docs/requirements.txt` — no new dependency is needed.

CRITICAL: Use `-x docs` NOT `--dest-dir docs`. The `--dest-dir` flag does not exist in ghp-import 2.1.0 — it would cause a CI failure. The correct long-form is `--prefix docs` but `-x docs` is preferred (shorter).

IMPORTANT: The `git config` step already sets `user.name` and `user.email` — ghp-import will use these for its git commit. No additional git config needed.
  </action>
