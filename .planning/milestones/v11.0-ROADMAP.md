## Milestone Summary

**Known Gaps:**

- EE-08: `axiom-ee` stub wheel not fully published to PyPI (name reserved via GitHub; full PyPI publish deferred)
- DIST-02: `axiom-ce` Docker Hub publish explicitly deferred to v12.0+; GHCR covers current deployment scenarios

**Key Decisions:**

- Cython chosen over Nuitka — Nuitka multi-module wheel workflow undocumented; Cython is the established standard
- `packages=[]` + `exclude_package_data` pattern to strip `.py` source from wheel
- `__init__.py` excluded from `ext_modules` — CPython bug #59828 breaks relative imports in compiled `__init__`
- Ed25519 offline validation (no call-home) — air-gapped deployments are a core use case
- devpi for internal wheel hosting in Docker Compose — simple HTTP index, no auth needed for internal use

**Issues Resolved:**

- `pkg_resources` deprecation warning eliminated (importlib.metadata)
- async `load_ee_plugins` coroutine was silently discarded — await now correct in lifespan
- `deps.audit()` Base.metadata guard removed — works in both CE and EE mode
- Cython compatibility: async DDL via `run_sync`, `Annotated` params for FastAPI routers
- `pyproject.toml` must not have `packages.find` when `setup.py` controls packages

**Issues Deferred:**

- EE-08: PyPI stub publication (low priority — name reserved)
- DIST-02: Docker Hub CE image (GHCR sufficient for current use)

**Technical Debt Incurred:**

- devpi service added to compose.server.yaml — requires manual wheel upload on EE version bumps
- `_LICENCE_PUBLIC_KEY_BYTES` hardcoded in EE plugin — must be replaced with real key before production EE release
