  <action>
Replace the existing `.env.example` at the repo root in full. The new file uses four grouped sections with `# === Section ===` headers.

**Section structure (in order):**

`# === Required ===`
- `SECRET_KEY=` — JWT signing secret. Comment: generate with `python -c "import secrets; print(secrets.token_hex(32))"`. Note: falls back to weak hardcoded dev value if absent — tokens are insecure without this.
- `ENCRYPTION_KEY=` — Fernet key for secrets at rest. Comment: generate with `python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"`. Note: auto-generates ephemeral key if absent — secrets are lost on container restart.
- `API_KEY=` — Shared API key. Note: **process crashes at import if absent** — no default.
- `ADMIN_PASSWORD=` — Initial admin account password. Comment: only read on first start (user does not yet exist in DB); set to something strong.

`# === Database ===`
- `DATABASE_URL=postgresql+asyncpg://puppet:password@db/puppet_db` — **uncommented with the Compose service name placeholder** (not commented-out, because without it the agent uses SQLite which is dev-only). Add comment: "Default is SQLite (dev only). Use PostgreSQL in production. `db` is the Compose service name from compose.server.yaml."

`# === Optional ===` (all commented-out with `#`)
- `# AGENT_URL=https://your-host` — Public URL used in join tokens; defaults to `https://localhost:8001`
- `# NODE_IMAGE=your-registry/puppet-node:latest` — Node image for Foundry-generated compose files
- `# NODE_EXECUTION_MODE=auto` — auto | direct | docker | podman
- `# AXIOM_LICENCE_KEY=` — Leave blank for Community Edition; paste EE licence key here

`# === Tunnel ===` (all commented-out with `#`)
- `# CLOUDFLARE_TUNNEL_TOKEN=` — Required only if using the Cloudflare tunnel service
- `# SERVER_HOSTNAME=` — Extra SAN added to the TLS cert; used for custom domain access
- `# DUCKDNS_TOKEN=` — Required only if using DDNS auto-update

Do NOT include EE/smelter mirror vars (`MIRROR_DATA_PATH`, `PYPI_MIRROR_URL`, `APT_MIRROR_URL`) — these are advanced EE-only deployment vars not relevant to the standard operator getting-started flow.

Use correct key names as found in source:
- `SECRET_KEY` (from `auth.py` `os.getenv("SECRET_KEY", "super-secret-jwt-key-change-me")`)
- `ENCRYPTION_KEY` (from `security.py`)
- `API_KEY` (from `security.py` `os.environ["API_KEY"]` — no default, crashes if absent)
- `DATABASE_URL` (from `db.py`)
