Scheduling Health metrics:
  fired:   scheduler successfully dispatched the job for this fire slot
  skipped: slot intentionally skipped (DRAFT status or overlapping previous run)
  late:    expected fire slot passed, grace period elapsed, BUT next scheduled fire has NOT yet passed — job is behind but could still fire
  missed:  expected fire slot passed AND the next scheduled fire has also passed — window is gone; investigate
  failed:  dispatched job subsequently reached FAILED status
  grace:   5 minutes — a job fired within 5 min of expected counts as on-time

Health roll-up:
  ok:      no missed and no failed
  warning: late > 0
  error:   missed > 0 or failed > 0

API endpoint:
  GET /api/health/scheduling?window=24h|7d|30d

Retention API:
  GET  /api/admin/retention   → { retention_days: 14, total: N, prunable: M }
  PATCH /api/admin/retention  → body: { "retention_days": 30 }
  Default: 14 days
  Effect: ScheduledFireLog rows older than retention_days are pruned.
          If retention < 30d, the 30d health window shows incomplete data.

Queue view:
  Route: /queue
  Purpose: dedicated view for inspecting the live job queue — pending jobs, queue depth per node, draining status. Separate from the Jobs (/jobs) list page.
