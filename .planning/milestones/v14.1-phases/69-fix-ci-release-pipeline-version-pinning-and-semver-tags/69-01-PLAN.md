  <name>Task 2: Fix release.yml — fetch-depth and Docker metadata tags</name>
  <files>.github/workflows/release.yml</files>
  <action>
Make two targeted edits to .github/workflows/release.yml:

**Edit 1 — Add fetch-depth: 0 to the build-python checkout step.**

The build-python job has this step at line 12:
```yaml
      - uses: actions/checkout@v4
```
Replace it with:
```yaml
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
```
This ensures setuptools-scm can read git tag metadata (shallow clone may miss it in edge
cases). Apply ONLY to the build-python job checkout — the docker-release job checkout does
not need fetch-depth because docker/metadata-action reads GITHUB_REF, not git history.

**Edit 2 — Replace semver tags with type=ref,event=tag in Docker metadata step.**

Find the tags block inside the "Extract Docker metadata" step:
```yaml
          tags: |
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
```
Replace it with:
```yaml
          tags: |
            type=ref,event=tag
```

This passes the raw git tag string (`v14.0`) directly as the Docker image tag without any
semver validation. Combined with `latest=auto` in the flavor block (already present, do not
touch), the image receives both `ghcr.io/axiom-laboratories/axiom:v14.0` and `:latest` tags.

No other changes to release.yml. Do not touch the docker-release checkout, the
publish-testpypi job, the publish-pypi job, or any permissions/environment blocks.
