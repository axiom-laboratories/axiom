<task type="checkpoint:human-action">
  <name>Task 1: External service setup — org, PyPI publishers, GitHub Environments</name>
  <action>
Execute the following steps in order. Each step depends on the previous.

**Step 1 — Create axiom-laboratories GitHub org:**
1. Go to github.com, click the `+` menu, select "New organization"
2. Choose the free plan (or appropriate plan)
3. Set organisation name: `axiom-laboratories`

**Step 2 — Transfer or fork repo under axiom-laboratories:**
- Option A (transfer — preferred): Current repo Settings -> Danger Zone -> Transfer repository -> set new owner to `axiom-laboratories`
- Option B (fork/create new): Create a new repo `axiom` under `axiom-laboratories` and push current main branch to it
- After this step, the repo lives at `github.com/axiom-laboratories/axiom`

**Step 3 — Create GitHub Environments on the repo:**
1. Go to repo Settings -> Environments -> New environment
2. Create environment named exactly: `testpypi` (no protection rules needed for dry-run)
3. Create environment named exactly: `pypi` (add manual approval rule for production safety)

**Step 4 — Configure PyPI pending publisher on test.pypi.org:**
1. Log in to test.pypi.org
2. Go to Account -> Publishing -> "Add a new pending publisher"
3. Fill in:
   - PyPI project name: `axiom-sdk`
   - GitHub owner: `axiom-laboratories`
   - Repository: `axiom`
   - Workflow filename: `release.yml`
   - Environment name: `testpypi`
4. Save the pending publisher

**Step 5 — Configure PyPI pending publisher on pypi.org (production):**
1. Log in to pypi.org
2. Same flow as Step 4 but with:
   - Environment name: `pypi`
3. Save

**Note on OIDC matching:** All four fields (owner, repo, workflow, environment) must match release.yml exactly. Any mismatch causes a 403 at publish time with an unhelpful error message.
  </action>
