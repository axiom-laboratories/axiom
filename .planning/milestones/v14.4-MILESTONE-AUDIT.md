## Tech Debt

### Phase 78 — CLI Signing UX

**TD-01: `prog` string mismatch**
- `mop_sdk/cli.py` line 14: `prog="mop-push"` should be `prog="axiom-push"`
- Impact: `axiom-push --help` prints `usage: mop-push ...` — contradicts all documentation
- Fix: one-line change in cli.py
- Affects: CLI-01, CLI-02, CLI-03

**TD-02: Stale error message in client.py**
- `mop_sdk/client.py` line 158: `"Run 'mop-push login' first."` should say `axiom-push login`
- Impact: cosmetic only — misleads users on auth error
- Fix: one-line change

**TD-03: Stale egg-info entry point**
- `mop_sdk/mop_sdk.egg-info/entry_points.txt` registers `mop-push = mop_sdk.cli:main`
- Impact: installed binary name on developer machines that haven't re-run `pip install -e .` will be `mop-push`
- Fix: `pip install -e .` in the mop_sdk directory regenerates this automatically

**TD-04: Missing base_url fallback in from_store() (migration edge case)**
- `client.py` `from_store()` will raise `KeyError` if stored credentials.json lacks `base_url` (pre-Phase-78 format)
- Impact: only affects developers who had the old CLI installed and haven't re-logged-in
- Fix: add `.get("base_url", os.getenv("AXIOM_URL", "http://localhost:8001"))` fallback

### Phase 80 — GitHub Pages Deploy

**TD-05: SUMMARY file references wrong workflow filenames**
- `80-01-SUMMARY.md` and `80-02-SUMMARY.md` list `docs-deploy.yml` / `homepage-deploy.yml` as modified files
- Actual file: `.github/workflows/gh-pages-deploy.yml`
- Impact: audit trail inconsistency only; deploy mechanics are correct
- Fix: update SUMMARY key-files entries to reference `gh-pages-deploy.yml`
