- [x] **VAL-07**: EE install with post-grace expired licence: DEGRADED_CE mode; `pull_work` returns empty (not 402); running jobs unaffected; no startup crash
- [x] **VAL-08**: EE install with `AXIOM_LICENCE_KEY` absent: CE mode at startup; all EE stubs return 402; no crash
- [x] **VAL-09**: EE install with invalid/tampered licence signature: CE mode at startup; no crash; clear log entry
- [x] **VAL-10**: EE install with tampered wheel manifest (bad SHA256): `_verify_wheel_manifest()` raises `RuntimeError`; EE does not load; CE runs standalone
- [x] **VAL-11**: EE install with non-whitelisted entry point value: loader raises `RuntimeError`; EE does not load
- [x] **VAL-12**: Node limit enforcement: enrollment returns HTTP 402 when active node count ≥ `node_limit`; existing enrolled nodes continue operating
- [x] **VAL-13**: Boot log HMAC chain: clock-rollback scenario raises `RuntimeError` on EE; CE emits warning only; no data loss
- [x] **VAL-14**: All VAL-01 through VAL-13 scenarios covered by automated tests in `mop_validation`; zero scenarios require manual execution to verify
