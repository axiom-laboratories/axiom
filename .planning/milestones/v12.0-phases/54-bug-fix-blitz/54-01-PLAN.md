Purpose: The job detail drawer already has conditional rendering for these fields — it just receives `undefined` because the API never sends them. This one-line-per-field addition is the entire fix.
Output: Updated `job_service.py` with 4 new fields in `list_jobs()` response dict + new pytest test file asserting correct response shape.
