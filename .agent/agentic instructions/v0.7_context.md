# v0.7 Implementation Context

**Date:** 2026-01-16
**Status:** Implementation Complete

## Summary
The system has been upgraded to v0.7, featuring:
1.  **Containerization**: Full Podman support (Server, Node, Dashboard).
2.  **PostgreSQL**: Replaced SQLite for the Agent Service.
3.  **RBAC**: Implemented granular roles (Viewer, Operator, Admin) and JWT Authentication.
4.  **Heartbeats**: Nodes now proactively PUSH telemetry (CPU/RAM) to the Agent.
5.  **Dashboard**: A complete React-based UI with Metrics, Node Grid, and Admin Console.

## Artifacts
- **Dockerfiles**: `Containerfile.server`, `Containerfile.node`, `dashboard/Containerfile`.
- **Compose**: `compose.server.yaml`.
- **Installers**: `install_server.ps1/sh`, `installer/install_node.ps1/sh`.

## Architecture
- **Agent Service**: FastAPI + SQLAlchemy + Postgres. Port 8001.
- **Model Service**: APScheduler + Postgres. Port 8000.
- **Node**: Python + httpx + psutil. Auto-enrolls via `step-ca`.
- **Dashboard**: Vite + React. Port 5173.

## Usage
Start the stack:
```powershell
podman-compose -f compose.server.yaml up -d
```

## Credential Locations
- `secrets/api_key`: Master API Key.
- `secrets/encryption_key`: Fernet Key for DB encryption.
- `secrets/ca_password.txt`: CA Provisioner Password.

## Known State
- Admin User: `admin` / `admin` (Bootstrap).
- SSL: Self-signed certs generated by `install_server.ps1`.
- Node Enrollment: Requires `JOIN_TOKEN` (Generate via Admin Dashboard).
