**Date:** 2026-01-17
**Status:** Stable / Secure
**Architecture:** Token-Embedded Trust

## 🛡️ Core Architecture (v0.8)

We have successfully migrated from insecure SSL bypasses to a **Fully Managed Internal PKI**.

### 1. Internal CA Service
*   **Location:** `agent_service/pki.py`
*   **Behavior:** On startup, the Agent Service generates a persistent Root CA (`secrets/ca/root_ca.crt`) and issues a self-signed server certificate (`secrets/server.crt`).
*   **Persistence:** All keys are stored in the `secrets/` volume, ensuring identity survives container restarts.

### 2. Token-Embedded Trust
*   **Problem:** Nodes need to trust the Server *before* they can safely download configuration.
*   **Solution:** The `JOIN_TOKEN` is now a Base64-encoded JSON blob containing:
    *   `t`: The authentication token.
    *   `ca`: The PEM-encoded Root CA Certificate.
*   **Flow:** The `installer/install_node.ps1` script decodes the token, saves the CA to disk (`bootstrap_ca.crt`), and uses it into `curl --cacert` to strictly validate the initial connection.

### 3. Security Fixes (Critical)
*   **RCE Signature Verification**: The `environment_service/node.py` was patched to restore Ed25519 signature verification. Nodes now **REJECT** unsigned execution tasks.
*   **Installer Hardening**: Added `--ssl-no-revoke` to `curl` commands to resolve Windows Schannel issues with internal CAs (since no CRL exists).
*   **Dashboard**: Updated `Admin.jsx` to gracefully handle browser-blocked SSL requests and prompt the user.
