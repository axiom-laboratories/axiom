# Requirements Integration Map — v20.0 Milestone

## Requirement ENFC-03: Limits Set in Dashboard GUI Reach Inner Container Runtime Flags End-to-End

**Status**: COMPLETE (100%)

### Requirement Definition
Operators set memory and CPU limits in the dashboard dispatch form. These limits flow through the orchestrator, reach nodes via /work/pull, are validated at the node layer, and finally apply to the container runtime flags (--memory, --cpus).

### Integration Path Summary

```
Operator GUI
    ↓
JobDispatchForm (Dashboard)
    ↓
POST /api/jobs (with memory_limit="512m", cpu_limit="0.5")
    ↓
JobCreate model validators (Phase 120)
    ↓
job_service.create_job() (Phase 121)
    ├─ Admission check: parse_bytes("512m") vs node capacity
    └─ Reject with 422 if insufficient, else store in Job table
    ↓
Job persisted in DB with memory_limit, cpu_limit
    ↓
Node polls POST /work/pull
    ↓
job_service.pull_work() (Phase 120-121)
    ├─ Secondary capacity check
    └─ Return WorkResponse with memory_limit, cpu_limit
    ↓
Node receives WorkResponse (JSON)
    ↓
execute_task() (Phase 122)
    ├─ Extract memory_limit, cpu_limit from job dict
    ├─ Validate formats with parse_bytes(), parse_cpu()
    └─ Report structured errors if invalid
    ↓
runtime.run(memory_limit="512m", cpu_limit="0.5") (Phase 122)
    ↓
Container runtime applies: --memory 512m --cpus 0.5
```

### Requirement Satisfaction Verification

**Dashboard GUI input**: ✓
- Memory limit field in dispatch form (JobDispatchForm)
- CPU limit field in dispatch form (JobDispatchForm)
- Both optional (nullable)
- Accepted formats: "512m", "1g", "1Gi", "0.5", "2"

**Reach orchestrator**: ✓
- JobCreate model accepts both fields with validators
- Pydantic field_validator regex patterns enforce format
- create_job() parses limits with parse_bytes() and parse_cpu()

**Reach nodes**: ✓
- WorkResponse includes memory_limit and cpu_limit
- Populated in pull_work() from Job DB row
- Serialized as JSON in /work/pull response
- Deserialized by node.py into job dict

**Node validation**: ✓
- execute_task() extracts limits from job dict
- parse_bytes() validates memory format
- parse_cpu() validates CPU format
- Invalid formats return structured errors
- Valid formats pass through to runtime.run()

**Reach container runtime**: ✓
- runtime.run() receives memory_limit and cpu_limit parameters unchanged
- Parameters applied to container runtime: --memory, --cpus
- Docker and Podman both support these flags

### Requirements with No Cross-Phase Wiring

None. ENFC-03 has complete cross-phase integration spanning phases 120, 121, and 122.
