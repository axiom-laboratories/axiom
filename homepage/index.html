Zero-trust by default

Security without compromise

Every node enrolls by generating a CSR on first boot. The control plane signs a short-lived client certificate against its internal Root CA. From that point forward, all communication between the puppeteer and every puppet is mutually authenticated with TLS — no shared secrets, no API keys passed over the wire, no trust on first use.

Job scripts must be signed with a registered Ed25519 key before the node will execute them. The node fetches the registered public key, verifies the signature against the script bytes, and refuses outright if verification fails. Bad signatures are not logged and retried — they are rejected.
